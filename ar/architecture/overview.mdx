# نظرة عامة على البنية

OpenYida هي أداة CLI مبنية على Node.js بتصميم معماري متعدد الطبقات يجمع بشكل عضوي بين قدرات الذكاء الاصطناعي وتكامل منصة Yida وتجربة المطور.

## سير العمل العام

![سير عمل OpenYida](/images/architecture-workflow.png)

سير عمل OpenYida يتكون من 6 مراحل:

1. **User Requirement** - المستخدم يصف المتطلبات باللغة الطبيعية
2. **AI Understanding** - الذكاء الاصطناعي يفهم ويحلل متطلبات المستخدم
3. **PRD Document** - إنشاء مستند متطلبات المنتج
4. **CLI Execution** - تنفيذ أوامر CLI لإنشاء التطبيق
5. **Yida Platform** - بناء التطبيق على منصة Yida
6. **Online Access** - التطبيق يصبح متاحاً عبر الإنترنت

## تدفق المصادقة

![تدفق المصادقة](/images/architecture-auth.png)

تدفق مصادقة OpenYida:

1. **config.json** - تخزين معلومات التكوين
2. **Login Session** - إنشاء جلسة تسجيل الدخول
3. **Cookie Cache** - تخزين معلومات المصادقة مؤقتاً
4. **API Request** - إرسال طلبات API
5. **Yida Backend** - معالجة الواجهة الخلفية Yida
6. **Page/Form Output** - إخراج الصفحة/النموذج

## بنية CLI

![بنية CLI](/images/architecture-cli.png)

تفاعل OpenYida CLI مع منصة Yida:

- **AI Coding Tool** → **OpenYida CLI**
  - Login (تسجيل الدخول)
  - Create App (إنشاء تطبيق)
  - Create Page (إنشاء صفحة)
  - Publish (نشر)
  - Get Schema (الحصول على الهيكل)
- **Yida Low-Code Platform**
  - App Management (إدارة التطبيقات)
  - Form Pages (صفحات النماذج)
  - Custom Pages (صفحات مخصصة)

## نظام المهارات

![نظام المهارات](/images/architecture-skills.png)

تنقسم مهارات OpenYida إلى ثلاث فئات:

### مهارات تسجيل الدخول
- `login` - تسجيل الدخول إلى منصة Yida
- `logout` - تسجيل الخروج

### مهارات الإنشاء
- `create-app` - إنشاء تطبيق
- `create-page` - إنشاء صفحة
- `create-form` - إنشاء نموذج
- `get-schema` - الحصول على هيكل البيانات

### مهارات النشر
- `publish` - نشر التطبيق

## مبادئ التصميم الأساسية

### 1. التصميم المعياري

كل وحدة وظيفية مغلفة بشكل مستقل وتتفاعل من خلال واجهات واضحة:

- **طبقة الأوامر**: تعالج مدخلات CLI، تستدعي الخدمات الأساسية
- **طبقة الخدمات**: تغلف منطق الأعمال، توفر API موحد
- **طبقة المحولات**: واجهة مع الأنظمة الخارجية (AI، منصة Yida)

### 2. امتداد الإضافات

نظام المهارات يدعم التحميل الديناميكي والامتداد:

```typescript
// تعريف واجهة Skill
interface Skill {
  name: string;
  version: string;
  activate(context: SkillContext): void;
  deactivate(): void;
}
```

### 3. مدفوع بالتكوين

يدعم تكوين متعدد البيئات مع تبديل مرن:

- تكوين عالمي محلي (~/.openyida/)
- تكوين على مستوى المشروع (.openyidarc.json)
- تجاوز متغيرات البيئة

### 4. أمان الأنواع

دعم أنواع TypeScript على السلسلة الكاملة لتقليل أخطاء وقت التشغيل.

## تدفق البيانات

### تدفق إنشاء التطبيق

```
User Input → AI Service → Code Generation → Yida API → Application
                ↑                              ↓
           PRD Analysis                  Validation
                ↑                              ↓
           Template Selection           Deployment
```

### تدفق عمليات البيانات

```
User Command → Config Load → Auth Check → Yida API → Data Response
                                              ↓
                                         Cache/Store
```

## المكدس التقني

| الطبقة | التقنية |
|------|------|
| وقت التشغيل | Node.js 18+ |
| اللغة | TypeScript 5.x |
| إطار عمل CLI | Commander.js |
| عميل HTTP | Axios |
| SDK الذكاء الاصطناعي | OpenAI SDK, Anthropic SDK |
| إدارة التكوين | Cosmiconfig |
| التسجيل | Winston |
| الاختبارات | Jest |

## تصميم الأمان

### أمان المصادقة

- تدفق OAuth 2.0 القياسي
- آلية التحديث التلقائي للرموز
- التخزين المشفر محلياً للمعلومات الحساسة

### نقل البيانات

- HTTPS على السلسلة الكاملة
- التحقق من توقيع API
- التحكم في إعادة المحاولة والمهلة للطلبات

## تحسين الأداء

- **تجمع الاتصالات**: إعادة استخدام اتصالات HTTP
- **التخزين المؤقت الذكي**: تخزين مؤقت محلي للتكوين والبيانات الوصفية
- **التحديثات التزايدية**: نقل المحتوى المتغير فقط
- **التحكم في التزامن**: تحديد مكالمات API المتزامنة
