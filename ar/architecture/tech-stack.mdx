## بنية المكدس التقني

![بنية المكدس التقني](/images/architecture-tech-stack.png)

ينقسم المكدس التقني لـ OpenYida إلى جزأين: بيئة التطوير ومنصة السحابة:

### بيئة التطوير (Development Environment)
- **Node.js** - بيئة التشغيل
- **Git** - التحكم في الإصدارات

### الاختبارات الآلية (Playwright)
- أتمتة المتصفح
- اختبارات من الطرف إلى الطرف
- مقارنة لقطات شاشة الصفحات

### منصة Yida السحابية (YiDa Cloud Platform)
- **App** - إدارة التطبيقات
- **Form** - صفحات النماذج
- **Custom Page** - صفحات مخصصة

## تفاصيل الوحدات الأساسية

### التدويل (i18n.js)

نظام تدويل يدعم 12 لغة:

```javascript
const { t, getLanguage, setLanguage } = require('./i18n');

// استخدام الترجمة
console.log(t('login.success'));
console.log(t('create_app.usage', 'appName'));

// الحصول/تعيين اللغة
console.log(getLanguage()); // 'zh'
setLanguage('en');
```

**أولوية اكتشاف اللغة:**
1. متغير البيئة `OPENYIDA_LANG`
2. متغيرات بيئة النظام `LANG` / `LC_ALL`
3. الافتراضي: الصينية المبسطة

### اكتشاف البيئة (env.js)

يكتشف تلقائيًا بيئة أداة AI قيد التشغيل:

| طريقة الاكتشاف | الأداة |
|----------|------|
| اكتشاف وجود الدليل | Claude Code, Cursor, OpenCode, Aone Copilot, Qoder |
| اكتشاف الملف الخاص | Wukong (悟空) |

### محرك التشخيص (doctor.js)

تصميم بنية ثلاثي الطبقات:

1. **DiagnosticEngine** - مجدول التشخيص
2. **EnvironmentChecker** - مدقق البيئة
3. **ApplicationChecker** - مدقق التطبيق
4. **FixEngine** - محرك الإصلاح الذكي

## تفاصيل الوحدات الوظيفية

### وحدة المصادقة (auth/)

```
auth/
├.js          # تسجيل الدخول إلى Yida (تخزين Cookie مؤقت + مسح رمز QR)
├.js           # إدارة حالة تسجيل الدخول
├.js            # إدارة المنظمة
└-login.js       # تسجيل الدخول برمز QR على الطرفية
```

**عملية تسجيل الدخول:**
1. اكتشاف بيئة أداة AI الحالية
2. اختيار طريقة تسجيل الدخول وفقًا للبيئة (مسح رمز QR / CDP / قراءة الملف)
3. الحصول على Cookie وتخزينها في التخزين المؤقت المحلي المشفر
4. تستخدم الأوامر اللاحقة تلقائيًا حالة تسجيل الدخول المخزنة مؤقتًا

### وحدة الموصل (connector/)

```
connector/
├.js                    # تغليف API موصل Yida
├-parser.js            # محلل أوامر curl
├-generator.js       # مولد الإجراءات
├-generator.js         # مولد الأوصاف
├-smart-create.js # إدخال الإنشاء الذكي
└
```

**عملية الإنشاء الذكي:**
```
أمر curl → المحلل → استخراج معلومات API → اكتشاف المصادقة → مطابقة الموصل الموجود → إنشاء التكوين → إنشاء/تحديث
```

### وحدة إدارة البيانات (data-management.js)

واجهة عمليات بيانات موحدة، تدعم:

| نوع المورد | العمليات |
|----------|------|
| form | query, get, create, update |
| process | query, get, create |
| task | query, execute |
| subform | query |

## استدعاءات API لـ Yida

### التكوين الأساسي

```javascript
const BASE_URL = 'https://www.aliwork.com';

// رؤوس الطلب
const headers = {
  'Content-Type': 'application/json',
  'X-Csrf-Token': csrfToken,
  'Cookie': cookies,
};
```

### APIs الأساسية

| نقطة النهاية | الاستخدام |
|------|------|
| `/dingtalk/web/APP_xxx/xxx` | الواجهة داخل التطبيق |
| `/dingtalk/web/APP_xxx/form/getFormDataList` | استعلام بيانات النموذج |
| `/dingtalk/web/APP_xxx/form/saveFormData` | حفظ بيانات النموذج |
| `/dingtalk/web/APP_xxx/proc/getProcessInstances` | استعلام مثيلات العملية |

## تصميم الأمان

1. **تخزين Cookie مشفر** - التخزين المؤقت المحلي مخزن مشفر
2. **التحقق من Token CSRF** - يحمل كل طلب Token
3. **لا توجد بيانات اعتماد مشفرة ثابتة** - يتم الحصول على جميع بيانات الاعتماد ديناميكيًا
4. **عزل البيئة** - بيئات أدوات AI المختلفة مستقلة

## تحسين الأداء

1. **تحميل ملفات الترجمة كسول** - تحميل حزم اللغة حسب الطلب
2. **تخزين Schema مؤقت محلي** - تقليل استدعاءات API
3. **التحكم في الطلبات المتزامنة** - تجنب التقييد بسبب الطلبات الزائدة
4. **التحديثات التزايدية** - تستخدم تحديثات النموذج الدمج بدلاً من الكتابة فوق

## آلية التوسع

### إضافة أمر جديد

1. إنشاء `new-feature.js` في `lib/`
2. تسجيل توجيه الأمر في `bin/yida.js`
3. تحديث `README.md` والوثائق
4. إضافة مفتاح التدويل

### إضافة لغة جديدة

1. إنشاء `xx.js` في `lib/core/locales/`
2. ترجمة جميع المفاتيح (باستخدام `zh.js` كأساس)
3. تحديث `SUPPORTED_LANGUAGES` في `i18n.js`
4. تشغيل `npm run validate-i18n` للتحقق من الاكتمال
