## الإنشاء الذكي للموصلات

أنشئ الموصلات بأمر curl واحد فقط:

```bash
openyida connector smart-create \
  --curl "curl 'https://api.example.com/v1/users' -H 'Authorization: Bearer xxx'" \
  --name "موصل إدارة المستخدمين" \
  --desc "موصل HTTP لإدارة بيانات المستخدمين"
```

عملية الإنشاء الذكي:

1. **تحليل معلومات API** - استخراج البروتوكول و Host والطريقة والمسار ونوع المصادقة
2. **مطابقة مع الموصلات الموجودة** - البحث عن الموصلات الموجودة بنفس Host والمصادقة
3. **إنشاء تكوين الموصل** - بناء تعريف الموصل بتنسيق OpenAPI تلقائيًا
4. **إنشاء أو تحديث** - إنشاء موصل جديد أو إضافة إجراء إلى موصل موجود

## أوامر إدارة الموصلات

### سرد الموصلات

```bash
openyida connector list
openyida connector list --page 1 --size 20
```

### إنشاء موصل

```bash
openyida connector create "اسم الموصل" "api.example.com" \
  --operations ./operations.json \
  --protocol https \
  --desc "وصف الموصل"
```

<Note>
يتم تحويل حقل `method` في ملف JSON الخاص بالعمليات تلقائيًا إلى أحرف صغيرة (مثلاً، `"POST"` يصبح `"post"`). إذا لم يتم تحديد `method`، فإن القيمة الافتراضية هي `"get"`.
</Note>

### عرض تفاصيل الموصل

```bash
openyida connector detail <connector-id>
```

### حذف الموصل

```bash
openyida connector delete <connector-id> --force
```

## إدارة الإجراءات

### إضافة إجراء تنفيذ

```bash
openyida connector add-action \
  --connector-id <id> \
  --operations ./new-action.json \
  --confirm
```

<Note>
يتم تحويل حقل `method` في إعدادات العمليات تلقائيًا إلى أحرف صغيرة (مثلاً، `"POST"` يصبح `"post"`). يمكنك استخدام أي حالة أحرف. إذا لم يتم تحديده، فإن القيمة الافتراضية هي `"get"`.
</Note>

### سرد الإجراءات الموجودة

```bash
openyida connector list-actions <connector-id>
```

### اختبار الإجراء

```bash
openyida connector test \
  --connector-id <id> \
  --action <actionId> \
  --body '{"key": "value"}'
```

### حذف الإجراء

```bash
openyida connector delete-action <connector-id> <operation-id> --force
```

## إدارة حسابات المصادقة

### سرد حسابات المصادقة

```bash
openyida connector list-connections <connector-id>
```

### إنشاء حساب مصادقة

```bash
openyida connector create-connection <connector-id> "حساب الإنتاج" \
  --token "Bearer xxx" \
  --type bearer
```

```bash
openyida connector gen-template ./my-api-template.json
```

## أفضل الممارسات

### 1. استخدام متغيرات البيئة للمعلومات الحساسة

```bash
export API_TOKEN="رمزك-السري"
openyida connector smart-create --curl "curl -H 'Authorization: Bearer $API_TOKEN' ..."
```

### 2. إنشاء الموصلات المعقدة على مراحل

```bash
# الخطوة 1: إنشاء الموصل الأساسي
openyida connector create "API DingTalk" "oapi.dingtalk.com"

# الخطوة 2: إضافة الإجراءات واحدًا تلو الآخر
openyida connector add-action --connector-id <id> --operations ./action1.json
openyida connector add-action --connector-id <id> --operations ./action2.json
```

### 3. الاختبار قبل التفعيل

```bash
# إنشاء ولكن لا تؤكد
openyida connector add-action --connector-id <id> --operations ./action.json

# اختبار الإجراء
openyida connector test --connector-id <id> --action <actionId>

# تأكيد生效
# إكمال التأكيد النهائي في خلفية Yida
```

## الأسئلة الشائعة

<AccordionGroup>
  <Accordion title="ماذا أفعل إذا فشل تحليل أمر curl؟">
    تأكد من أن أمر curl بتنسيق صحيح، يتضمن URL كامل و headers ضرورية. إذا كان يحتوي على أحرف خاصة، استخدم علامات الاقتباس.
  </Accordion>

  <Accordion title="كيف أحدث إجراءات موصل موجود؟">
    استخدم الأمر `connector add-action`، إذا كان معرف الإجراء موجودًا بالفعل، سيسألك النظام عما إذا كنت تريد التحديث.
  </Accordion>

  <Accordion title="ما طرق المصادقة المدعومة؟">
    يدعم حاليًا Bearer Token و API Key و Basic Auth و OAuth2. يتم التعرف على معلومات المصادقة في أوامر curl تلقائيًا.
  </Accordion>
</AccordionGroup>
