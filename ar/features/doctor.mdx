## الاستخدام الأساسي

### تشغيل التشخيص

```bash
openyida doctor
```

### الإصلاح التلقائي

```bash
openyida doctor --fix
```

### إنشاء تقرير

```bash
# تنسيق Markdown
openyida doctor --report markdown

# تنسيق JSON
openyida doctor --report json

# تنسيق HTML
openyida doctor --report html
```

## عناصر فحص البيئة

| العنصر | الوصف | الإصلاح التلقائي |
|--------|------|----------|
| إصدار CLI | تحقق من تطابق إصدار openyida مع SKILL.md | ✅ |
| تهيئة project/ | تحقق من وجود دليل عمل project/ | ✅ |
| إصدار Node.js | تحقق إذا ≥ 18 | ❌ |
| Playwright | تحقق إذا تم التثبيت | ✅ |
| config.json | تحقق من ملف التكوين | ✅ |
| تسجيل دخول Yida | تحقق من حالة تسجيل الدخول | ✅ |
| اتصال الشبكة | تحقق من اتصال aliwork.com | ❌ |

---

## عناصر تشخيص التطبيق

| العنصر | الوصف |
|--------|------|
| ملف PRD | تحقق إذا كان مستند المتطلبات موجودًا |
| كود مصدر الصفحات | تحقق من ملفات كود المصدر و console.log والملفات الفارغة |
| ذاكرة Schema المؤقتة | تحقق من تنسيق ملف الذاكرة المؤقتة |
| React Hooks | تحقق من معايير استخدام Hooks |

## وضع مراقبة الصحة

ابدأ المراقبة المستمرة، تحقق كل دقيقة:

```bash
openyida doctor --monitor
```

<Warning>
ميزة مراقبة الصحة حاليًا في الإصدار التجريبي. قد تتغير واجهة برمجة التطبيقات وتنسيقات الإخراج في الإصدارات المستقبلية.
</Warning>

مثال الإخراج:

```
[14:32:01] الصحة: 95/100  ناجح: 18/19 | أخطاء: 0 | تحذيرات: 1

[14:33:01] الصحة: 100/100➡
  ناجح: 19/19 | أخطاء: 0 | تحذيرات: 0
```

اضغط `Ctrl+C` لإيقاف المراقبة.

## التشخيص عبر الإنترنت

التشخيص عبر الإنترنت للتطبيقات المنشورة بالفعل:

```bash
openyida doctor --production --app <appId>
```

محتوى الفحص:
- صلاحية معرف التطبيق
- سجلات الأخطاء عبر الإنترنت
- مقاييس الأداء

## التذاكر و VOC

### إنشاء تذكرة (Bug)

```bash
openyida doctor --create-ticket
```

إدخال تفاعلي للعنوان والوصف، إرسال تلقائي إلى GitHub Issues.

### إنشاء VOC (متطلب)

```bash
openyida doctor --create-voc
```

### إرسال ذكي

يحكم النظام تلقائيًا ما إذا كان Bug أو متطلب:

```bash
openyida doctor --auto-submit
```

## مثال تقرير التشخيص

### إخراج وحدة التحكم

```
🔍 فحص تبعيات بيئة OpenYida...

✅ إصدار openyida: 2026.04.02 (SKILL المتوقع: 2026.04.02)
✅ دليل عمل project/ تم تهيئته
✅ Node.js v18.19.0 (مطلوب ≥ 18)
✅ Playwright مثبت
✅ config.json موجود والتنسيق صحيح
✅ حالة تسجيل دخول Yida: مسجل الدخول
✅ اتصال الشبكة (aliwork.com)
✅ ملفات PRD (ملفان)
✅ كود مصدر الصفحات (3 ملفات)
✅ ذاكرة Schema المؤقتة (5 ملفات)
✅ معايير React Hooks

11 فحص ناجح، لم يتم العثور على مشاكل
```

<Note>
إنشاء التقارير بتنسيق إخراج مخصص هو ميزة تجريبية. قد يتم تحسين مخططات إخراج HTML و JSON في التحديثات المستقبلية.
</Note>

### تقرير Markdown

```bash
openyida doctor --report markdown
```

تم حفظ التقرير في `.cache/reports/doctor-2024-01-15T10-30-00-000Z.md`

## الفحص المسبق

قم بتشغيل الفحص تلقائيًا قبل العمليات الرئيسية:

### الفحص قبل النشر

```bash
# تشغيل تلقائي
openyida publish يشغل الفحص المسبق تلقائيًا
```

### الفحص قبل الإنشاء

```bash
# تشغيل تلقائي
openyida create-app يشغل الفحص المسبق تلقائيًا
```

## أنواع الإصلاح

| النوع | الوصف | مثال |
|------|------|------|
| `AUTO` | إصلاح تلقائي | إنشاء config.json، حذف الذاكرة المؤقتة التالفة |
| `COMMAND` | إصلاح بالأمر | تشغيل `npx playwright install` |
| `MANUAL` | إصلاح يدوي | تثبيت gh CLI، كتابة PRD |

## حساب الصحة

صيغة حساب درجة الصحة (0-100):

```
الصحة = معدل النجاح × 100 - الأخطاء × 10 - التحذيرات × 3
```

| الدرجة | الحالة |
|------|------|
| 90-100 | ممتاز
| 70-89 | جيد
| 50-69 | عادي
| < 50 | يحتاج تحسين🔴
