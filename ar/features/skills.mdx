يقدم OpenYida أوامر CLI غنية، تدعم إدارة دورة حياة التطبيق الكاملة.

---

## البيئة والمصادقة

```bash
# فحص البيئة
openyida env

# تسجيل الدخول
openyida login [--qr]              # --qr استخدام رمز QR في المحطة
openyida login --wukong            # خاص ببيئة Wukong

# تسجيل الخروج
openyida logout

# إدارة حالة تسجيل الدخول
openyida auth status               # عرض حالة تسجيل الدخول
openyida auth login                # تنفيذ تسجيل الدخول
openyida auth refresh              # تحديث حالة تسجيل الدخول
openyida auth logout               # تسجيل الخروج

# إدارة المنظمات
openyida org list                  # سرد المنظمات القابلة للوصول
openyida org switch --corp-id <id> # التبديل إلى المنظمة

# تهيئة دليل العمل
openyida copy [--force]            # نسخ دليل project

# أدوات التشخيص
openyida doctor [--fix]            # تشخيص البيئة
openyida doctor --monitor          # مراقبة الصحة
openyida doctor --report <format>  # إنشاء تقرير (json|markdown|html)
```

## إدارة التطبيقات

```bash
# إنشاء تطبيق
openyida create-app "<الاسم>" [desc] [icon] [color] [themeColor]

# تصدير التطبيق
openyida export <appType> [output] # إنشاء حزمة الترحيل

# استيراد التطبيق
openyida import <file> [name]      # إعادة بناء التطبيق من حزمة الترحيل
```

## إدارة النماذج

```bash
# إنشاء نموذج
openyida create-form create <appType> "<اسم النموذج>" <JSON الحقول> \
  [--layout <التخطيط>] \
  [--theme <الموضوع>] \
  [--label-align <المحاذاة>]

# تحديث النموذج
openyida create-form update <appType> <formUuid> <JSON التعديلات>

# الحصول على Schema النموذج
openyida get-schema <appType> <formUuid>

# تحديث تكوين النموذج
openyida update-form-config <appType> <formUuid> <isRenderNav> <title>
```

### أنواع الحقول المدعومة

يقبل الأمر `create-form` أنواع الحقول التالية فقط. استخدام نوع غير مدعوم يُرجع خطأً.

#### الحقول الأساسية

حقول مدمجة لا تتطلب أي إضافات.

| النوع | الوصف |
|-------|-------|
| `TextField` | نص سطر واحد |
| `TextareaField` | نص متعدد الأسطر |
| `NumberField` | رقم |
| `RateField` | تقييم |
| `DateField` | تاريخ |
| `RadioField` | أزرار اختيار |
| `SelectField` | قائمة منسدلة (اختيار فردي) |
| `CheckboxField` | مربعات اختيار |
| `MultiSelectField` | قائمة منسدلة (اختيار متعدد) |
| `CascadeSelectField` | اختيار متتالي |
| `CascadeDateField` | تاريخ متتالي |
| `EmployeeField` | محدد الأعضاء |
| `DepartmentSelectField` | محدد القسم |
| `AssociationFormField` | نموذج مرتبط |
| `AssociationQuery` | استعلام مرتبط |
| `AttachmentField` | مرفق |
| `ImageField` | صورة |
| `AddressField` | عنوان |
| `CountrySelectField` | محدد البلد/المنطقة |
| `TableField` | نموذج فرعي / تفاصيل |
| `SerialNumberField` | رقم تسلسلي |

#### الحقول المتقدمة

حقول مدمجة بوظائف موسعة. قد تتطلب بعض الميزات التفعيل.

| النوع | الوصف |
|-------|-------|
| `EditorField` | محرر نصوص غنية |
| `DigitalSignatureField` | توقيع رقمي |
| `LocationField` | موقع GPS |

#### حقول الإضافات

تتطلب هذه الحقول تثبيت الإضافة المقابلة من مركز الإضافات قبل الاستخدام.

| النوع | الإضافة المطلوبة |
|-------|-------------------|
| `TimeZoneDateField` | مكون المنطقة الزمنية الدولية |
| `CC_AIButtonField` | مكون الذكاء الاصطناعي |
| `CC_PG_ESignField` | مكون المالية الذكية |
| `CC_TeambitionProjectLinksField` | إضافة Teambition |
| `CC_TeambitionTaskLinksField` | إضافة Teambition |
| `CC_MoneyField` | مكون المالية الذكية |
| `CC_PhoneNumberField` | مكون التوسعة |
| `CC_ChineseIdField` | مكون التوسعة |
| `YidaDingtalkAgentField` | إضافة وكيل DingTalk |

<Warning>
ستفشل حقول الإضافات إذا لم تكن الإضافة المطلوبة مثبتة في مؤسستك. قم بتثبيت الإضافات من مركز الإضافات قبل استخدام أنواع الحقول هذه.
</Warning>

<Warning>
أنواع الحقول مثل `MobileField` و `EmailField` غير مدعومة وسيتم رفضها. استخدم فقط الأنواع المدرجة أعلاه.
</Warning>

## إدارة البيانات

```bash
# إدارة البيانات الموحدة
openyida data <action> <resource> [args]

# بيانات النموذج
openyida data query form <appType> <formUuid> [--page N] [--size N] [--search-json JSON]
openyida data get form <appType> --inst-id <id>
openyida data create form <appType> <formUuid> --data <JSON>  # مفاتيح JSON يمكن أن تكون معرّفات حقول أو تسميات حقول
openyida data update form <appType> --inst-id <id> --data <JSON>

# بيانات العملية
openyida data query process <appType> <formUuid> [--instance-status STATUS]
openyida data get process <appType> --process-inst-id <id>
openyida data create process <appType> <formUuid> --process-code <code> --data <JSON>

# إدارة المهام
openyida data query tasks <appType> --type <todo|done|submitted|cc>
openyida data execute task <appType> --task-id <id> --process-inst-id <id> --out-result <AGREE|DISAGREE>

# النماذج الفرعية
openyida data query subform <appType> <formUuid> --inst-id <id> --table-field-id <fieldId>
```

## مركز المهام العالمي

استعلم عن بيانات مركز المهام العالمي للمستخدم الحالي دون تحديد `appType`.

```bash
# المهام المعلقة
openyida task-center todo [--page N] [--size N] [--keyword TEXT]

# أنشأتها
openyida task-center created [--page N] [--size N] [--keyword TEXT] [--no-detail]

# المعالجة
openyida task-center done [--page N] [--size N] [--keyword TEXT]

# نسخة إلي
openyida task-center cc [--page N] [--size N] [--keyword TEXT]

# تقديم بالنيابة
openyida task-center proxy [--page N] [--size N] [--keyword TEXT]
```

### أنواع المهام

| النوع | الوصف | نقطة نهاية API |
|-------|-------|---------------|
| `todo` | مهام معلقة تحتاج إلى إجراء | `getTodoTasksInCorp` |
| `created` | عمليات بدأتها أنت | `getMyCreateInCorp` |
| `done` | مهام تمت معالجتها | `getDoneTasksInCorp` |
| `cc` | مهام تم نسخك فيها | `getNotifyMeInCorp` |
| `proxy` | عمليات قُدمت بالنيابة | `getSubmitAgentInCorp` |

### المعلمات الاختيارية

| المعلمة | الوصف | القيمة الافتراضية |
|---------|-------|-----------------|
| `--page N` | رقم الصفحة | `1` |
| `--size N` | عناصر لكل صفحة (الحد الأقصى 100) | `20` |
| `--keyword TEXT` | كلمة البحث | — |
| `--no-detail` | حذف التفاصيل (فقط لنوع `created`) | — |

<Info>
  على عكس `openyida data query tasks`، يستعلم أمر `task-center` عن بيانات مركز المهام العالمي ولا يتطلب معلمة `appType`.
</Info>

## إدارة الموصلات

```bash
# سرد الموصلات
openyida connector list [--page N] [--size N]

# إنشاء موصل
openyida connector create "<الاسم>" "<النطاق>" --operations <file> [خيارات]

# عرض تفاصيل الموصل
openyida connector detail <connector-id>

# حذف الموصل
openyida connector delete <connector-id> [--force]

# إدارة الإجراءات
openyida connector add-action --connector-id <id> --operations <file> [--confirm]
openyida connector list-actions <connector-id>
openyida connector delete-action <connector-id> <operation-id> [--force]
openyida connector test --connector-id <id> --action <actionId> [خيارات]

# إدارة حسابات المصادقة
openyida connector list-connections <connector-id>
openyida connector create-connection <connector-id> "<الاسم>" [خيارات]

# الإنشاء الذكي
openyida connector smart-create --curl "<أمر curl>" [--name <name>] [--desc <desc>]

# أدوات مساعدة
openyida connector parse-api --curl "<أمر curl>" --output <file>
openyida connector gen-template [مسار الإخراج]
```

## الخيارات العامة

| الخيار | الوصف |
|------|------|
| `--help`, `-h` | عرض معلومات المساعدة |
| `--version`, `-v` | عرض رقم الإصدار |
| `--qr` | استخدام رمز QR في المحطة لتسجيل الدخول |
| `--force` | فرض العملية، تخطي التأكيد |
| `--skip-lint` | تخطي فحص معايير ترميز Yida أثناء `publish` |
