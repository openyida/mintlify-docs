## 1. كتابة مطالبات مفصلة

جودة التطبيقات المُنشأة تتناسب طرديًا مع مستوى تفصيل المطالبات. مقارنة:

<CodeGroup>

```text غامض (تجنب)
إنشاء CRM
```

```text مفصل (موصى به)
إنشاء CRM مع ثلاثة نماذج مرتبطة:
1. نموذج العملاء المحتملين — الحقول: الاسم، الشركة، الهاتف، البريد الإلكتروني، المصدر (قائمة منسدلة: إحالة/موقع ويب/فعالية/مكالمة باردة)، الحالة (قائمة منسدلة: عميل محتمل جديد/تم الاتصال/مؤهل/غير مؤهل)، مُعيّن إلى (محدد العضو)
2. نموذج العميل — محوّل من العملاء المحتملين، الحقول: اسم الشركة، الصناعة، الإيرادات السنوية، جهة الاتصال الرئيسية، مدير الحساب
3. نموذج الفرصة — مرتبط بالعملاء، الحقول: اسم الفرصة، المبلغ، المرحلة (قائمة منسدلة: اقتراح/تفاوض/فوز/خسارة)، تاريخ الإغلاق المتوقع، ملاحظات
```

</CodeGroup>

## 2. استخدام مستندات المتطلبات

للتطبيقات المعقدة، اكتب PRD أولاً، ثم أعطه للذكاء الاصطناعي:

```
إنشاء تطبيق Yida بناءً على مستند المتطلبات التالي:

[لصق PRD — أسماء الحقول، الأنواع، قواعد التحقق، سير العمل، إلخ]
```

هذا النهج ينتج نتائج أكثر دقة من المطالبات التكرارية.

---

## 3. التكرار التدريجي

لا تحاول بناء كل شيء في مطالبة واحدة. ابنِ الهيكل الأساسي أولاً، ثم أضف التعقيد تدريجيًا:

```
الخطوة 1: إنشاء نموذج معلومات الموظف الأساسي
الخطوة 2: إضافة نموذج طلب إجازة مرتبط بالموظفين
الخطوة 3: تكوين سير عمل الموافقة على الإجازة
الخطوة 4: إضافة لوحة تحكم المدير تعرض الموافقات المعلقة
الخطوة 5: نشر لوحة التحكم كصفحة عامة
```

---

## 4. قم دائمًا بتشغيل get-schema قبل عمليات البيانات

معرّفات الحقول في Yida يتم إنشاؤها عشوائيًا بواسطة المنصة (مثال: `textField_eftt1aa5m`) ولا يمكن تخمينها من أسماء الحقول. قبل إنشاء أو تحديث أو استعلام البيانات بشروط الحقول، قم دائمًا بتشغيل:

```bash
openyida get-schema <appType> <formUuid>
```

بعد كتابة البيانات، تحقق من النتيجة باستعلام سجل واحد على الأقل:

```bash
openyida data query form <appType> <formUuid> --size 1
```

تأكد من أن حقول `formData` تحتوي على قيم فعلية. إذا كانت جميع القيم فارغة، فإن معرّفات الحقول غير صحيحة — أعد تشغيل `get-schema` وصحّح.

---

## 5. التحقق من كل خطوة

بعد أن ينشئ الذكاء الاصطناعي كل مكون، اجعله يتحقق:

```
عرض Schema الحالي لنموذج [اسم النموذج]
تحقق مما إذا كانت صفحة [اسم الصفحة] منشورة ويمكن الوصول إليها
```

اكتشف المشكلات مبكرًا قبل أن تتراكم إلى مشكلات أكبر.

---

## 6. إعادة استخدام دليل عمل المشروع

قم بتشغيل `openyida copy` مرة واحدة فقط لكل مشروع، وليس في كل جلسة. يخزن ملف `project/config.json` `appId` و `corpId` — يستخدمه الذكاء الاصطناعي لمعرفة التطبيق المستهدف. لا تحذف هذا الملف في منتصف المشروع.

---

## 7. تسجيل الدخول الخاص بالبيئة

تتطلب أدوات الذكاء الاصطناعي المختلفة طرق تسجيل دخول مختلفة:

| أداة الذكاء الاصطناعي | أمر تسجيل الدخول |
|---------|----------|
| Claude Code, Cursor, VSCode | `openyida login` |
| Wukong | `openyida login --wukong` |
| OpenCode, Aone Copilot | `openyida login` |

عند تبديل الأدوات، قم بتشغيل `openyida env` أولاً لتأكيد البيئة النشطة.

---

## 8. استخدام CDN لصور الصفحات المخصصة

عندما تحتوي الصفحات المخصصة على صور، قم بتحميلها إلى CDN أولاً لضمان العرض الموثوق:

```bash
openyida cdn-config   # تكوين Alibaba Cloud OSS + CDN (مرة واحدة)
openyida cdn-upload ./my-image.png   # الحصول على URL CDN مستقر
```

ثم أشر إلى URL CDN في مطالبة صفحتك:
```
أضف صورة بانر بطل إلى صفحة الهبوط باستخدام URL CDN هذا: https://cdn.example.com/banner.png
```

---

## 9. التعامل مع إعدادات المؤسسات المتعددة

عندما يمكنك الوصول إلى مؤسسات Yida متعددة:

```bash
openyida org list          # عرض جميع المؤسسات المتاحة
openyida org switch --corp-id <id>   # التبديل إلى المؤسسة المستهدفة
openyida env               # تأكيد التبديل
```

تحقق دائمًا من المؤسسة النشطة قبل إنشاء التطبيقات لتجنب البناء في المساحة الخاطئة.

---

## 10. تشغيل Doctor قبل التصحيح

قبل قضاء وقت في تصحيح الأوامر الفاشلة، قم بتشغيل:

```bash
openyida doctor --fix
```

يكتشف ويحل تلقائيًا المشكلات الأكثر شيوعًا: الجلسات منتهية الصلاحية، ملفات التكوين المفقودة، عدم تطابق البيئة.

---

## 11. PRs المدعومة بالذكاء الاصطناعي مرحب بها

عند المساهمة في OpenYida نفسه، اقبل تمامًا الكود المُنشأ بالذكاء الاصطناعي. اذكر أداة الذكاء الاصطناعي المستخدمة في وصف PR — هذا يساعد المشرفين على فهم سياق الإنشاء ومراجعة أكثر كفاءة.
