OpenYida ist für die Interaktion in natürlicher Sprache konzipiert. Die folgenden Prompts wurden getestet und können in allen unterstützten KI-Tools verwendet werden.

---

## Formular- und Seitenoperationen

### Formularseiten erstellen

```
Helfen Sie mir, eine xxx-Formularseite zu erstellen
```
```
Erstellen Sie ein Urlaubsantragsformular, einschließlich: Mitarbeitername, Abteilung, Startdatum, Enddatum, Urlaubsgrund
```

### Formularfelder hinzufügen

```
Fügen Sie ein Feld [Feldname] zur Seite [Seitenname] hinzu, Feldtyp: [TextField/NumberField/DateField/SelectField/...]
```
```
Fügen Sie ein Dropdown-Feld "Abteilung" zum Mitarbeiter-Informationsformular hinzu, Optionen: Ingenieurabteilung, Marketingabteilung, Vertriebsabteilung, Personalabteilung
```

<Note>
Nur [unterstützte Feldtypen](/de/features/skills#unterstützte-feldtypen) werden akzeptiert. Nicht unterstützte Typen (z.B. `MobileField`, `EmailField`) werden abgelehnt.
</Note>

### Feldeigenschaften ändern

```
Machen Sie das Feld [Feldname] auf der Seite [Seitenname] erforderlich
```
```
Machen Sie das Feld [Feldname] auf der Seite [Seitenname] schreibgeschützt
```
```
Ändern Sie die Bezeichnung des Felds [Feldname] auf der Seite [Seitenname] in [neue Bezeichnung]
```

### Benutzerdefinierte Anzeigeseiten

```
Erstellen Sie eine benutzerdefinierte Anzeigeseite für die Seite [Seitenname]
```
```
Generieren Sie eine Landing Page für ein Unternehmenskollaborationsprodukt
```

Erfordert Anmeldung. Der Befehl liest Meeting-Transkripte und verwendet KI, um strukturierte PRD-Dokumente zu generieren. Die Ausgabe wird im Verzeichnis `prd/` gespeichert.

## Integration & Automatisierung

### Integrationsfluss erstellen

```
Einen Integrationsfluss erstellen, der Benachrichtigungen sendet, wenn neue Datensätze eingereicht werden
```

```
Eine Automatisierung auf [Formularname] einrichten, um [Team/Benutzer] bei neuen Einreichungen zu benachrichtigen
```

```
Einen [Formularname]-Workflow erstellen, der E-Mail-Benachrichtigungen an [Empfänger] auslöst
```

### Automatisierte Aktionen hinzufügen

```
Eine Datenabfrage-Aktion zur Integration [Flussname] hinzufügen
```

```
Eine Datensatzerstellungsaktion hinzufügen, um automatisch Einträge in [Zielformular] zu erstellen
```

<Note>
Integrationsflüsse werden bei Formularereignissen (Einfügen, Aktualisieren, Löschen) ausgelöst und können Benachrichtigungen, Datenabfragen und Datensatzerstellungen ausführen. Verwenden Sie die CLI, um Flüsse zu erstellen und zu konfigurieren.
</Note>

```

```

```

```


### Erstellen & Verwalten

```
Eine Aufgabe erstellen: [Aufgabenbeschreibung]
```

```

```

### Anwesenheit abfragen

```
Anwesenheitsdatensätze auflisten
```

```
Meinen Anwesenheitsstatus überprüfen
```

<Info>
Der Befehl `dws` erfordert eine separate Installation der DingTalk-CLI. Führen Sie `openyida dws install` aus, falls noch nicht installiert. Die Ausgabe erfolgt standardmäßig im Tabellenformat; fügen Sie `-f json` für JSON-Ausgabe hinzu.
</Info>
