OpenYida bietet ein einheitliches Datenverwaltungs-CLI, das vollständige CRUD-Operationen für Formulardaten, Prozessinstanzen, Aufgaben, Unterformulare usw. unterstützt.

## Befehlsformat

```bash
openyida data <aktion> <ressource> [args]
```

- **aktion**: `query` | `get` | `create` | `update` | `execute`
- **ressource**: `form` | `process` | `task` | `subform` | `operation-records`

## Formulardatenverwaltung

### Formulardaten abfragen

```bash
# Basisabfrage
openyida data query form <appType> <formUuid>

# Paginierte Abfrage
openyida data query form <appType> <formUuid> --page 1 --size 50

# Bedingte Abfrage
openyida data query form <appType> <formUuid> \
  --search-json '{"field_xxx": "value"}'

# Spezifische Instanz abfragen
openyida data query form <appType> <formUuid> --inst-id <formInstId>
```

### Einzelne Daten abrufen

```bash
openyida data get form <appType> --inst-id <formInstId>
```

### Daten erstellen

```bash
openyida data create form <appType> <formUuid> \
  --data '{"field_xxx": "value", "field_yyy": 123}' \
  --dept-id <departmentId>
```

Sie können im `--data`-Parameter auch Feldbezeichnungen anstelle von Feld-IDs verwenden. Das CLI erkennt Feldbezeichnungen automatisch und wandelt sie durch Abrufen des Formularschemas in die entsprechenden Feld-IDs um.

```bash
# Feldbezeichnungen anstelle von Feld-IDs verwenden
openyida data create form <appType> <formUuid> \
  --data '{"Name": "Max", "Alter": 30, "Abteilung": "Engineering"}'
```

<Info>
  Die Auflösung von Feldbezeichnungen erfolgt durch Abgleich des Bezeichnungstextes mit dem Formularschema. Sowohl Feldbezeichnungen als auch Feld-IDs können im selben `--data`-Objekt verwendet werden — Schlüssel, die keiner bekannten Bezeichnung entsprechen, werden unverändert weitergegeben.
</Info>

### Daten aktualisieren

```bash
openyida data update form <appType> \
  --inst-id <formInstId> \
  --data '{"field_xxx": "new value"}' \
  --use-latest-version y
```

## Prozessdatenverwaltung

### Prozessinstanzen abfragen

```bash
# Prozessliste abfragen
openyida data query process <appType> <formUuid> \
  --page 1 --size 20 \
  --instance-status RUNNING

# Verfügbare Status: RUNNING (laufend), COMPLETED (abgeschlossen), TERMINATED (beendet)
```

### Prozessdetails abrufen

```bash
openyida data get process <appType> --process-inst-id <processInstanceId>
```

### Prozessinstanz erstellen

```bash
openyida data create process <appType> <formUuid> \
  --process-code <processCode> \
  --data '{"field_xxx": "value"}' \
  --dept-id <departmentId>
```

### Genehmigungsdatensätze abfragen

```bash
openyida data query operation-records <appType> \
  --process-inst-id <processInstanceId>
```

## Aufgabenverwaltung

### Aufgabenliste abfragen

```bash
# Zu erledigende Aufgaben
openyida data query tasks <appType> --type todo

# Erledigte Aufgaben
openyida data query tasks <appType> --type done

# Von mir eingereichte
openyida data query tasks <appType> --type submitted

# In Kopie
openyida data query tasks <appType> --type cc
```

### Aufgabe ausführen

```bash
openyida data execute task <appType> \
  --task-id <taskId> \
  --process-inst-id <processInstanceId> \
  --out-result AGREE \
  --remark "Zustimmung, bitte fortfahren" \
  --data '{"comment": "Genehmigung erfolgreich"}'
```

**out-result-Optionen:**
- `AGREE` - Zustimmen
- `DISAGREE` - Nicht zustimmen

## Unterformularverwaltung

### Unterformulardaten abfragen

```bash
openyida data query subform <appType> <formUuid> \
  --inst-id <formInstId> \
  --table-field-id <fieldId> \
  --page 1 --size 20
```

## Erweiterte Verwendung

### Komplexe Abfragebedingungen

```bash
# Mehrfeld-Abfrage
openyida data query form <appType> <formUuid> \
  --search-json '{
    "field_status": "approved",
    "field_amount": { "$gte": 1000 },
    "field_created_at": { "$gte": "2024-01-01" }
  }'
```

### Batch-Operationsskriptbeispiel

```bash
#!/bin/bash
# Batch-Export von Formulardaten

APP_TYPE="APP_XXXXXXXX"
FORM_UUID="FORM-XXXX-XXXX"
PAGE=1

while true; do
  RESULT=$(openyida data query form $APP_TYPE $FORM_UUID --page $PAGE --size 100)
  
  # Aktuelle Seitendaten verarbeiten
  echo "$RESULT" >> ./export.json
  
  # Prüfen, ob es eine nächste Seite gibt
  HAS_MORE=$(echo "$RESULT" | jq '.hasMore')
  if [ "$HAS_MORE" != "true" ]; then
    break
  fi
  
  PAGE=$((PAGE + 1))
done
```

## Datenexportbeispiel

```bash
# Daten eines bestimmten Zeitraums exportieren
openyida data query form <appType> <formUuid> \
  --search-json '{
    "field_created_at": {
      "$gte": "2024-01-01T00:00:00",
      "$lte": "2024-12-31T23:59:59"
    }
  }' \
  --size 100 > ./data-export.json
```

## Vorsichtsmaßnahmen

<Warning>
  Datenoperationen sind dauerhaft, Lösch- und Aktualisierungsvorgänge können nicht rückgängig gemacht werden. Es wird empfohlen, vor der Ausführung von Batch-Operationen kleine Tests durchzuführen.
</Warning>

<Info>
  Paginierte Abfragen geben standardmäßig 20 Ergebnisse pro Seite zurück, maximal 100. Für den Export großer Datenmengen wird die Verwendung von Schleifen-Pagination empfohlen.
</Info>
