OpenYida verfügt über ein leistungsstarkes Diagnosetool `doctor`, das Entwicklern hilft, die Umgebung zu überprüfen, Probleme zu diagnostizieren, automatisch zu reparieren und die Gesundheit von Anwendungen kontinuierlich zu überwachen.

## Grundlegende Verwendung

### Diagnose ausführen

```bash
openyida doctor
```

### Automatische Reparatur

```bash
openyida doctor --fix
```

### Bericht generieren

```bash
# Markdown-Format
openyida doctor --report markdown

# JSON-Format
openyida doctor --report json

# HTML-Format
openyida doctor --report html
```

## Umgebungsprüfungselemente

| Prüfelement | Beschreibung | Automatische Reparatur |
|--------|------|----------|
| Node.js-Version | Prüfen, ob ≥ 16 | ❌ |
| Python-Version | Prüfen, ob ≥ 3.10 | ❌ |
| Playwright | Installation prüfen | ✅ |
| Chromium | Browserinstallation prüfen | ✅ |
| gh CLI | GitHub CLI prüfen | ❌ |
| gh login | gh CLI-Verbindungsstatus prüfen | ✅ |
| config.json | Konfigurationsdatei prüfen | ✅ |
| Skills | Skill-Paket-Installation prüfen | ❌ |
| Yida-Verbindung | Verbindungsstatus prüfen | ✅ |
| Netzwerkkonnektivität | aliwork.com-Konnektivität prüfen | ❌ |

---

## Anwendungsdiagnoseelemente

| Prüfelement | Beschreibung |
|--------|------|
| PRD-Datei | Existenz des Anforderungsdokuments prüfen |
| Seitenquellcode | Quelldateien, console.log, leere Dateien prüfen |
| Schema-Cache | Cache-Dateiformat prüfen |
| React Hooks | Hooks-Nutzungsstandards prüfen |

## Gesundheitsüberwachungsmodus

Starten Sie die kontinuierliche Überwachung, prüfen Sie jede Minute:

```bash
openyida doctor --monitor
```

Beispielausgabe:

```
[14:32:01] Gesundheit: 95/100 📈
  Erfolgreich: 18/19 | Fehler: 0 | Warnungen: 1

[14:33:01] Gesundheit: 100/100 ➡️
  Erfolgreich: 19/19 | Fehler: 0 | Warnungen: 0
```

Drücken Sie `Ctrl+C`, um die Überwachung zu stoppen.

## Online-Diagnose

Online-Diagnose für bereits veröffentlichte Anwendungen:

```bash
openyida doctor --production --app <appId>
```

Prüfungsinhalt:
- Gültigkeit der Anwendungs-ID
- Online-Fehlerprotokolle
- Leistungskennzahlen

## Tickets und VOC

### Ticket erstellen (Bug)

```bash
openyida doctor --create-ticket
```

Geben Sie interaktiv Titel und Beschreibung ein, senden Sie automatisch an GitHub Issues.

### VOC erstellen (Bedarf)

```bash
openyida doctor --create-voc
```

### Intelligente Übermittlung

Das System bestimmt automatisch, ob es sich um einen Bug oder einen Bedarf handelt:

```bash
openyida doctor --auto-submit
```

## Vorprüfung

Führen Sie automatisch die Prüfung vor wichtigen Operationen aus:

### Prüfung vor Veröffentlichung

```bash
# Automatisch ausgeführt
openyida publish führt automatisch die Vorprüfung aus
```

### Prüfung vor Erstellung

```bash
# Automatisch ausgeführt
openyida create-app führt automatisch die Vorprüfung aus
```

## Reparaturtypen

| Typ | Beschreibung | Beispiel |
|------|------|------|
| `AUTO` | Automatische Reparatur | config.json erstellen, beschädigten Cache löschen |
| `COMMAND` | Befehlsreparatur | `pip install playwright` ausführen |
| `MANUAL` | Manuelle Reparatur | gh CLI installieren, PRD schreiben |

## Gesundheitsberechnung

Formel zur Berechnung des Gesundheitswerts (0-100):

```
Gesundheit = Erfolgsrate × 100 - Fehleranzahl × 10 - Warnungsanzahl × 3
```

| Wert | Zustand |
|------|------|
| 90-100 | Ausgezeichnet 🟢 |
| 70-89 | Gut 🟡 |
| 50-69 | Durchschnitt 🟠 |
| < 50 | Verbesserungsbedarf 🔴 |
