## Bericht erstellen

### Grundlegende Verwendung

```bash
openyida create-report <appType> "Berichtsname" '<Diagrammdefinition JSON>'
```

### Beispiel: Verkaufsbericht erstellen

```bash
openyida create-report APP_XXXXXX "Monatlicher Verkaufsbericht" '{
  "charts": [
    {
      "type": "bar",
      "title": "Monatliche Verkäufe",
      "dataSource": "form_sales",
      "xField": "month",
      "yField": "amount"
    },
    {
      "type": "pie",
      "title": "Produktvertriebsanteil",
      "dataSource": "form_sales",
      "categoryField": "product_type",
      "valueField": "amount"
    }
  ]
}'
```

### Diagrammdefinition aus Datei laden

```bash
openyida create-report APP_XXXXXX "Jahresstatistikbericht" ./charts-config.json
```

## Diagrammdefinitionsformat

### Balkendiagramm

```json
{
  "type": "bar",
  "title": "Balkendiagrammtitel",
  "dataSource": "form_uuid",
  "xField": "category_field",
  "yField": "value_field",
  "aggregation": "sum"
}
```

### Liniendiagramm

```json
{
  "type": "line",
  "title": "Trenddiagrammtitel",
  "dataSource": "form_uuid",
  "xField": "date_field",
  "yField": "value_field",
  "aggregation": "sum"
}
```

### Kreisdiagramm

```json
{
  "type": "pie",
  "title": "Anteildiagrammtitel",
  "dataSource": "form_uuid",
  "categoryField": "category_field",
  "valueField": "value_field"
}
```

### Datentabelle

```json
{
  "type": "table",
  "title": "Datendetails",
  "dataSource": "form_uuid",
  "columns": [
    { "field": "name", "title": "Name" },
    { "field": "amount", "title": "Betrag" },
    { "field": "date", "title": "Datum" }
  ]
}
```

## Diagramm zum Bericht hinzufügen

```bash
openyida append-chart <appType> <reportId> '<Diagrammdefinition JSON>'
```

### Beispiel

```bash
openyida append-chart APP_XXXXXX REPORT_XXXXX '{
  "type": "line",
  "title": "Wachstumstrend",
  "dataSource": "form_sales",
  "xField": "month",
  "yField": "growth_rate"
}'
```


## Diagrammtyp-Referenz

| Typ | Beschreibung | Erforderliche Felder |
|------|------|----------|
| `bar` | Balkendiagramm | xField, yField |
| `line` | Liniendiagramm | xField, yField |
| `pie` | Kreisdiagramm | categoryField, valueField |
| `table` | Datentabelle | columns |
| `area` | Flächendiagramm | xField, yField |
| `scatter` | Streudiagramm | xField, yField |

---

## Aggregationsfunktionen

| Funktion | Beschreibung |
|------|------|
| `sum` | Summe |
| `avg` | Durchschnitt |
| `count` | Anzahl |
| `max` | Maximum |
| `min` | Minimum |

## Best Practices

1. **Wählen Sie den Diagrammtyp vernünftig** - Trend mit Liniendiagramm, Vergleich mit Balkendiagramm, Anteil mit Kreisdiagramm
2. **Kontrollieren Sie die Anzahl der Diagramme** - Es wird empfohlen, nicht mehr als 6 Diagramme pro Bericht zu haben
3. **Verwenden Sie Datentabellen** - Ergänzen Sie Diagramme mit Tabellen, um detaillierte Daten anzuzeigen, was die Überprüfung der ursprünglichen Werte erleichtert
4. **Legen Sie eine angemessene Paginierung fest** - Für Datentabellen wird empfohlen, 10-20 Elemente pro Seite anzuzeigen
