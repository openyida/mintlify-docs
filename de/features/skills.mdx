OpenYida bietet umfangreiche CLI-Befehle, die die Verwaltung des vollständigen Lebenszyklus von Anwendungen unterstützen.

---

## Umgebung und Authentifizierung

```bash
# Umgebungserkennung
openyida env

# Anmeldung
openyida login [--qr]              # --qr verwendet Terminal-QR-Code
openyida login --wukong            # Spezifisch für Wukong-Umgebung

# Abmeldung
openyida logout

# Verwaltung des Anmeldestatus
openyida auth status               # Anmeldestatus anzeigen
openyida auth login                # Anmeldung ausführen
openyida auth refresh              # Anmeldestatus aktualisieren
openyida auth logout               # Abmelden

# Organisationsverwaltung
openyida org list                  # Zugängliche Organisationen auflisten
openyida org switch --corp-id <id> # Zur Organisation wechseln

# Arbeitsverzeichnis initialisieren
openyida copy [--force]            # project-Verzeichnis kopieren

# Diagnosetool
openyida doctor [--fix]            # Umgebungsdiagnose
openyida doctor --monitor          # Gesundheitsüberwachung
openyida doctor --report <format>  # Bericht generieren (json|markdown|html)
```

## Anwendungsverwaltung

```bash
# Anwendung erstellen
openyida create-app "<Name>" [desc] [icon] [color] [themeColor]

# Anwendung exportieren
openyida export <appType> [output] # Migrationspaket generieren

# Anwendung importieren
openyida import <file> [name]      # Anwendung aus Migrationspaket wiederherstellen
```

## Formularverwaltung

```bash
# Formular erstellen
openyida create-form create <appType> "<Formularname>" <FelderJSON> \
  [--layout <Layout>] \
  [--theme <Thema>] \
  [--label-align <Ausrichtung>]

# Formular aktualisieren
openyida create-form update <appType> <formUuid> <ÄnderungenJSON>

# Formular-Schema abrufen (gibt Feld-IDs und Berichts-Feldcodes zurück)
openyida get-schema <appType> <formUuid>

# Formularkonfiguration aktualisieren
openyida update-form-config <appType> <formUuid> <isRenderNav> <title>
```

### Unterstützte Feldtypen

Der Befehl `create-form` akzeptiert nur die folgenden Feldtypen. Die Verwendung eines nicht unterstützten Typs gibt einen Fehler zurück.

#### Basisfelder

Integrierte Felder, die keine zusätzlichen Plugins erfordern.

| Typ | Beschreibung |
|-----|--------------|
| `TextField` | Einzeiliger Text |
| `TextareaField` | Mehrzeiliger Text |
| `NumberField` | Zahl |
| `RateField` | Bewertung |
| `DateField` | Datum |
| `RadioField` | Optionsfelder |
| `SelectField` | Dropdown (Einzelauswahl) |
| `CheckboxField` | Kontrollkästchen |
| `MultiSelectField` | Dropdown (Mehrfachauswahl) |
| `CascadeSelectField` | Kaskadierende Auswahl |
| `CascadeDateField` | Kaskadierendes Datum |
| `EmployeeField` | Mitgliedauswahl |
| `DepartmentSelectField` | Abteilungsauswahl |
| `AssociationFormField` | Verknüpftes Formular |
| `AssociationQuery` | Verknüpfte Abfrage |
| `AttachmentField` | Anhang |
| `ImageField` | Bild |
| `AddressField` | Adresse |
| `CountrySelectField` | Land/Region-Auswahl |
| `TableField` | Unterformular / Detail |
| `SerialNumberField` | Seriennummer |

#### Erweiterte Felder

Integrierte Felder mit erweiterten Funktionen. Einige Funktionen erfordern möglicherweise eine Aktivierung.

| Typ | Beschreibung |
|-----|--------------|
| `EditorField` | Rich-Text-Editor |
| `DigitalSignatureField` | Digitale Signatur |
| `LocationField` | GPS-Standort |

#### Plugin-Felder

Diese Felder erfordern die Installation des entsprechenden Plugins aus dem Plugin-Center vor der Verwendung.

| Typ | Erforderliches Plugin |
|-----|-----------------------|
| `TimeZoneDateField` | Internationale Zeitzonen-Komponente |
| `CC_AIButtonField` | KI-Komponente |
| `CC_PG_ESignField` | Smart-Finance-Komponente |
| `CC_TeambitionProjectLinksField` | Teambition-Plugin |
| `CC_TeambitionTaskLinksField` | Teambition-Plugin |
| `CC_MoneyField` | Smart-Finance-Komponente |
| `CC_PhoneNumberField` | Erweiterungs-Komponente |
| `CC_ChineseIdField` | Erweiterungs-Komponente |
| `YidaDingtalkAgentField` | DingTalk-Agent-Plugin |

<Warning>
Plugin-Felder schlagen fehl, wenn das erforderliche Plugin nicht in Ihrer Organisation installiert ist. Installieren Sie Plugins aus dem Plugin-Center, bevor Sie diese Feldtypen verwenden.
</Warning>

<Warning>
Feldtypen wie `MobileField` und `EmailField` werden nicht unterstützt und abgelehnt. Verwenden Sie nur die oben aufgeführten Typen.
</Warning>

## Datenverwaltung

```bash
# Einheitliche Datenverwaltung
openyida data <aktion> <ressource> [args]

# Formulardaten
openyida data query form <appType> <formUuid> [--page N] [--size N] [--search-json JSON]
openyida data get form <appType> --inst-id <id>
openyida data create form <appType> <formUuid> --data <JSON>  # JSON-Schlüssel können Feld-IDs oder Feldbezeichnungen sein
openyida data update form <appType> --inst-id <id> --data <JSON>

# Prozessdaten
openyida data query process <appType> <formUuid> [--instance-status STATUS]
openyida data get process <appType> --process-inst-id <id>
openyida data create process <appType> <formUuid> --process-code <code> --data <JSON>

# Aufgabenverwaltung
openyida data query tasks <appType> --type <todo|done|submitted|cc>
openyida data execute task <appType> --task-id <id> --process-inst-id <id> --out-result <AGREE|DISAGREE>

# Unterformulare
openyida data query subform <appType> <formUuid> --inst-id <id> --table-field-id <fieldId>
```

## Globales Aufgabenzentrum

Fragen Sie die globalen Aufgabenzentrumsdaten des aktuellen Benutzers ab, ohne einen `appType` anzugeben.

```bash
# Offene Aufgaben
openyida task-center todo [--page N] [--size N] [--keyword TEXT]

# Von mir erstellt
openyida task-center created [--page N] [--size N] [--keyword TEXT] [--no-detail]

# Erledigt
openyida task-center done [--page N] [--size N] [--keyword TEXT]

# In Kopie
openyida task-center cc [--page N] [--size N] [--keyword TEXT]

# Stellvertretend eingereicht
openyida task-center proxy [--page N] [--size N] [--keyword TEXT]
```

### Aufgabentypen

| Typ | Beschreibung | API-Endpunkt |
|-----|-------------|-------------|
| `todo` | Ausstehende Aufgaben | `getTodoTasksInCorp` |
| `created` | Von Ihnen gestartete Prozesse | `getMyCreateInCorp` |
| `done` | Bereits bearbeitete Aufgaben | `getDoneTasksInCorp` |
| `cc` | Aufgaben in Kopie | `getNotifyMeInCorp` |
| `proxy` | Stellvertretend eingereichte Prozesse | `getSubmitAgentInCorp` |

### Optionale Parameter

| Parameter | Beschreibung | Standardwert |
|-----------|-------------|-------------|
| `--page N` | Seitennummer | `1` |
| `--size N` | Einträge pro Seite (max. 100) | `20` |
| `--keyword TEXT` | Suchbegriff | — |
| `--no-detail` | Details weglassen (nur für Typ `created`) | — |

<Info>
  Im Gegensatz zu `openyida data query tasks` fragt der Befehl `task-center` globale Aufgabenzentrumsdaten ab und benötigt keinen `appType`-Parameter.
</Info>

## Flash-Notiz zu PRD

Konvertiert DingTalk-Flash-Notizen (Besprechungstranskripte) mithilfe von KI in strukturierte PRD-Dokumente.

```bash
# Aus Datei lesen
openyida flash-to-prd --file <Dateipfad> [--name <Projektname>] [--max-tokens <Anzahl>]

# Aus Standardeingabe lesen
cat meeting.txt | openyida flash-to-prd --name <Projektname> [--max-tokens <Anzahl>]
```

### Parameter

| Parameter | Kurzform | Beschreibung | Standard |
|-----------|----------|--------------|----------|
| `--file` | `-f` | Pfad zur Flash-Notiz-Datei | — |
| `--name` | `-n` | Projektname (wird automatisch erkannt, wenn nicht angegeben) | — |
| `--max-tokens` | — | Maximale Tokenanzahl für die KI-Antwort | `8000` |

Der Befehl liest den Inhalt der Flash-Notiz aus der angegebenen Datei oder der Standardeingabe, verarbeitet den Text vor (einschließlich Extraktion von Besprechungsmetadaten und Sprecheridentifikation) und ruft den Yida-KI-Dienst auf, um ein strukturiertes PRD zu generieren. Die PRD-Datei wird im `prd/`-Verzeichnis Ihres Projekts gespeichert.

### Ausgabe

Der Befehl gibt ein JSON-Ergebnis auf stdout aus:

```json
{
  "success": true,
  "projectName": "Mein Projekt",
  "prdFile": "/path/to/prd/Mein-Projekt.md",
  "contentLength": 4096,
  "meetingRecognition": {
    "metaFields": 3,
    "a1Sections": 2,
    "speakers": 5
  }
}
```

<Info>
  Sie müssen angemeldet sein, bevor Sie diesen Befehl ausführen. Führen Sie zuerst `openyida login` aus, wenn Sie noch nicht authentifiziert sind.
</Info>

## Globale Optionen

| Option | Beschreibung |
|------|------|
| `--help`, `-h` | Hilfeinformationen anzeigen |
