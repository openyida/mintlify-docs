OpenYida eignet sich für verschiedene Geschäftsszenarien. Hier sind Beispiele für gängige Anwendungstypen.

---

## 🏢 Unternehmensgeschäftssysteme

Erstellen Sie Multi-Modul-Geschäftsplattformen, die komplexe Formulare, Workflows und Datenbeziehungen unterstützen.

<AccordionGroup>
  <Accordion title="IPD (Integrated Product Development)">
    **Prompt:**
    ```
    Helfen Sie mir, ein IPD-System mit Yida zu erstellen, um den gesamten Chip-Herstellungsprozess zu verwalten,
    einschließlich Projektinitialisierung, Design-Review, Wafer-Fertigung, Massenproduktion usw.
    ```
    **Ergebnis:** Anwendung mit mehreren Formularen, einschließlich Phasenverfolgung, Prüfungslisten, Meilenstein-Management, Teamübergreifende Sichtbarkeit.
  </Accordion>

  <Accordion title="CRM (Customer Relationship Management)">
    **Prompt:**
    ```
    Helfen Sie mir, ein CRM zu erstellen, einschließlich Module für Leads, Kunden, Opportunities, Follow-up-Aufzeichnungen usw.
    ```
    **Ergebnis:** Verknüpfte Formulare: Leads → Kunden → Opportunities → Verträge, mit Statusverfolgung und Zuweisungsfeldern.
  </Accordion>

  <Accordion title="HR- und Urlaubsverwaltung">
    **Prompt:**
    ```
    Erstellen Sie ein HR-Urlaubsverwaltungssystem, einschließlich Urlaubsanträge, Genehmigungsprozess, Urlaubskontostatistiken
    ```
    **Ergebnis:** Urlaubsantragsformular mit Genehmigungsprozess, Kontenrechner, Manager-Dashboard.
  </Accordion>

  <Accordion title="Projektverwaltung">
    **Prompt:**
    ```
    Erstellen Sie eine Projektverwaltungsanwendung, einschließlich Aufgaben, Meilensteine, Teammitglieder, Fortschrittsverfolgung
    ```
    **Ergebnis:** Aufgabenliste mit Prioritäts-/Status-/Verantwortlichen-Feldern, Meilenstein-Tracker, Gantt-Diagramm-Ansicht.
  </Accordion>
</AccordionGroup>

Einmalige Rechen- oder Datensammelwerkzeuge.

<AccordionGroup>
  <Accordion title="Persönlicher Gehaltsrechner">
    **Prompt:**

## 🔌 API-Integration (Konnektoren)

Verbinden Sie Yida-Anwendungen mit externen Systemen über HTTP-Konnektoren.

<AccordionGroup>
  <Accordion title="Webhook-Benachrichtigungen">
    **Prompt:**
    ```
    Erstellen Sie einen HTTP-Konnektor für unseren DingTalk Webhook, fügen Sie eine POST-Aktion hinzu, um Formularübermittlungsbenachrichtigungen zu senden
    ```
    **Ergebnis:** Konfigurierter Konnektor mit Testfähigkeit, kann bei Formularereignissen ausgelöst werden.
  </Accordion>

  <Accordion title="Intelligente Erstellung aus curl">
    **Prompt:**
    ```
    Erstellen Sie intelligent einen Konnektor aus diesem curl-Befehl: curl -X POST https://api.example.com/v1/users -H "Authorization: Bearer TOKEN" -d '{"name":"test"}'
    ```
    **Ergebnis:** Automatisch konfigurierter Konnektor, extrahiert Basis-URL, Anforderungsheader, Aktionsparameter.
  </Accordion>
</AccordionGroup>

<Note>
Dies sind Ausgangspunkte. Da das Ergebnis eine vollständig bearbeitbare Anwendung auf der Yida-Plattform ist (und keine Blackbox-Konfiguration), können Sie die generierte Anwendung jederzeit über den visuellen Editor von Yida oder durch nachfolgende Anweisungen an die KI optimieren.
</Note>
