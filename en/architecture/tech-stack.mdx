## Core Module Design

### CLI Entry (bin/yida.js)

```javascript
#!/usr/bin/env node
import { program } from 'commander';
import { createApp } from '../lib/app.js';

program
  .name('openyida')
  .description('AI-powered Yida CLI')
  .version('1.0.0');

program
  .command('create-app <name>')
  .description('Create a new Yida app')
  .action(createApp);

program.parse();
```

## Data Flow

### Command Execution Flow

```
User Input
    ↓
CLI Parser (commander)
    ↓
Parameter Validation
    ↓
Auth Check
    ↓
Business Logic
    ↓
Yida API Call
    ↓
Result Processing
    ↓
Output Display
```

### Error Handling Flow

```
Error Occurs
    ↓
Catch at Boundary
    ↓
Error Classification
    ↓
User-Friendly Message
    ↓
Suggested Solution
    ↓
Exit with Code
```

## Security design

1. **Cookie encrypted storage** - Local cache is stored encrypted
2. **CSRF token verification** - Every request includes a token
3. **No hardcoded credentials** - All credentials are dynamically obtained
4. **Environment isolation** - Independent across different AI tool environments

---

## Performance optimization

1. **Lazy-load translation files** - Language packs loaded on demand
2. **Local schema caching** - Reduces API calls
3. **Concurrent request control** - Prevents rate limiting from excessive requests
4. **Incremental updates** - Form updates use merge instead of overwrite

## Design principles

### 1. Single Responsibility

Each module has a clear, single responsibility:

- CLI layer: Parse input, display output
- Core layer: Provide shared utilities
- Feature layer: Implement business logic

### 2. Dependency Injection

Services are injected rather than hard-coded:

```typescript
class AppService {
  constructor(
    private api: YidaAPI,
    private config: ConfigService
  ) {}
}
```

### 3. Error Boundaries

Errors are caught and handled at appropriate levels:

- CLI level: User-facing messages
- Service level: Business logic errors
- API level: Network/HTTP errors

### 4. Async Patterns

Consistent use of async/await:

```typescript
async function createApp(name: string): Promise<void> {
  const config = await loadConfig();
  await validateName(name);
  await api.createApp({ name, config });
}
```

## Extension Points

### Adding New Commands

1. Create command file in `lib/commands/`
2. Implement command logic
3. Register in CLI entry
4. Add tests

### Adding New AI Providers

1. Implement provider interface
2. Add configuration schema
3. Register in AI service
4. Update documentation

### Adding New Features

1. Design feature module
2. Implement core logic
3. Add CLI commands
4. Write tests
5. Update docs
