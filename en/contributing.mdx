## Ways to Contribute

1. **Report a Bug** → Open an Issue with reproduction steps and environment info
2. **Suggest a Feature** → Start a Discussion or Issue first, then implement
3. **Improve Docs** → PRs for documentation are always welcome
4. **Add Skills** → Extend the skill pack
5. **Fix Bugs / New Features** → Follow the development workflow below

---

## Development Setup

```bash
# 1. Fork and clone the repo
git clone git@github.com:your-username/openyida.git
cd openyida

# 2. Install dependencies
npm install

# 3. Link globally for local debugging
npm link

# 4. Run tests
npm test
```

---

## Development Workflow

### 1. Create a Branch

```bash
git checkout -b feat/your-feature-name
# or
git checkout -n fix/issue-description
```

### 2. Make Changes

- Write code with TypeScript
- Add tests for new features
- Update documentation

### 3. Test

```bash
# Run all tests
npm test

# Run linting
npm run lint

# Type check
npm run typecheck
```

### 4. Commit

We use [Conventional Commits](https://conventionalcommits.org/):

```bash
git commit -m "feat: add new command for X"
git commit -m "fix: resolve issue with Y"
git commit -m "docs: update README"
```

### 5. Push and PR

```bash
git push origin feat/your-feature-name
```

Then open a Pull Request on GitHub.

## Code Standards

### TypeScript

- Use strict mode
- Avoid `any` types
- Add JSDoc for public APIs

### Testing

- Unit tests for utilities
- Integration tests for commands
- Minimum 80% coverage

### Documentation

- Update README for new features
- Add JSDoc comments
- Update CLI help text

## Adding Skills

Skills extend OpenYida's capabilities:

```typescript
// skills/my-skill/index.ts
import { Skill } from '@openyida/core';

export default {
  name: 'my-skill',
  version: '1.0.0',
  
  activate(context) {
    context.registerCommand('my-command', async () => {
      // Implementation
    });
  }
} satisfies Skill;
```

## Review Process

1. **Automated Checks** → CI runs tests and linting
2. **Code Review** → Maintainers review the code
3. **Feedback** → Address any requested changes
4. **Merge** → PR is merged to main branch
