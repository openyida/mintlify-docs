OpenYida is designed for natural language interaction. The following prompts have been verified and work across all supported AI tools.

---

```

```

## Form & Page Operations

### Create Form Page

```
Help me create a xxx form page
```
```
Create a leave application form with: employee name, department, start date, end date, reason
```

### Add Form Fields

```
Add a [field name] field to [page name], type: [TextField/NumberField/DateField/SelectField/...]
```
```
Add a "Department" dropdown field to the employee info form, options: Engineering, Marketing, Sales, HR
```

<Note>
Only [supported field types](/en/features/skills#supported-field-types) are accepted. Unsupported types (e.g. `MobileField`, `EmailField`) will be rejected.
</Note>

### Modify Field Properties

```
Make [field name] field in [page name] required
```
```
Make [field name] field in [page name] read-only
```
```
Change [field name] field label in [page name] to [new label]
```

### Custom Display Pages

```
Create a custom display page for [page name]
```
```
Generate a landing page for an enterprise collaboration product
```

## Flash Note to PRD

```
Convert DingTalk flash notes to PRD documents
```

```
Generate a PRD from this meeting transcript: [paste notes]
```

```
Create a project requirements document from the attached flash note
```

```
Convert meeting notes into a structured product requirements document
```

<Note>
Requires login. The command reads meeting transcripts and uses AI to generate structured PRD documents. Output is saved to the `prd/` directory.
</Note>

## Integration & Automation

### Create Integration Flow

```
Create an integration flow that sends notifications when new records are submitted
```

```
Set up an automation on [form name] to notify [team/user] on new submissions
```

```
Create a [form name] workflow that triggers email notifications to [recipients]
```

### Add Automated Actions

```
Add a data lookup action to the [flow name] integration
```

```
Add a record creation action to automatically create entries in [target form]
```

<Note>
Integration flows trigger on form events (insert, update, delete) and can execute notifications, data lookups, and record creation. Use the CLI to create and configure flows.
</Note>

## DingTalk Integration

### Search & Query

```
Search for a contact named [name]
```

```
List all calendar events
```

```
Query my approval requests
```

### Create & Manage

```
Create a todo task: [task description]
```

```
Send a message: [message content]
```

### Query Attendance

```
List attendance records
```

```
Check my attendance status
```

<Info>
The `dws` command requires DingTalk CLI to be installed separately. Run `openyida dws install` if not already installed. Output defaults to table format; add `-f json` for JSON output.
</Info>

</Tip>
