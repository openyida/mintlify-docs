## Basic Usage

### Run Diagnostics

```bash
openyida doctor
```

### Auto Fix

```bash
openyida doctor --fix
```

### Generate Report

```bash
openyida doctor --report markdown
openyida doctor --report json
openyida doctor --report html
```

### Health Monitoring

```bash
openyida doctor --monitor
```

<Warning>
The health monitoring feature is currently in beta. API and output formats may change in future releases.
</Warning>

## Environment Checks

### Check All

```bash
openyida doctor env
```

### Check Specific Items

```bash
# Node.js version
openyida doctor env --check node

# Playwright browsers
openyida doctor env --check playwright

# Login status
openyida doctor env --check auth

# Network connectivity
openyida doctor env --check network
```

### Environment requirements

The doctor command runs the following checks in order:

| Check | Description | Auto-fixable |
|-------|-------------|--------------|
| CLI version | Compares installed version with SKILL.md metadata | Yes |
| project/ init | Verifies the project/ working directory exists | Yes |
| Node.js version | Requires ≥ 18 | No |
| Playwright | Checks if Playwright is installed | Yes |
| config.json | Validates configuration file | Yes |
| Login status | Checks Yida authentication | Yes |
| Network | Tests connectivity to aliwork.com | No |

## Application Diagnostics

### Full App Check

```bash
openyida doctor app --app <appId>
```

### Check Specific Areas

```bash
# Check PRD files
openyida doctor app --check prd

# Check page source
openyida doctor app --check pages

# Check schema cache
openyida doctor app --check schema

# Check React Hooks
openyida doctor app --check hooks
```

### Common Issues Detected

| Issue | Severity | Auto-fixable |
|-------|----------|--------------|
| CLI version mismatch | Warning | Yes |
| Missing project/ directory | High | Yes |
| Missing dependencies | High | Yes |
| Invalid page config | High | No |
| Schema cache stale | Medium | Yes |
| React Hooks errors | High | No |
| Missing environment vars | High | No |

## Smart Fix

### Auto-fix All

```bash
openyida doctor --fix
```

### Fix Specific Issues

```bash
# Fix dependencies
openyida doctor --fix --category deps

# Fix cache
openyida doctor --fix --category cache

# Fix config
openyida doctor --fix --category config
```

### Fix with Confirmation

```bash
openyida doctor --fix --interactive
```

## Health Monitoring

<Warning>
The `--duration` parameter and metric-specific monitoring (`--metric` flag) are experimental features. Their behavior and syntax may change in upcoming releases.
</Warning>

### Start Monitoring

```bash
openyida doctor --monitor --interval 60
```

### Monitor Specific Metrics

```bash
# Monitor auth status
openyida doctor --monitor --metric auth

# Monitor API latency
openyida doctor --monitor --metric api

# Monitor page build
openyida doctor --monitor --metric build
```

### Generate Health Report

```bash
openyida doctor --monitor --report --duration 24h
```

## CI/CD Integration

<Warning>
CI/CD integration with OpenYida doctor is in beta. The command syntax, output formats, and integration points may change. Test thoroughly before deploying to production pipelines.
</Warning>

### GitHub Actions

```yaml
- name: Run OpenYida Diagnostics
  run: |
    npx openyida doctor --report json
    
- name: Check for Critical Issues
  run: |
    if npx openyida doctor --check critical | grep -q "FAILED"; then
      exit 1
    fi
```

### Pre-commit Hook

```bash
#!/bin/bash
# .git/hooks/pre-commit

openyida doctor --check critical || exit 1
```

## Troubleshooting Guide

### Login Issues

```bash
# Check auth status
openyida doctor env --check auth

# Refresh token if needed
openyida auth refresh
```

### Build Failures

```bash
# Check environment
openyida doctor env

# Fix dependencies
openyida doctor --fix --category deps
```

### Page Errors

```bash
# Diagnose page issues
openyida doctor app --check pages --verbose
```
