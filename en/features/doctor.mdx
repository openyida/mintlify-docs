## Basic usage

### Run diagnostics

```bash
openyida doctor
```

### Auto fix

```bash
openyida doctor --fix
```

You can also use `--repair` as an alias for `--fix`:

```bash
openyida doctor --repair
```

### Generate report

```bash
# Markdown format
openyida doctor --report markdown

# JSON format
openyida doctor --report json

# HTML format
openyida doctor --report html
```

## Environment checks

The doctor command runs the following checks in order:

| Check | Description | Auto-fixable |
|-------|-------------|--------------|
| CLI version | Compares installed version with SKILL.md metadata | Yes |
| project/ init | Verifies the project/ working directory exists | Yes |
| Node.js version | Requires >= 16 | No |
| Playwright | Checks if Playwright is installed | Yes |
| config.json | Validates configuration file | Yes |
| Login status | Checks Yida authentication | Yes |
| Network | Tests connectivity to aliwork.com | No |

---

## Application diagnostics

| Check | Description |
|-------|-------------|
| PRD files | Checks if requirements documents exist |
| Page source | Checks source files, console.log usage, empty files |
| Schema cache | Checks cache file format |
| React Hooks | Checks Hooks usage compliance |

## Production diagnostics

Diagnose issues with a published application:

```bash
openyida doctor --production --app <appId>
```

This checks:
- Application ID validity
- Production error logs
- Performance metrics

---

## Health monitoring

Start continuous monitoring (checks every 60 seconds):

```bash
openyida doctor --monitor
```

<Warning>
The health monitoring feature is currently in beta. API and output formats may change in future releases.
</Warning>

Example output:

```
[14:32:01] Health: 95/100
  Passed: 18/19 | Errors: 0 | Warnings: 1

[14:33:01] Health: 100/100
  Passed: 19/19 | Errors: 0 | Warnings: 0
```

Press `Ctrl+C` to stop monitoring.

## Tickets and VOC

### Create a ticket (bug report)

```bash
openyida doctor --create-ticket
```

Interactively prompts for a title and description, then submits to GitHub Issues.

### Create a VOC (feature request)

```bash
openyida doctor --create-voc
```

### Smart submit

Automatically determines whether to file a bug ticket or a feature request:

```bash
openyida doctor --auto-submit
```

### Auto-fix all

## Pre-checks

The doctor automatically runs pre-checks before critical operations:

- Before `openyida publish` — validates environment and dependencies
- Before `openyida create-app` — validates environment and dependencies

---

## Fix types

| Type | Description | Example |
|------|-------------|---------|
| `AUTO` | Automatic fix | Create config.json, delete corrupted cache |
| `COMMAND` | Command fix | Run `pip install playwright` |
| `MANUAL` | Manual fix | Install gh CLI, write PRD |

---

## Health score

Health score (0–100) formula:

```
Health = pass_rate × 100 - errors × 10 - warnings × 3
```

| Score | Status |
|-------|--------|
| 90–100 | Excellent |
| 70–89 | Good |
| 50–69 | Fair |
| < 50 | Needs improvement |

## Complete flag reference

| Flag | Description |
|------|-------------|
| `--fix` | Diagnose and auto-fix repairable issues |
| `--repair` | Alias for `--fix` |
| `--production --app <appId>` | Run production diagnostics for a specific app |
| `--monitor` | Start continuous health monitoring |
| `--report <format>` | Generate report (`json`, `markdown`, or `html`) |
| `--create-ticket` | Create a bug ticket from diagnosis results |
| `--create-voc` | Create a VOC (feature request) |
| `--auto-submit` | Automatically determine and submit a ticket or VOC |
