## Creating reports

### Basic usage

```bash
openyida create-report <appType> "Report Name" '<chart-definition-JSON-or-file>'
```

The third argument can be either an inline JSON string or a path to a JSON file.

### Example: create a sales report

```bash
openyida create-report APP_XXXXXX "Sales Dashboard" '{
  "charts": [
    {
      "type": "bar",
      "title": "Monthly Sales",
      "cubeCode": "FORM_XXXXXX",
      "xField": "month",
      "yField": "amount"
    }
  ]
}'
```

### Load chart definition from a file

```bash
openyida create-report APP_XXXXXX "Annual Report" ./charts-config.json
```

## Appending charts to a report

```bash
openyida append-chart <appType> <reportId> '<chart-definition-JSON-or-file>'
```

### Example

```bash
openyida append-chart APP_XXXXXX REPORT_XXXXX '{
  "type": "line",
  "title": "Growth Trend",
  "cubeCode": "FORM_XXXXXX",
  "xField": "month",
  "yField": "growth_rate"
}'
```

## Filter configuration

Reports support filters that link to charts for interactive data filtering. You can configure filters using the `filters` array in the chart definition JSON.

### Full configuration with filters

```json
{
  "filters": [
    {
      "title": "Project filter",
      "placeholder": "Select a project",
      "cubeCode": "FORM_XXXXXX",
      "valueField": {
        "fieldCode": "selectField_xxx",
        "aliasName": "Project value",
        "dataType": "STRING"
      },
      "labelField": {
        "fieldCode": "selectField_xxx",
        "aliasName": "Project name",
        "dataType": "STRING"
      },
      "linkTo": [0, 1]
    }
  ],
  "charts": [
    {
      "type": "bar",
      "title": "Bar chart",
      "cubeCode": "FORM_XXXXXX",
      "xField": "name",
      "yField": "value"
    }
  ]
}
```

### Filter field reference

| Field | Required | Description |
|-------|----------|-------------|
| `title` | Yes | Filter title |
| `placeholder` | No | Placeholder text |
| `cubeCode` | Yes | Data source form ID (e.g., `FORM_XXXXXX`) |
| `valueField` | Yes | Value field definition (contains `fieldCode`, `aliasName`, `dataType`) |
| `labelField` | Yes | Display field definition (contains `fieldCode`, `aliasName`, `dataType`) |
| `linkTo` | No | Target charts to link. Array elements can be chart indices (numbers) or chart titles (strings). If omitted, the filter links to all charts |

<Info>
  If you do not configure `filters`, the system automatically generates dropdown filters for `selectField`-type fields in your charts.
</Info>

## Chart types

| Type | Description | Best for |
|------|-------------|----------|
| **bar** | Bar chart | Comparing categories |
| **line** | Line chart | Trends over time |
| **pie** | Pie chart | Part-to-whole relationships |
| **table** | Data table | Detailed data view |
| **area** | Area chart | Cumulative trends |
| **scatter** | Scatter chart | Correlation analysis |
| **indicator** | Indicator card | Key metrics display |
| **combo** | Combo chart (bar + line) | Dual-axis comparisons |
| **pivot** | Cross pivot table | Multi-dimensional analysis |
| **gauge** | Gauge chart | Progress or target tracking |
| **radar** | Radar chart | Multi-dimensional comparisons |
| **funnel** | Funnel chart | Stage-based conversion |

## Chart layout sizing

You can control the width and height of each chart in the report grid using the `w` and `h` properties. The report uses a 6-column grid system.

- **`w`** — grid width (1-6 columns). When a chart exceeds the remaining space on the current row, it wraps to the next row.
- **`h`** — grid height (in grid units).

Setting `w` or `h` to `0` is valid and will be respected. If omitted, the default values below are used.

| Chart type | Default `w` | Default `h` |
|------------|-------------|-------------|
| **indicator** | 6 | 6 |
| **pie** | 3 | 22 |
| **bar** | 3 | 22 |
| **line** | 3 | 22 |
| **combo** | 6 | 22 |
| **table** | 6 | 38 |
| **pivot** | 6 | 30 |
| **gauge** | 2 | 18 |
| Other types | 3 | 22 |

## Chart definition format

### Bar chart

```json
{
  "type": "bar",
  "title": "Sales by Region",
  "cubeCode": "FORM_XXXXXX",
  "xField": "region",
  "yField": "amount",
  "aggregation": "sum",
  "w": 6,
  "h": 22
}
```

### Line chart

```json
{
  "type": "line",
  "title": "Traffic Trend",
  "cubeCode": "FORM_XXXXXX",
  "xField": "date",
  "yField": "visitors",
  "aggregation": "sum"
}
```

### Pie chart

```json
{
  "type": "pie",
  "title": "Market Share",
  "cubeCode": "FORM_XXXXXX",
  "categoryField": "product",
  "valueField": "share"
}
```

### Data table

```json
{
  "type": "table",
  "title": "Detail View",
  "cubeCode": "FORM_XXXXXX",
  "columns": [
    { "field": "name", "title": "Name" },
    { "field": "amount", "title": "Amount" },
    { "field": "date", "title": "Date" }
  ]
}
```

## cubeCode format

The `cubeCode` in chart definitions is the report engine's data source identifier. It is derived from the form's `formUuid` but uses **underscores** instead of hyphens.

```
formUuid: FORM-AB4ACB9D-D12C-470D-8204-7E05CDC19166
cubeCode: FORM_AB4ACB9D_D12C_470D_8204_7E05CDC19166
```

<Tip>
  You can use `openyida get-schema` to retrieve a form's `formUuid`, then replace hyphens with underscores to get the `cubeCode`. The system also accepts hyphenated format and converts it automatically.
</Tip>

## Complete configuration example

### sales-report-charts.json

```json
{
  "charts": [
    {
      "type": "bar",
      "title": "Monthly Sales",
      "cubeCode": "FORM_MONTHLY_SALES",
      "xField": "month",
      "yField": "sales_amount",
      "aggregation": "sum"
    },
    {
      "type": "pie",
      "title": "Product Categories",
      "cubeCode": "FORM_SALES",
      "categoryField": "product_category",
      "valueField": "sales_amount"
    },
    {
      "type": "line",
      "title": "Sales Trend",
      "cubeCode": "FORM_DAILY_SALES",
      "xField": "date",
      "yField": "sales_amount",
      "aggregation": "sum"
    },
    {
      "type": "table",
      "title": "Sales Detail",
      "cubeCode": "FORM_SALES",
      "columns": [
        { "field": "order_no", "title": "Order No.", "width": 150 },
        { "field": "customer_name", "title": "Customer", "width": 200 },
        { "field": "product_name", "title": "Product", "width": 200 },
        { "field": "amount", "title": "Amount", "width": 120 },
        { "field": "sales_date", "title": "Date", "width": 150 }
      ],
      "pageSize": 10
    }
  ]
}
```

Create the report:

```bash
openyida create-report APP_XXXXXX "Sales Analysis Report" ./sales-report-charts.json
```

## Best practices

1. **Choose appropriate chart types** - Use line charts for trends, bar charts for comparisons, pie charts for proportions
2. **Limit chart count** - Keep reports to 6 or fewer charts
3. **Include data tables** - Pair charts with tables for detailed data inspection
4. **Set reasonable page sizes** - For data tables, use 10-20 rows per page
