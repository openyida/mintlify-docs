OpenYida is suitable for various business scenarios. Here are examples of common application types.

---

## 🏢 Enterprise Business Systems

Build multi-module business platforms with complex forms, workflows, and data relationships.

<AccordionGroup>
  <Accordion title="IPD (Integrated Product Development)">
    **Prompt:**
    ```
    Help me create an IPD system on Yida to manage the full chip production workflow,
    including project initiation, design review, tape-out, and mass production phases
    ```
    **Output:** Multi-form application with phase tracking, review checklists, milestone management, and cross-team visibility.
  </Accordion>

  <Accordion title="CRM (Customer Relationship Management)">
    **Prompt:**
    ```
    Help me build a CRM with leads, contacts, opportunities, and follow-up records modules
    ```
    **Output:** Related forms: Leads → Contacts → Opportunities → Contracts, with status tracking and assignment fields.
  </Accordion>

  <Accordion title="HR & Leave Management">
    **Prompt:**
    ```
    Create an HR leave management system with leave applications, approval workflows, and leave balance statistics
    ```
    **Output:** Leave application form with approval workflow, balance calculator, and manager dashboard.
  </Accordion>

  <Accordion title="Project Management">
    **Prompt:**
    ```
    Build a project management app with tasks, milestones, team members, and progress tracking
    ```
    **Output:** Task list with priority/status/assignee fields, milestone tracker, and Gantt chart view.
  </Accordion>
</AccordionGroup>

## 🔌 API Integration (Connectors)

Connect Yida applications with external systems via HTTP connectors.

<AccordionGroup>
  <Accordion title="Webhook Notifications">
    **Prompt:**
    ```
    Create an HTTP connector for our DingTalk webhook, add POST action to send form submission notifications
    ```
    **Output:** Configured connector with test capability, can trigger on form events.
  </Accordion>

  <Accordion title="Smart Create from Curl">
    **Prompt:**
    ```
    Smart create connector from this curl command: curl -X POST https://api.example.com/v1/users -H "Authorization: Bearer TOKEN" -d '{"name":"test"}'
    ```
    **Output:** Auto-configured connector extracting base URL, headers, and action parameters.
  </Accordion>
</AccordionGroup>

---

<Note>
These are starting points. Since the output is a fully editable application on the Yida platform (not a black-box configuration), you can always refine the generated app through the Yida visual editor or by giving AI follow-up instructions.
</Note>
