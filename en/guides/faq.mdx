  <Accordion title="Do I need a YiDa account to use OpenYida?">
    Yes. OpenYida creates and manages apps on the [YiDa](https://www.aliwork.com) low-code platform, requiring an Alibaba/DingTalk account with YiDa enabled. Run `openyida login` to authenticate.
  </Accordion>

  <Accordion title="Which AI tools are supported?">
    OpenYida is compatible with any AI coding tool that supports MCP (Model Context Protocol) or can read markdown skill documents. Confirmed support: **Claude Code, Aone Copilot, OpenCode, Cursor, Visual Studio Code, Qoder, Wukong**. In practice, any tool that can execute shell commands works.
  </Accordion>

  <Accordion title="What Node.js version is required?">
    Node.js **≥ 18** is required. Check your version with `node --version`. Download the latest LTS from [nodejs.org](https://nodejs.org).
  </Accordion>

  <Accordion title="What if the login QR code doesn't display?">
    Try running `openyida login` in a terminal that supports browser launching. If in a headless environment (e.g., remote server), use `openyida login --wukong` in DingTalk/Wukong environments, or copy the login URL from terminal output and open it in your local browser.
  </Accordion>

  <Accordion title="How to refresh an expired session?">
    Run `openyida auth refresh` to renew the session without going through the full login flow. If it fails, run `openyida login` to re-authenticate.
  </Accordion>

  <Accordion title="How to switch between multiple YiDa organizations?">
    ```bash
    openyida org list       # List all organizations and corpIds
    openyida org switch --corp-id <corpId>   # Switch to target organization
    openyida env            # Confirm the switch
    ```
  </Accordion>

  <Accordion title="How to add missing fields to AI-generated forms?">
    Tell the AI directly:
    ```
    Add a [field name] field to [page name], field type: [TextField/NumberField/DateField/SelectField/...], make it required
    ```
    OpenYida will call `openyida create-form` with the updated Schema, merging new fields without overwriting existing ones. Only [supported field types](/en/features/skills#supported-field-types) are accepted — unsupported types will be rejected with an error.
  </Accordion>

  <Accordion title="Can I edit generated apps in YiDa visual editor?">
    Absolutely. The output is a standard YiDa app — you can open it in the YiDa platform visual editor anytime and make manual modifications. OpenYida generates apps, not locked configurations.
  </Accordion>

  <Accordion title="How to make an app publicly accessible?">
    ```
    Make [page name] page publicly accessible
    ```
    Or run directly:
    ```bash
    openyida save-share-config
    ```
    Configure page sharing settings so anyone can access it via link without login.
  </Accordion>

  <Accordion title="Can I migrate existing YiDa apps with OpenYida?">
    Yes. Use `openyida export` to generate a migration package from an existing app, and `openyida import` to rebuild it in another organization or environment.
  </Accordion>

  <Accordion title="What's the difference between `create-form` and `create-page`?">
    - **`create-form`**: Creates a YiDa **form page** — a data entry form using YiDa's built-in data storage underneath. Used for structured data collection (registrations, applications, records).
    - **`create-page`**: Creates a **custom display page** — a JSX-based page for dashboards, landing pages, interactive UI. Compiled and published via `openyida publish`.
  </Accordion>

  <Accordion title="Does OpenYida support process/approval workflows?">
    Yes. Use `openyida create-process` to create process forms with approval steps, and `openyida configure-process` to define workflow rules (approvers, conditions, notifications).
  </Accordion>

  <Accordion title="Something went wrong, how to debug?">
    1. Run `openyida doctor --fix` — automatically detects and fixes common issues
    2. Run `openyida env` — verify login status and environment
    3. Run `openyida auth status` — check session validity
    4. Check [GitHub Issues](https://github.com/openyida/openyida/issues) for known problems
    5. If it's a new bug, submit an issue with: Node.js version, command run, full error output
  </Accordion>

  <Accordion title="How to contribute to OpenYida?">
    Read the [Contributing Guide](/en/contributing) for the complete process. In short: fork the repo, make changes on a feature branch, run `npm test`, submit a PR. AI-assisted contributions are welcome!
  </Accordion>
