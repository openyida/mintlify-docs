## What is OpenYida?

OpenYida is a command-line interface that bridges AI agents with the [Yida](https://www.aliwork.com) low-code platform. It allows you to create, manage, and deploy enterprise applications using natural language prompts — no manual drag-and-drop required.

Think of it as a way to describe what you want ("a CRM with leads, contacts, and deals") and have AI build it for you on the Yida platform.

## How It Works

```
┌─────────────────┐     ┌──────────────┐     ┌─────────────────┐
│   You / AI      │────▶│   OpenYida   │────▶│     Yida        │
│   (Natural      │     │    CLI       │     │   Platform      │
│    Language)    │     │              │     │                 │
└─────────────────┘     └──────────────┘     └─────────────────┘
                              │
                              ▼
                       ┌──────────────┐
                       │  Local Files │
                       │  (Config,    │
                       │   Code)      │
                       └──────────────┘
```

1. **Describe** what you want in natural language
2. **OpenYida** translates to Yida API calls
3. **Yida** creates the application
4. **Result** is a real, editable Yida app

## Quick Example

```bash
# Login to Yida
openyida login

# Create an application
openyida create-app --name "My CRM"

# AI generates forms from your description
openyida create-form --page "Leads" --schema '{...}'
```

Or simply tell your AI assistant:

> "Create a CRM with leads, contacts, and deals forms. Each lead should have name, company, phone, and status fields."

The AI will use OpenYida commands to build it.

## Architecture

OpenYida is built with a modular architecture:

- **Core Layer** — Authentication, environment, i18n, logging
- **Yida API Layer** — HTTP client with automatic retries and CSRF handling
- **Feature Modules** — App management, form/page creation, data operations
- **CLI Interface** — Command parsing and execution

Read more in [Technical Stack](/openyida/en/architecture/tech-stack).
