## Prerequisites

Before you begin, ensure you have the following dependencies installed:

| Dependency | Version | Check Command |
|------|------|----------|
| [Node.js](https://nodejs.org) | ≥ 18 | `node --version` |

<Tip>
  That's all you need. OpenYida is a pure Node.js implementation — no Python, no Playwright, no build steps.
</Tip>

---

## Step 1 — Installation

```bash
npm install -g openyida
```

Verify the installation:

```bash
openyida --version
```

---

## Step 2 — Login to Yida

```bash
openyida login
```

OpenYida will open a browser window (or display a QR code) for you to authenticate with your Yida/Alibaba account. Credentials are cached locally — you only need to do this once.

<Info>
  If you are using **Wukong** as your AI tool, use the special login flag:

  ```bash
  openyida login --wukong
  ```
</Info>

---

## Step 3 — Check Environment

```bash
openyida env
```

This detects the currently active AI coding tool (Claude Code, Cursor, Wukong, etc.) and displays your current login status. Run this command anytime you switch environments.

---

## Step 4 — Initialize Project Workspace

```bash
openyida copy
```

This copies the `project/` workspace template to your current AI tool's working directory, allowing the AI to access your app configuration and page templates.

---

## Step 5 — Start Building

Now simply talk to your AI coding assistant. Here are some starter prompts:

```
Build me an IPD system on Yida to manage the complete chip production workflow
```

```
Help me build a CRM system with leads, contacts, and deals
```

```
Create a personal salary calculator app
```

```
Generate a birthday greeting mini-game on Yida
```

The AI will use OpenYida's skill pack to call the correct CLI commands in sequence: `create-app` → `create-form` / `create-page` → `publish` → `save-share-config`.

---

## Step 6 — Publish the App

After the AI generates the app, use the following command to publish:

```bash
openyida publish
```

Or ask the AI:
```
Publish the [page name] page
Make the page publicly accessible
```

---

## Run Diagnostics

If anything feels off, run the built-in diagnostic tool:

```bash
openyida doctor
```

It checks your Node.js version, login status, and environment configuration, and can automatically fix common issues with `--fix`.
