# Visión General de la Arquitectura

OpenYida es una herramienta CLI basada en Node.js que adopta un diseño arquitectónico en capas, integrando orgánicamente capacidades de IA, integración con la plataforma Yida y experiencia del desarrollador.

## Flujo de Trabajo General

![Flujo de trabajo de OpenYida](/images/architecture-workflow.png)

El flujo de trabajo de OpenYida se divide en 6 etapas:

1. **User Requirement** - El usuario describe los requisitos en lenguaje natural
2. **AI Understanding** - La IA entiende y analiza los requisitos del usuario
3. **PRD Document** - Genera el documento de requisitos del producto
4. **CLI Execution** - Ejecuta comandos CLI para crear la aplicación
5. **Yida Platform** - Construye la aplicación en la plataforma Yida
6. **Online Access** - La aplicación se publica y es accesible

## Flujo de Autenticación

![Flujo de autenticación](/images/architecture-auth.png)

Flujo de autenticación de OpenYida:

1. **config.json** - Almacena información de configuración
2. **Login Session** - Establece sesión de inicio de sesión
3. **Cookie Cache** - Almacena en caché información de autenticación
4. **API Request** - Envía solicitudes API
5. **Yida Backend** - Backend de Yida procesa
6. **Page/Form Output** - Salida de página/formulario

## Arquitectura CLI

![Arquitectura CLI](/images/architecture-cli.png)

Interacción de OpenYida CLI con la plataforma Yida:

- **AI Coding Tool** → **OpenYida CLI**
  - Login（Inicio de sesión）
  - Create App（Crear aplicación）
  - Create Page（Crear página）
  - Publish（Publicar）
  - Get Schema（Obtener estructura）
- **Yida Low-Code Platform**
  - App Management（Gestión de aplicaciones）
  - Form Pages（Páginas de formulario）
  - Custom Pages（Páginas personalizadas）

## Sistema de Skills

![Sistema de skills](/images/architecture-skills.png)

Los skills de OpenYida se dividen en tres categorías:

### Login Skills（Skills de inicio de sesión）
- `login` - Iniciar sesión en la plataforma Yida
- `logout` - Cerrar sesión

### Create Skills（Skills de creación）
- `create-app` - Crear aplicación
- `create-page` - Crear página
- `create-form` - Crear formulario
- `get-schema` - Obtener estructura de datos

### Publish Skills（Skills de publicación）
- `publish` - Publicar aplicación

## Principios de Diseño Central

### 1. Diseño Modular

Cada módulo funcional está encapsulado independientemente, interactuando a través de interfaces claras:

- **Capa de comandos**: Procesa entrada CLI, llama a servicios centrales
- **Capa de servicios**: Encapsula lógica de negocio, proporciona API unificada externamente
- **Capa de adaptación**: Conecta sistemas externos（IA, plataforma Yida）

### 2. Extensión por Plugins

El sistema Skill soporta carga dinámica y extensión:

```typescript
// Definición de interfaz Skill
interface Skill {
  name: string;
  version: string;
  activate(context: SkillContext): void;
  deactivate(): void;
}
```

### 3. Conducido por Configuración

Soporta configuración multi-entorno, cambio flexible:

- Configuración global local（~/.openyida/）
- Configuración a nivel de proyecto（.openyidarc.json）
- Sobrescritura por variables de entorno

### 4. Seguridad de Tipos

Soporte de tipos TypeScript en toda la cadena, reduce errores en tiempo de ejecución.

| Cliente HTTP | Axios |
| SDK AI | OpenAI SDK, Anthropic SDK |
| Gestión de Configuración | Cosmiconfig |
| Logging | Winston |
| Testing | Jest |

## Diseño de Seguridad

### Seguridad de Autenticación

- Flujo estándar OAuth 2.0
- Mecanismo de actualización automática de Token
- Almacenamiento cifrado local de información sensible

### Transmisión de Datos

- HTTPS en toda la cadena
- Verificación de firma API
- Control de reintentos y tiempo de espera de solicitudes

## Optimización de Rendimiento

- **Pool de Conexiones**: Reutiliza conexiones HTTP
- **Caché Inteligente**: Caché local de configuración y metadatos
- **Actualización Incremental**: Solo transmite contenido cambiado
- **Control de Concurrencia**: Limita llamadas API simultáneas

## Puntos de Extensión

Los desarrolladores pueden extender en los siguientes niveles:

1. **Nuevos Comandos**: Agregar comandos CLI personalizados
2. **Nuevos Skills**: Extender capacidades de IA
3. **Nuevas Plantillas**: Plantillas de generación de código personalizadas
4. **Nuevos AI Provider**: Integrar otros modelos grandes
