# Estructura del proyecto

OpenYida adopta un diseño de arquitectura modular con una organización de código clara, facilitando el mantenimiento y la extensión.

## Resumen de directorios

```
openyida/
├── src/
│   ├── commands/          # Implementación de comandos CLI
│   │   ├── create-app.ts
│   │   ├── create-form.ts
│   │   ├── create-page.ts
│   │   ├── publish.ts
│   │   ├── share.ts
│   │   ├── data.ts
│   │   ├── connector.ts
│   │   ├── report.ts
│   │   ├── doctor.ts
│   │   └── login.ts
│   ├── core/              # Módulos de funcionalidad principal
│   │   ├── yida-api.ts    # Encapsulación API Yida
│   │   ├── ai-service.ts  # Integración de servicios IA
│   │   ├── config.ts      # Gestión de configuración
│   │   └── auth.ts        # Manejo de autenticación
│   ├── utils/             # Funciones de utilidad
│   │   ├── logger.ts
│   │   ├── file.ts
│   │   └── validator.ts
│   └── types/             # Definiciones de tipos TypeScript
│       ├── yida.ts
│       ├── ai.ts
│       └── index.ts
├── skills/                # Sistema de habilidades
│   ├── builtin/           # Habilidades integradas
│   └── custom/            # Habilidades personalizadas
├── templates/             # Plantillas de código
│   ├── app/
│   ├── form/
│   └── page/
├── docs/                  # Documentación
├── tests/                 # Archivos de prueba
├── package.json
├── tsconfig.json
└── README.md
```

## Descripción de módulos principales

### src/commands/

Contiene la implementación de todos los comandos CLI, cada archivo corresponde a un subcomando:

| Archivo | Función |
|------|------|
| `create-app.ts` | Crear aplicación Yida |
| `create-form.ts` | Crear página de formulario |
| `create-page.ts` | Crear página personalizada |
| `publish.ts` | Publicar aplicación |
| `share.ts` | Compartir aplicación |
| `data.ts` | Operaciones de gestión de datos |
| `connector.ts` | Gestión de conectores |
| `report.ts` | Operaciones de reportes |
| `doctor.ts` | Herramienta de diagnóstico |
| `login.ts` | Autenticación de login |

### src/core/

Módulos de lógica de negocio principal:

- **yida-api.ts**: Encapsula la API de la plataforma abierta Yida, proporciona interfaz de llamada unificada
- **ai-service.ts**: Integra múltiples servicios de IA (OpenAI, Claude, Tongyi Qwen, etc.)
- **config.ts**: Gestiona archivos de configuración local (~/.openyida/config.json)
- **auth.ts**: Maneja estado de login, actualización de tokens, cambio multi-organización

### src/utils/

Funciones de utilidad generales:

- **logger.ts**: Salida de logs, soporte de niveles y colores
- **file.ts**: Funciones auxiliares de operaciones de archivos
- **validator.ts**: Validación y verificación de entradas

### src/types/

Definiciones de tipos TypeScript, asegurando la seguridad de tipos del código:

- **yida.ts**: Tipos relacionados con API Yida
- **ai.ts**: Tipos relacionados con servicios de IA
- **index.ts**: Exportaciones de tipos generales

## Sistema de habilidades

```
skills/
├── builtin/               # Habilidades integradas
│   ├── code-generator/    # Generación de código
│   ├── form-builder/      # Construcción de formularios
│   ├── page-designer/     # Diseño de páginas
│   └── data-analyzer/     # Análisis de datos
└── custom/                # Habilidades personalizadas del usuario
    └── (extensiones de usuario)
```

Las habilidades adoptan una arquitectura de plugins, soportando carga en caliente y extensión dinámica.

## Gestión de configuración

### Configuración local

Ubicación de almacenamiento: `~/.openyida/config.json`

```json
{
  "defaultOrg": "org-default",
  "orgs": {
    "org-default": {
      "accessToken": "xxx",
      "refreshToken": "xxx",
      "expiresAt": 1234567890
    }
  },
  "aiProvider": "openai",
  "aiConfig": {
    "apiKey": "sk-xxx",
    "model": "gpt-4"
  }
}
```

### Configuración de proyecto

Se puede crear `.openyidarc.json` en el directorio raíz del proyecto:

```json
{
  "appId": "APP-XXX",
  "defaultEnv": "dev",
  "environments": {
    "dev": { "appKey": "xxx", "appSecret": "xxx" },
    "prod": { "appKey": "xxx", "appSecret": "xxx" }
  }
}
```

## Mecanismo de extensión

### Agregar nuevo comando

1. Crear nuevo archivo en `src/commands/`
2. Implementar función de manejo de comando
3. Registrar comando en `src/index.ts`

### Agregar nueva habilidad

1. Crear directorio de habilidad en `skills/custom/`
2. Implementar `skill.json` y lógica principal
3. Cargar con `openyida skill load`

## Normas de desarrollo

- Todas las nuevas funcionalidades deben incluir definiciones de tipos
- Las implementaciones de comandos deben incluir documentación `--help`
- Las funcionalidades principales deben tener cobertura de pruebas unitarias
- Seguir las configuraciones de ESLint y Prettier
