## Formas de Contribuir

1. **Reportar un Bug** → Abrir un Issue con pasos de reproducción e información del entorno
2. **Sugerir una Función** → Iniciar una Discussion o Issue primero, luego implementar
3. **Mejorar Docs** → Los PRs para documentación siempre son bienvenidos
4. **Añadir Skills** → Extender el paquete de skills bajo `yida-skills/`
5. **Corregir Bugs / Nuevas Funciones** → Seguir el flujo de desarrollo a continuación

## Configuración de Desarrollo

```bash
# 1. Hacer fork y clonar el repo
git clone git@github.com:your-username/openyida.git
cd openyida

# 2. Instalar dependencias
npm install

# 3. Vincular globalmente para depuración local
npm link

# 4. Ejecutar pruebas
npm test
```

## Lista de Verificación PR

Antes de enviar un pull request, verifica:

- [ ] Probada la función relevante con una cuenta Yida real localmente
- [ ] Todas las pruebas pasan: `npm test`
- [ ] La verificación de sintaxis JS pasa:
  ```bash
  node --check bin/yida.js && for f in lib/*.js lib/locales/*.js; do node --check "$f"; done
  ```
- [ ] La descripción PR explica claramente qué cambió y por qué
- [ ] Capturas de pantalla o grabaciones adjuntas si hay cambios de UI/comportamiento

---

## Directrices PR

- **Un PR, una cosa** — no mezclar cambios no relacionados
- **Formato título PR:** `feat: add xxx` / `fix: fix xxx` / `docs: update xxx`
- **Descripción** debe cubrir: qué, por qué y cómo probar
- Si el PR cierra un Issue, añadir `Closes #123` en la descripción

---

## Convención de Commits

Seguir [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add export-app command
fix: fix get-page-config path error
docs: update CLI command reference
refactor: refactor login module
test: add utils unit tests
chore: upgrade dependencies
```

---

## Estilo de Código

- Seguir el estilo de código existente (módulos CommonJS, preferir APIs nativas de Node.js)
- Usar nombres en inglés significativos para variables y funciones; evitar abreviaturas
- Manejar errores completamente — no tragar excepciones silenciosamente
- Al añadir un nuevo comando, actualizar la tabla de comandos CLI en `README.md`
