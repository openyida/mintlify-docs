OpenYida proporciona comandos CLI completos que soportan la gestión del ciclo de vida completo de aplicaciones.

---

## Entorno y autenticación

```bash
# Detección de entorno
openyida env

# Iniciar sesión
openyida login [--qr]              # --qr usa código QR en terminal
openyida login --wukong            # Exclusivo para entorno Wukong

# Cerrar sesión
openyida logout

# Gestión de sesión
openyida auth status               # Ver estado de inicio de sesión
openyida auth login                # Ejecutar inicio de sesión
openyida auth refresh              # Actualizar sesión
openyida auth logout               # Cerrar sesión

# Gestión de organizaciones
openyida org list                  # Listar organizaciones accesibles
openyida org switch --corp-id <id> # Cambiar organización

# Inicializar directorio de trabajo
openyida copy [--force]            # Copiar directorio project

# Herramienta de diagnóstico
openyida doctor [--fix]            # Diagnóstico de entorno
openyida doctor --monitor          # Monitoreo de salud
openyida doctor --report <format>  # Generar informe (json|markdown|html)
```

## Gestión de aplicaciones

```bash
# Crear aplicación
openyida create-app "<nombre>" [desc] [icon] [color] [themeColor]

# Exportar aplicación
openyida export <appType> [output] # Generar paquete de migración

# Importar aplicación
openyida import <file> [name]      # Reconstruir aplicación desde paquete
```

## Gestión de formularios

```bash
# Crear formulario
openyida create-form create <appType> "<nombre>" <camposJSON> \
  [--layout <diseño>] \
  [--theme <tema>] \
  [--label-align <alineación>]

# Actualizar formulario
openyida create-form update <appType> <formUuid> <modificacionesJSON>

# Obtener Schema del formulario
openyida get-schema <appType> <formUuid>

# Actualizar configuración del formulario
openyida update-form-config <appType> <formUuid> <isRenderNav> <título>
```

### Tipos de campo soportados

El comando `create-form` solo acepta los siguientes tipos de campo. Usar un tipo no soportado devuelve un error.

#### Campos base

Campos integrados que no requieren plugins adicionales.

| Tipo | Descripción |
|------|-------------|
| `TextField` | Texto de una línea |
| `TextareaField` | Texto multilínea |
| `NumberField` | Número |
| `RateField` | Calificación |
| `DateField` | Fecha |
| `RadioField` | Botones de radio |
| `SelectField` | Desplegable (selección única) |
| `CheckboxField` | Casillas de verificación |
| `MultiSelectField` | Desplegable (selección múltiple) |
| `CascadeSelectField` | Selección en cascada |
| `CascadeDateField` | Fecha en cascada |
| `EmployeeField` | Selector de miembro |
| `DepartmentSelectField` | Selector de departamento |
| `AssociationFormField` | Formulario asociado |
| `AssociationQuery` | Consulta de asociación |
| `AttachmentField` | Archivo adjunto |
| `ImageField` | Imagen |
| `AddressField` | Dirección |
| `CountrySelectField` | Selector de país/región |
| `TableField` | Subformulario / detalle |
| `SerialNumberField` | Número de serie |

#### Campos avanzados

Campos integrados con funcionalidad extendida. Algunas funciones pueden requerir activación.

| Tipo | Descripción |
|------|-------------|
| `EditorField` | Editor de texto enriquecido |
| `DigitalSignatureField` | Firma digital |
| `LocationField` | Ubicación GPS |

#### Campos de plugin

Estos campos requieren instalar el plugin correspondiente desde el centro de plugins antes de su uso.

| Tipo | Plugin requerido |
|------|------------------|
| `TimeZoneDateField` | Componente de zona horaria internacional |
| `CC_AIButtonField` | Componente de IA |
| `CC_PG_ESignField` | Componente de finanzas inteligentes |
| `CC_TeambitionProjectLinksField` | Plugin de Teambition |
| `CC_TeambitionTaskLinksField` | Plugin de Teambition |
| `CC_MoneyField` | Componente de finanzas inteligentes |
| `CC_PhoneNumberField` | Componente de extensión |
| `CC_ChineseIdField` | Componente de extensión |
| `YidaDingtalkAgentField` | Plugin de agente DingTalk |

<Warning>
Los campos de plugin fallarán si el plugin requerido no está instalado en su organización. Instale los plugins desde el centro de plugins antes de usar estos tipos de campo.
</Warning>

<Warning>
Los tipos de campo como `MobileField` y `EmailField` no son soportados y serán rechazados. Use solo los tipos listados arriba.
</Warning>

## Gestión de páginas

```bash
# Crear página personalizada
openyida create-page <appType> "<nombre>" [--datasource <jsonOrFile>]

# Compilar y publicar página
openyida publish <ruta_archivo_fuente> <appType> <formUuid> [--skip-lint]

# Configuración de compartir página
openyida save-share-config <appType> <formUuid> <url> <isOpen> [openAuth]
openyida get-page-config <appType> <formUuid>
openyida verify-short-url <appType> <formUuid> <url>
```

## Gestión de datos

```bash
# Gestión unificada de datos
openyida data <acción> <recurso> [args]

# Datos de formulario
openyida data query form <appType> <formUuid> [--page N] [--size N] [--search-json JSON]
openyida data get form <appType> --inst-id <id>
openyida data create form <appType> <formUuid> --data <JSON>  # Las claves JSON pueden ser IDs o etiquetas de campo
openyida data update form <appType> --inst-id <id> --data <JSON>

# Datos de proceso
openyida data query process <appType> <formUuid> [--instance-status ESTADO]
openyida data get process <appType> --process-inst-id <id>
openyida data create process <appType> <formUuid> --process-code <código> --data <JSON>

# Gestión de tareas
openyida data query tasks <appType> --type <todo|done|submitted|cc>
openyida data execute task <appType> --task-id <id> --process-inst-id <id> --out-result <AGREE|DISAGREE>

# Subformularios
openyida data query subform <appType> <formUuid> --inst-id <id> --table-field-id <fieldId>
```

## Centro de tareas global

Consulte los datos del centro de tareas global del usuario actual sin especificar un `appType`.

```bash
# Tareas pendientes
openyida task-center todo [--page N] [--size N] [--keyword TEXT]

# Creadas por mí
openyida task-center created [--page N] [--size N] [--keyword TEXT] [--no-detail]

# Procesadas
openyida task-center done [--page N] [--size N] [--keyword TEXT]

# Con copia
openyida task-center cc [--page N] [--size N] [--keyword TEXT]

# Enviadas por delegación
openyida task-center proxy [--page N] [--size N] [--keyword TEXT]
```

### Tipos de tareas

| Tipo | Descripción | Endpoint API |
|------|-------------|-------------|
| `todo` | Tareas pendientes de acción | `getTodoTasksInCorp` |
| `created` | Procesos que usted inició | `getMyCreateInCorp` |
| `done` | Tareas ya procesadas | `getDoneTasksInCorp` |
| `cc` | Tareas en copia | `getNotifyMeInCorp` |
| `proxy` | Procesos enviados por delegación | `getSubmitAgentInCorp` |

### Parámetros opcionales

| Parámetro | Descripción | Valor predeterminado |
|-----------|-------------|---------------------|
| `--page N` | Número de página | `1` |
| `--size N` | Elementos por página (máx. 100) | `20` |
| `--keyword TEXT` | Palabra clave de búsqueda | — |
| `--no-detail` | Omitir detalles (solo para tipo `created`) | — |

<Info>
  A diferencia de `openyida data query tasks`, el comando `task-center` consulta datos del centro de tareas global y no requiere un parámetro `appType`.
</Info>

## Gestión de conectores

```bash
# Listar conectores
openyida connector list [--page N] [--size N]

# Crear conector
openyida connector create "<nombre>" "<dominio>" --operations <archivo> [opciones]

# Ver detalles del conector
openyida connector detail <connector-id>

# Eliminar conector
openyida connector delete <connector-id> [--force]

# Gestión de acciones
openyida connector add-action --connector-id <id> --operations <archivo> [--confirm]
openyida connector list-actions <connector-id>
openyida connector delete-action <connector-id> <operation-id> [--force]
openyida connector test --connector-id <id> --action <actionId> [opciones]

# Gestión de cuentas de autenticación
openyida connector list-connections <connector-id>
openyida connector create-connection <connector-id> "<nombre>" [opciones]

# Creación inteligente
openyida connector smart-create --curl "<comando_curl>" [--name <nombre>] [--desc <desc>]

# Herramientas auxiliares
openyida connector parse-api --curl "<comando_curl>" --output <archivo>
openyida connector gen-template [ruta_salida]
```

## Opciones globales

| Opción | Descripción |
|------|------|
| `--help`, `-h` | Mostrar información de ayuda |
| `--version`, `-v` | Mostrar número de versión |
| `--qr` | Usar código QR en terminal para iniciar sesión |
| `--force` | Forzar operación, omitir confirmación |
| `--skip-lint` | Omitir la verificación de estándares de codificación Yida durante `publish` |
