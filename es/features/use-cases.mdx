OpenYida es aplicable a múltiples escenarios de negocio, a continuación se muestran ejemplos comunes de tipos de aplicación.

---

## Empresariales

Construye plataformas de negocio multimodulares, soportando formularios complejos, flujos de trabajo y asociaciones de datos.

<AccordionGroup>
  <Accordion title="IPD (Desarrollo Integrado de Productos)">
    **Prompt:**
    ```
    Ayúdame a crear un sistema IPD en Yida para gestionar todo el proceso de producción de chips,
    incluyendo fases de inicio de proyecto, revisión de diseño, producción de obleas y producción en masa
    ```
    **Resultado:** Aplicación multi-formulario con seguimiento de fases, listas de revisión, gestión de hitos y visibilidad inter-equipos.
  </Accordion>

  <Accordion title="CRM (Gestión de Relaciones con Clientes)">
    **Prompt:**
    ```
    Ayúdame a construir un CRM con módulos de leads, clientes, oportunidades y registros de seguimiento
    ```
    **Resultado:** Formularios asociados: Lead → Cliente → Oportunidad → Contrato, con seguimiento de estado y campos de asignación.
  </Accordion>

  <Accordion title="Gestión de RRHH y Permisos">
    **Prompt:**
    ```
    Crea un sistema de gestión de permisos de RRHH con solicitudes de permiso, flujo de aprobación y estadísticas de saldo de vacaciones
    ```
    **Resultado:** Formulario de solicitud de permiso con flujo de aprobación, calculadora de saldo y panel de gerente.
  </Accordion>

  <Accordion title="Gestión de Proyectos">
    **Prompt:**
    ```
    Construye una aplicación de gestión de proyectos con tareas, hitos, miembros del equipo y seguimiento de progreso
    ```
    **Resultado:** Lista de tareas con campos de prioridad/estado/responsable, rastreador de hitos y vista de diagrama de Gantt.
  </Accordion>
</AccordionGroup>

##🔌ación API (Conectores)

Conecta aplicaciones Yida con sistemas externos mediante conectores HTTP.

<AccordionGroup>
  <Accordion title="Notificaciones Webhook">
    **Prompt:**
    ```
    Crea un conector HTTP para nuestro webhook de DingTalk, agrega una acción POST para enviar notificaciones de envío de formularios
    ```
    **Resultado:** Conector configurado con capacidad de prueba, activable en eventos de formulario.
  </Accordion>

  <Accordion title="Creación Inteligente desde curl">
    **Prompt:**
    ```
    Crea inteligentemente un conector desde este comando curl: curl -X POST https://api.example.com/v1/users -H "Authorization: Bearer TOKEN" -d '{"name":"test"}'
    ```
    **Resultado:** Conector configurado automáticamente, extrayendo URL base, encabezados y parámetros de acción.
  </Accordion>
</AccordionGroup>

<Note>
Estos son puntos de partida. Dado que el resultado es una aplicación completamente editable en la plataforma Yida (no una configuración de caja negra), siempre puedes optimizar la aplicación generada a través del editor visual de Yida o dando instrucciones posteriores a la IA.
</Note>
