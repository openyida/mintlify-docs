# Vue d'ensemble de l'architecture

OpenYida est un outil CLI basé sur Node.js, utilisant une architecture en couches, combinant les capacités IA, l'intégration de la plateforme Yida et l'expérience développeur.

## Flux de travail global

![Flux de travail OpenYida](/images/architecture-workflow.png)

Le flux de travail d'OpenYida se divise en 6 étapes :

1. **User Requirement** - L'utilisateur décrit ses besoins en langage naturel
2. **AI Understanding** - L'IA comprend et analyse les besoins de l'utilisateur
3. **PRD Document** - Génération du document de besoins produit
4. **CLI Execution** - Exécution des commandes CLI pour créer l'application
5. **YiDa Platform** - Construction de l'application sur la plateforme Yida
6. **Online Access** - Application en ligne et accessible

## Flux d'authentification

![Flux d'authentification](/images/architecture-auth.png)

Le flux d'authentification d'OpenYida :

1. **config.json** - Stockage des informations de configuration
2. **Login Session** - Établissement de la session de connexion
3. **Cookie Cache** - Mise en cache des informations d'authentification
4. **API Request** - Envoi de requêtes API
5. **YiDa Backend** - Traitement par le backend Yida
6. **Page/Form Output** - Sortie de page/formulaire

## Architecture CLI

![Architecture CLI](/images/architecture-cli.png)

Interaction entre OpenYida CLI et la plateforme Yida :

- **AI Coding Tool** → **OpenYida CLI**
  - Login (connexion)
  - Create App (créer une application)
  - Create Page (créer une page)
  - Publish (publier)
  - Get Schema (obtenir la structure)
- **YiDa Low-Code Platform**
  - App Management (gestion des applications)
  - Form Pages (pages de formulaire)
  - Custom Pages (pages personnalisées)

## Système de compétences

![Système de compétences](/images/architecture-skills.png)

Les compétences d'OpenYida sont divisées en trois catégories :

### Login Skills (compétences de connexion)
- `login` - Connexion à la plateforme Yida
- `logout` - Déconnexion

### Create Skills (compétences de création)
- `create-app` - Créer une application
- `create-page` - Créer une page
- `create-form` - Créer un formulaire
- `get-schema` - Obtenir la structure des données

### Publish Skills (compétences de publication)
- `publish` - Publier une application

## Principes de conception fondamentaux

### 1. Conception modulaire

Chaque module fonctionnel est encapsulé de manière indépendante, interagissant via des interfaces claires :

- **Couche de commandes** : Traite les entrées CLI, appelle les services de base
- **Couche de services** : Encapsule la logique métier, fournit une API unifiée
- **Couche d'adaptation** : Connecte les systèmes externes (IA, plateforme Yida)

### 2. Extension par plugins

Le système de compétences prend en charge le chargement dynamique et l'extension :

```typescript
// Définition de l'interface Skill
interface Skill {
  name: string;
  version: string;
  activate(context: SkillContext): void;
  deactivate(): void;
}
```

### 3. Piloté par la configuration

Prend en charge la configuration multi-environnement, commutation flexible :

- Configuration globale locale (~/.openyida/)
- Configuration au niveau du projet (.openyidarc.json)
- Remplacement par variables d'environnement

### 4. Sécurité des types

Support TypeScript sur toute la chaîne, réduit les erreurs d'exécution.

## Conception de la sécurité

### Sécurité de l'authentification

- Flux standard OAuth 2.0
- Mécanisme de rafraîchissement automatique des jetons
- Stockage chiffré local des informations sensibles

### Transmission des données

- HTTPS sur toute la chaîne
- Vérification de la signature API
- Reprise et contrôle de délai des requêtes

## Optimisation des performances

- **Pool de connexions** : Réutilisation des connexions HTTP
- **Cache intelligent** : Cache local de la configuration et des métadonnées
- **Mises à jour incrémentales** : Transfert uniquement du contenu modifié
- **Contrôle de concurrence** : Limite des appels API simultanés

## Points d'extension

Les développeurs peuvent étendre aux niveaux suivants :

1. **Nouvelles commandes** : Ajouter des commandes CLI personnalisées
2. **Nouvelles compétences** : Étendre les capacités IA
3. **Nouveaux modèles** : Personnaliser les modèles de génération de code
4. **Nouveaux fournisseurs IA** : Intégrer d'autres grands modèles
