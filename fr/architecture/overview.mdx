# Aperçu de l'architecture

OpenYida est un outil CLI basé sur Node.js avec une architecture en couches qui combine organiquement les capacités IA, l'intégration de la plateforme Yida et l'expérience développeur.

## Flux de travail global

![Flux de travail OpenYida](/images/architecture-workflow.png)

Le flux de travail d'OpenYida se compose de 6 étapes :

1. **User Requirement** - L'utilisateur décrit ses besoins en langage naturel
2. **AI Understanding** - L'IA comprend et analyse les besoins de l'utilisateur
3. **PRD Document** - Génération du document des exigences produit
4. **CLI Execution** - Exécution des commandes CLI pour créer l'application
5. **YiDa Platform** - Construction de l'application sur la plateforme Yida
6. **Online Access** - L'application est mise en ligne

## Flux d'authentification

![Flux d'authentification](/images/architecture-auth.png)

Le flux d'authentification d'OpenYida :

1. **config.json** - Stockage des informations de configuration
2. **Login Session** - Établissement de la session de connexion
3. **Cookie Cache** - Mise en cache des informations d'authentification
4. **API Request** - Envoi des requêtes API
5. **YiDa Backend** - Traitement backend Yida
6. **Page/Form Output** - Sortie page/formulaire

## Architecture CLI

![Architecture CLI](/images/architecture-cli.png)

Interaction entre OpenYida CLI et la plateforme Yida :

- **AI Coding Tool** → **OpenYida CLI**
  - Login (Connexion)
  - Create App (Créer application)
  - Create Page (Créer page)
  - Publish (Publier)
  - Get Schema (Obtenir structure)
- **YiDa Low-Code Platform**
  - App Management (Gestion applications)
  - Form Pages (Pages de formulaire)
  - Custom Pages (Pages personnalisées)

## Système de compétences

![Système de compétences](/images/architecture-skills.png)

Les compétences d'OpenYida sont divisées en trois catégories :

### Compétences de connexion
- `login` - Connexion à la plateforme Yida
- `logout` - Déconnexion

### Compétences de création
- `create-app` - Créer une application
- `create-page` - Créer une page
- `create-form` - Créer un formulaire
- `get-schema` - Obtenir la structure de données

### Compétences de publication
- `publish` - Publier une application

## Principes de conception fondamentaux

### 1. Conception modulaire

Chaque module fonctionnel est encapsulé indépendamment et interagit via des interfaces claires :

- **Couche de commandes** : Gère les entrées CLI, appelle les services principaux
- **Couche de services** : Encapsule la logique métier, fournit une API unifiée
- **Couche d'adaptation** : Interface avec les systèmes externes (IA, plateforme Yida)

### 2. Extension par plugins

Le système de compétences supporte le chargement dynamique et l'extension :

```typescript
// Définition de l'interface Skill
interface Skill {
  name: string;
  version: string;
  activate(context: SkillContext): void;
  deactivate(): void;
}
```

### 3. Piloté par configuration

Support de configurations multi-environnements avec changement flexible :

- Configuration globale locale (~/.openyida/)
- Configuration au niveau projet (.openyidarc.json)
- Surcharge par variables d'environnement

### 4. Sécurité de type

Support de type TypeScript sur toute la chaîne pour réduire les erreurs d'exécution.

## Stack technologique

| Couche | Technologie |
|------|------|
| Runtime | Node.js 18+ |
| Langage | TypeScript 5.x |
| Framework CLI | Commander.js |
| Client HTTP | Axios |
| SDK IA | OpenAI SDK, Anthropic SDK |
| Gestion configuration | Cosmiconfig |
| Logging | Winston |
| Tests | Jest |

## Conception de sécurité

### Sécurité d'authentification

- Flux standard OAuth 2.0
- Mécanisme de rafraîchissement automatique des tokens
- Stockage chiffré local des informations sensibles

### Transmission de données

- HTTPS sur toute la chaîne
- Vérification de signature API
- Contrôle des retry et timeout des requêtes

## Optimisation des performances

- **Pool de connexions** : Réutilisation des connexions HTTP
- **Cache intelligent** : Mise en cache locale des configurations et métadonnées
- **Mises à jour incrémentielles** : Transmission uniquement du contenu modifié
- **Contrôle de concurrence** : Limitation des appels API simultanés

## Points d'extension

Les développeurs peuvent étendre aux niveaux suivants :

1. **Nouvelles commandes** : Ajouter des commandes CLI personnalisées
2. **Nouvelles compétences** : Étendre les capacités IA
3. **Nouveaux modèles** : Modèles de génération de code personnalisés
4. **Nouveaux fournisseurs IA** : Intégrer d'autres LLM
