OpenYida est conçu pour l'interaction en langage naturel. Les prompts suivants ont été testés et peuvent être utilisés dans tous les outils IA pris en charge.

---

## Opérations sur formulaires et pages

### Créer des pages de formulaire

```
Aidez-moi à créer une page de formulaire xxx
```
```
Créer un formulaire de demande de congé, incluant : nom de l'employé, département, date de début, date de fin, motif de congé
```

### Ajouter des champs de formulaire

```
Ajouter un champ [nom du champ] à la page [nom de la page], type de champ : [TextField/NumberField/DateField/SelectField/...]
```
```
Ajouter un champ "département" de type liste déroulante au formulaire d'informations des employés, options : département d'ingénierie, département marketing, département des ventes, département des ressources humaines
```

<Note>
Seuls les [types de champs supportés](/fr/features/skills#types-de-champs-supportés) sont acceptés. Les types non supportés (ex : `MobileField`, `EmailField`) seront rejetés.
</Note>

### Modifier les propriétés des champs

```
Rendre le champ [nom du champ] de la page [nom de la page] obligatoire
```
```
Rendre le champ [nom du champ] de la page [nom de la page] en lecture seule
```
```
Changer l'étiquette du champ [nom du champ] de la page [nom de la page] pour [nouvelle étiquette]
```

### Pages d'affichage personnalisées

```
Créer une page d'affichage personnalisée pour la page [nom de la page]
```
```
Générer une landing page pour un produit de collaboration d'entreprise
```

## Flash Note vers PRD

```
Convertir les notes flash DingTalk en documents PRD
```

```
Générer un PRD à partir de cette transcription de réunion : [coller les notes]
```

```
Créer un document d'exigences de projet à partir de la note flash jointe
```

```
Convertir les notes de réunion en un document d'exigences produit structuré
```

<Note>
Nécessite une connexion. La commande lit les transcriptions de réunions et utilise l'IA pour générer des documents PRD structurés. La sortie est enregistrée dans le répertoire `prd/`.
</Note>

## Intégration et automatisation

### Créer un flux d'intégration

```
Créer un flux d'intégration qui envoie des notifications lorsque de nouveaux enregistrements sont soumis
```

```
Configurer une automatisation sur [nom du formulaire] pour notifier [équipe/utilisateur] lors des nouvelles soumissions
```

```
Créer un workflow [nom du formulaire] qui déclenche des notifications par e-mail aux [destinataires]
```

### Ajouter des actions automatisées

```
Ajouter une action de recherche de données à l'intégration [nom du flux]
```

```
Ajouter une action de création d'enregistrement pour créer automatiquement des entrées dans [formulaire cible]
```

<Note>
Les flux d'intégration se déclenchent sur les événements de formulaire (insertion, mise à jour, suppression) et peuvent exécuter des notifications, des recherches de données et des créations d'enregistrements. Utilisez la CLI pour créer et configurer les flux.
</Note>

```

```

```

```

```

```

```
Envoyer un message : [contenu du message]
```

### Interroger les présences

```
Lister les enregistrements de présence
```

```
Vérifier mon statut de présence
```

<Info>
La commande `dws` nécessite l'installation séparée de la CLI DingTalk. Exécutez `openyida dws install` si ce n'est pas déjà installé. La sortie est par défaut au format tableau ; ajoutez `-f json` pour une sortie JSON.
</Info>

---

<Tip>
**Astuce :** Plus vous fournissez de contexte, meilleurs seront les résultats. Au lieu de dire "créer un formulaire", dites "créer un formulaire de demande de congé, incluant nom de l'employé, département, type de congé (liste déroulante : congé annuel/maladie/personnel), dates de début et de fin, motif — nom de l'employé et dates obligatoires".
</Tip>
