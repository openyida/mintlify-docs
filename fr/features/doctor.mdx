OpenYida intègre un puissant outil de diagnostic `doctor`, aidant les développeurs à vérifier l'environnement, diagnostiquer les problèmes, réparer automatiquement, surveiller en continu la santé des applications.

## Utilisation de base

### Exécuter le diagnostic

```bash
openyida doctor
```

### Réparation automatique

```bash
openyida doctor --fix
```

### Générer un rapport

```bash
# Format Markdown
openyida doctor --report markdown

# Format JSON
openyida doctor --report json

# Format HTML
openyida doctor --report html
```

## Éléments de vérification de l'environnement

| Élément de vérification | Description | Réparation automatique |
|--------|------|----------|
| Version CLI | Vérifiez que la version openyida correspond à SKILL.md | ✅ |
| Initialisation project/ | Vérifiez que le répertoire de travail project/ existe | ✅ |
| Version de Node.js | Vérifiez si ≥ 18 | ❌ |
| Playwright | Vérifiez l'installation | ✅ |
| config.json | Vérifiez le fichier de configuration | ✅ |
| Connexion Yida | Vérifiez l'état de connexion | ✅ |
| Connectivité réseau | Vérifiez la connectivité aliwork.com | ❌ |

---

## Éléments de diagnostic d'application

| Élément de vérification | Description |
|--------|------|
| Fichier PRD | Vérifiez l'existence du document de besoins |
| Code source des pages | Vérifiez les fichiers source, console.log, fichiers vides |
| Cache Schema | Vérifiez le format des fichiers cache |
| React Hooks | Vérifiez les normes d'utilisation des Hooks |

## Mode de surveillance de la santé

Démarrez la surveillance continue, vérifiez chaque minute :

```bash
openyida doctor --monitor
```

<Warning>
La fonction de surveillance de la santé est actuellement en version bêta. L'API et les formats de sortie peuvent changer dans les futures versions.
</Warning>

Exemple de sortie :

```
[14:32:01] Santé : 95/100 📈
  Réussi : 18/19 | Erreurs : 0 | Avertissements : 1

[14:33:01] Santé : 100/100 ➡️
  Réussi : 19/19 | Erreurs : 0 | Avertissements : 0
```

Appuyez sur `Ctrl+C` pour arrêter la surveillance.

## Diagnostic en ligne

Diagnostic en ligne pour les applications déjà publiées :

```bash
openyida doctor --production --app <appId>
```

Contenu de vérification :
- Validité de l'ID d'application
- Journaux d'erreurs en ligne
- Indicateurs de performance

## Tickets et VOC

### Créer un ticket (Bug)

```bash
openyida doctor --create-ticket
```

Entrez de manière interactive le titre et la description, soumettez automatiquement à GitHub Issues.

### Créer un VOC (besoin)

```bash
openyida doctor --create-voc
```

### Soumission intelligente

Le système détermine automatiquement s'il s'agit d'un Bug ou d'un besoin :

```bash
openyida doctor --auto-submit
```

## Pré-vérification

Exécutez automatiquement la vérification avant les opérations clés :

### Vérification avant publication

```bash
# Exécuté automatiquement
openyida publish exécutera automatiquement la pré-vérification
```

### Vérification avant création

```bash
# Exécuté automatiquement
openyida create-app exécutera automatiquement la pré-vérification
```

---

## Types de réparation

| Type | Description | Exemple |
|------|------|------|
| `AUTO` | Réparation automatique | Créer config.json, supprimer le cache corrompu |
| `COMMAND` | Réparation par commande | Exécuter `npx playwright install` |
| `MANUAL` | Réparation manuelle | Installer gh CLI, écrire PRD |

## Calcul de la santé

Formule de calcul du score de santé (0-100) :

```
Santé = Taux de réussite × 100 - Nombre d'erreurs × 10 - Nombre d'avertissements × 3
```

| Score | État |
|------|------|
| 90-100 | Excellent 🟢 |
| 70-89 | Bon 🟡 |
| 50-69 | Moyen 🟠 |
| < 50 | À améliorer 🔴 |
