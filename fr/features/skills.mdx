OpenYida fournit de riches commandes CLI, prenant en charge la gestion du cycle de vie complet des applications.

---

## Environnement et authentification

```bash
# Détection de l'environnement
openyida env

# Connexion
openyida login [--qr]              # --qr utilise le code QR du terminal
openyida login --wukong            # Spécifique à l'environnement Wukong

# Déconnexion
openyida logout

# Gestion de l'état de connexion
openyida auth status               # Voir l'état de connexion
openyida auth login                # Exécuter la connexion
openyida auth refresh              # Actualiser l'état de connexion
openyida auth logout               # Se déconnecter

# Gestion des organisations
openyida org list                  # Lister les organisations accessibles
openyida org switch --corp-id <id> # Basculer vers une organisation

# Initialiser le répertoire de travail
openyida copy [--force]            # Copier le répertoire project

# Outil de diagnostic
openyida doctor [--fix]            # Diagnostic de l'environnement
openyida doctor --monitor          # Surveillance de la santé
openyida doctor --report <format>  # Générer un rapport (json|markdown|html)
```

## Gestion des applications

```bash
# Créer une application
openyida create-app "<nom>" [desc] [icon] [color] [themeColor]

# Exporter une application
openyida export <appType> [output] # Générer un package de migration

# Importer une application
openyida import <file> [name]      # Reconstruire l'application à partir du package de migration
```

## Gestion des formulaires

```bash
# Créer un formulaire
openyida create-form create <appType> "<nom du formulaire>" <champsJSON> \
  [--layout <mise en page>] \
  [--theme <thème>] \
  [--label-align <alignement>]

# Mettre à jour un formulaire
openyida create-form update <appType> <formUuid> <modificationsJSON>

# Obtenir le Schema du formulaire
openyida get-schema <appType> <formUuid>

# Mettre à jour la configuration du formulaire
openyida update-form-config <appType> <formUuid> <isRenderNav> <title>
```

### Types de champs supportés

La commande `create-form` n'accepte que les types de champs suivants. L'utilisation d'un type non supporté renvoie une erreur.

#### Champs de base

Champs intégrés ne nécessitant aucun plugin supplémentaire.

| Type | Description |
|------|-------------|
| `TextField` | Texte sur une ligne |
| `TextareaField` | Texte multiligne |
| `NumberField` | Nombre |
| `RateField` | Notation |
| `DateField` | Date |
| `RadioField` | Boutons radio |
| `SelectField` | Liste déroulante (sélection unique) |
| `CheckboxField` | Cases à cocher |
| `MultiSelectField` | Liste déroulante (sélection multiple) |
| `CascadeSelectField` | Sélection en cascade |
| `CascadeDateField` | Date en cascade |
| `EmployeeField` | Sélecteur de membre |
| `DepartmentSelectField` | Sélecteur de département |
| `AssociationFormField` | Formulaire associé |
| `AssociationQuery` | Requête d'association |
| `AttachmentField` | Pièce jointe |
| `ImageField` | Image |
| `AddressField` | Adresse |
| `CountrySelectField` | Sélecteur de pays/région |
| `TableField` | Sous-formulaire / détail |
| `SerialNumberField` | Numéro de série |

#### Champs avancés

Champs intégrés avec des fonctionnalités étendues. Certaines fonctions peuvent nécessiter une activation.

| Type | Description |
|------|-------------|
| `EditorField` | Éditeur de texte riche |
| `DigitalSignatureField` | Signature numérique |
| `LocationField` | Localisation GPS |

#### Champs de plugin

Ces champs nécessitent l'installation du plugin correspondant depuis le centre de plugins avant utilisation.

| Type | Plugin requis |
|------|---------------|
| `TimeZoneDateField` | Composant de fuseau horaire international |
| `CC_AIButtonField` | Composant IA |
| `CC_PG_ESignField` | Composant finance intelligente |
| `CC_TeambitionProjectLinksField` | Plugin Teambition |
| `CC_TeambitionTaskLinksField` | Plugin Teambition |
| `CC_MoneyField` | Composant finance intelligente |
| `CC_PhoneNumberField` | Composant d'extension |
| `CC_ChineseIdField` | Composant d'extension |
| `YidaDingtalkAgentField` | Plugin agent DingTalk |

<Warning>
Les champs de plugin échoueront si le plugin requis n'est pas installé dans votre organisation. Installez les plugins depuis le centre de plugins avant d'utiliser ces types de champs.
</Warning>

<Warning>
Les types de champs tels que `MobileField` et `EmailField` ne sont pas supportés et seront rejetés. N'utilisez que les types listés ci-dessus.
</Warning>

## Gestion des données

```bash
# Gestion unifiée des données
openyida data <action> <ressource> [args]

# Données de formulaire
openyida data query form <appType> <formUuid> [--page N] [--size N] [--search-json JSON]
openyida data get form <appType> --inst-id <id>
openyida data create form <appType> <formUuid> --data <JSON>  # Les clés JSON peuvent être des IDs ou des libellés de champ
openyida data update form <appType> --inst-id <id> --data <JSON>

# Données de processus
openyida data query process <appType> <formUuid> [--instance-status STATUS]
openyida data get process <appType> --process-inst-id <id>
openyida data create process <appType> <formUuid> --process-code <code> --data <JSON>

# Gestion des tâches
openyida data query tasks <appType> --type <todo|done|submitted|cc>
openyida data execute task <appType> --task-id <id> --process-inst-id <id> --out-result <AGREE|DISAGREE>

# Sous-formulaires
openyida data query subform <appType> <formUuid> --inst-id <id> --table-field-id <fieldId>
```

## Centre de tâches global

Interrogez les données du centre de tâches global de l'utilisateur actuel sans spécifier d'`appType`.

```bash
# Tâches à faire
openyida task-center todo [--page N] [--size N] [--keyword TEXT]

# Créées par moi
openyida task-center created [--page N] [--size N] [--keyword TEXT] [--no-detail]

# Traitées
openyida task-center done [--page N] [--size N] [--keyword TEXT]

# En copie
openyida task-center cc [--page N] [--size N] [--keyword TEXT]

# Soumises par délégation
openyida task-center proxy [--page N] [--size N] [--keyword TEXT]
```

### Types de tâches

| Type | Description | Point de terminaison API |
|------|-------------|------------------------|
| `todo` | Tâches en attente de traitement | `getTodoTasksInCorp` |
| `created` | Processus que vous avez initiés | `getMyCreateInCorp` |
| `done` | Tâches que vous avez traitées | `getDoneTasksInCorp` |
| `cc` | Tâches en copie | `getNotifyMeInCorp` |
| `proxy` | Processus soumis par délégation | `getSubmitAgentInCorp` |

### Paramètres optionnels

| Paramètre | Description | Valeur par défaut |
|-----------|-------------|-------------------|
| `--page N` | Numéro de page | `1` |
| `--size N` | Éléments par page (max 100) | `20` |
| `--keyword TEXT` | Mot-clé de recherche | — |
| `--no-detail` | Omettre les détails (uniquement pour le type `created`) | — |

<Info>
  Contrairement à `openyida data query tasks`, la commande `task-center` interroge les données du centre de tâches global et ne nécessite pas de paramètre `appType`.
</Info>

## Intégration et automatisation

Créez et gérez des flux logiques (intégration et automatisation) prenant en charge les déclencheurs d'événements de formulaire, les notifications et les opérations de données.

```bash
# Créer un flux logique
openyida integration create <appType> <formUuid> <flowName> [options]
```

### Paramètres

| Paramètre | Obligatoire | Description |
|-----------|-------------|-------------|
| `appType` | Oui | ID de l'application (ex : `APP_XXXX`) |
| `formUuid` | Oui | UUID du formulaire déclencheur (ex : `FORM-XXXX`) |
| `flowName` | Oui | Nom du flux logique |

### Options

| Option | Défaut | Description |
|--------|--------|-------------|
| `--process-code <code>` | Auto-généré | processCode existant (`LPROC-xxx`) |
| `--receivers <userId,...>` | Vide | ID utilisateurs pour notifications DingTalk (séparés par virgules) |
| `--title <title>` | Identique à flowName | Titre de notification, supporte `#{fieldId-ComponentType}#` |
| `--content <content>` | `"Nouvel enregistrement soumis"` | Contenu de la notification |
| `--events <insert,update>` | `insert` | Événements déclencheurs : `insert`/`update`/`delete`/`comment` |
| `--add-data-form-uuid <formUuid>` | — | UUID du formulaire cible pour le nœud d'ajout de données |
| `--add-data-assignment <targetFieldId:valueType:value>` | — | Attribution de champ (spécifiable plusieurs fois). `valueType` : `processVar`/`literal`/`column` |
| `--publish` | Non publié | Publier immédiatement après l'enregistrement |

### Sortie

```json
{
  "success": true,
  "published": false,
  "processCode": "LPROC-XXXX",
  "flowName": "Notification de nouvel enregistrement",
  "appType": "APP_XXX",
  "formUuid": "FORM-XXX",
  "formEventTypes": ["insert"]
}
```

<Info>
  Vous devez être connecté avant d'exécuter cette commande. Avec `--publish`, `published` sera `true` ; en cas d'échec, `published` sera `false` avec un champ `warning`.
</Info>

## Note flash vers PRD

Convertit les notes flash DingTalk (transcriptions de réunions) en documents PRD structurés à l'aide de l'IA.

```bash
# Depuis un fichier
openyida flash-to-prd --file <chemin> [--name <nom du projet>] [--max-tokens <nombre>]

# Depuis l'entrée standard
cat meeting.txt | openyida flash-to-prd --name <nom du projet> [--max-tokens <nombre>]
```

### Paramètres

| Paramètre | Alias | Description | Par défaut |
|-----------|-------|-------------|------------|
| `--file` | `-f` | Chemin vers le fichier de note flash | — |
| `--name` | `-n` | Nom du projet (détecté automatiquement si omis) | — |
| `--max-tokens` | — | Nombre maximal de tokens pour la réponse IA | `8000` |

La commande lit le contenu de la note flash depuis le fichier spécifié ou depuis l'entrée standard, prétraite le texte (y compris l'extraction des métadonnées de réunion et l'identification des intervenants), puis appelle le service IA Yida pour générer un PRD structuré. Le fichier PRD est enregistré dans le répertoire `prd/` de votre projet.

### Sortie

La commande affiche un résultat JSON sur stdout :

```json
{
  "success": true,
  "projectName": "Mon Projet",
  "prdFile": "/path/to/prd/Mon-Projet.md",
  "contentLength": 4096,
  "meetingRecognition": {
    "metaFields": 3,
    "a1Sections": 2,
    "speakers": 5
  }
}
```

<Info>
  Vous devez être connecté avant d'exécuter cette commande. Exécutez `openyida login` au préalable si vous n'êtes pas authentifié.
</Info>
