# आर्किटेक्चर अवलोकन

OpenYida Node.js पर आधारित एक CLI टूल है जो परतदार आर्किटेक्चर डिज़ाइन अपनाता है, AI क्षमताओं, Yida प्लेटफ़ॉर्म एकीकरण और डेवलपर अनुभव को कार्बनिक रूप से जोड़ता है।

## समग्र वर्कफ़्लो

![OpenYida वर्कफ़्लो](/images/architecture-workflow.png)

OpenYida का वर्कफ़्लो 6 चरणों में विभाजित है:

1. **User Requirement** - उपयोगकर्ता प्राकृतिक भाषा में आवश्यकताओं का वर्णन करता है
2. **AI Understanding** - AI उपयोगकर्ता की आवश्यकताओं को समझता और विश्लेषण करता है
3. **PRD Document** - उत्पाद आवश्यकता दस्तावेज़ उत्पन्न करता है
4. **CLI Execution** - एप्लिकेशन बनाने के लिए CLI कमांड निष्पादित करता है
5. **Yida Platform** - Yida प्लेटफ़ॉर्म पर एप्लिकेशन बनाता है
6. **Online Access** - एप्लिकेशन प्रकाशित और सुलभ होता है

## प्रमाणीकरण प्रवाह

![प्रमाणीकरण प्रवाह](/images/architecture-auth.png)

OpenYida का प्रमाणीकरण प्रवाह:

1. **config.json** - कॉन्फ़िगरेशन जानकारी संग्रहीत करता है
2. **Login Session** - लॉगिन सत्र स्थापित करता है
3. **Cookie Cache** - प्रमाणीकरण जानकारी कैश करता है
4. **API Request** - API अनुरोध भेजता है
5. **Yida Backend** - Yida बैकएंड प्रोसेस करता है
6. **Page/Form Output** - पेज/फ़ॉर्म आउटपुट

## CLI आर्किटेक्चर

![CLI आर्किटेक्चर](/images/architecture-cli.png)

OpenYida CLI और Yida प्लेटफ़ॉर्म के बीच इंटरैक्शन:

- **AI Coding Tool** → **OpenYida CLI**
  - Login（लॉगिन）
  - Create App（एप्लिकेशन बनाएं）
  - Create Page（पेज बनाएं）
  - Publish（प्रकाशित करें）
  - Get Schema（संरचना प्राप्त करें）
- **Yida Low-Code Platform**
  - App Management（एप्लिकेशन प्रबंधन）
  - Form Pages（फ़ॉर्म पेज）
  - Custom Pages（कस्टम पेज）

## Skill सिस्टम

![Skill सिस्टम](/images/architecture-skills.png)

OpenYida के skills तीन श्रेणियों में विभाजित हैं:

### Login Skills（लॉगिन skills）
- `login` - Yida प्लेटफ़ॉर्म में लॉगिन करें
- `logout` - लॉगआउट करें

### Create Skills（निर्माण skills）
- `create-app` - एप्लिकेशन बनाएं
- `create-page` - पेज बनाएं
- `create-form` - फ़ॉर्म बनाएं
- `get-schema` - डेटा संरचना प्राप्त करें

### Publish Skills（प्रकाशन skills）
- `publish` - एप्लिकेशन प्रकाशित करें

## केंद्रीय डिज़ाइन सिद्धांत

### 1. मॉड्यूलर डिज़ाइन

प्रत्येक फ़ंक्शनल मॉड्यूल स्वतंत्र रूप से encapsulated है, स्पष्ट interfaces के माध्यम से इंटरैक्ट करता है:

- **कमांड लेयर**: CLI इनपुट प्रोसेस करता है, कोर services को कॉल करता है
- **सर्विस लेयर**: बिज़नेस लॉजिक encapsulate करता है, बाहरी रूप से unified API प्रदान करता है
- **एडैप्टेशन लेयर**: बाहरी systems को कनेक्ट करता है（AI, Yida प्लेटफ़ॉर्म）

### 2. प्लगइन एक्सटेंशन

Skill सिस्टम dynamic loading और extension का समर्थन करता है:

```typescript
// Skill interface परिभाषा
interface Skill {
  name: string;
  version: string;
  activate(context: SkillContext): void;
  deactivate(): void;
}
```

### 3. कॉन्फ़िगरेशन-ड्रिवन

multi-environment कॉन्फ़िगरेशन का समर्थन करता है, लचीला switching:

- स्थानीय global कॉन्फ़िगरेशन（~/.openyida/）
- प्रोजेक्ट-स्तरीय कॉन्फ़िगरेशन（.openyidarc.json）
- environment variables द्वारा override

### 4. प्रकार सुरक्षा

full-chain TypeScript प्रकार समर्थन, runtime errors को कम करता है।

## डेटा प्रवाह

### एप्लिकेशन निर्माण प्रवाह

```
User Input → AI Service → Code Generation → Yida API → Application
                ↑                              ↓
           PRD Analysis                  Validation
                ↑                              ↓
           Template Selection           Deployment
```

### डेटा संचालन प्रवाह

```
User Command → Config Load → Auth Check → Yida API → Data Response
                                              ↓
                                         Cache/Store
```

| टेस्टिंग | Jest |

## सुरक्षा डिज़ाइन

### प्रमाणीकरण सुरक्षा

- OAuth 2.0 मानक प्रवाह
- Token स्वचालित refresh mechanism
- संवेदनशील जानकारी स्थानीय encrypted storage

### डेटा ट्रांसमिशन

- full-chain HTTPS
- API signature verification
- अनुरोध retry और timeout control

## प्रदर्शन अनुकूलन

- **Connection Pool**: HTTP connections को reuse करता है
- **Smart Cache**: कॉन्फ़िगरेशन और metadata स्थानीय cache
- **Incremental Update**: केवल changed content को transmit करता है
- **Concurrency Control**: simultaneous API calls को limit करता है

## एक्सटेंशन पॉइंट्स

डेवलपर्स निम्नलिखित स्तरों पर extension कर सकते हैं:

1. **नए कमांड**: custom CLI commands जोड़ें
2. **नए Skills**: AI क्षमताओं को extend करें
3. **नए टेम्पलेट**: custom code generation templates
4. **नए AI Provider**: अन्य large models को integrate करें
