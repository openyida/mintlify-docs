# आर्किटेक्चर अवलोकन

OpenYida एक Node.js-आधारित CLI टूल है जो परतदार आर्किटेक्चर डिज़ाइन का उपयोग करता है, जो AI क्षमताओं, Yida प्लेटफॉर्म एकीकरण और डेवलपर अनुभव को जैविक रूप से जोड़ता है।

## समग्र कार्यप्रवाह

![OpenYida कार्यप्रवाह](/images/architecture-workflow.png)

OpenYida का कार्यप्रवाह 6 चरणों में विभाजित है:

1. **User Requirement** - उपयोगकर्ता प्राकृतिक भाषा में आवश्यकताओं का वर्णन करता है
2. **AI Understanding** - AI उपयोगकर्ता आवश्यकताओं को समझता और विश्लेषण करता है
3. **PRD Document** - उत्पाद आवश्यकता दस्तावेज़ उत्पन्न करता है
4. **CLI Execution** - ऐप बनाने के लिए CLI कमांड निष्पादित करता है
5. **YiDa Platform** - Yida प्लेटफॉर्म पर ऐप बनाता है
6. **Online Access** - ऐप ऑनलाइन हो जाता है

## प्रमाणीकरण प्रवाह

![प्रमाणीकरण प्रवाह](/images/architecture-auth.png)

OpenYida का प्रमाणीकरण प्रवाह:

1. **config.json** - कॉन्फ़िगरेशन जानकारी संग्रहीत करता है
2. **Login Session** - लॉगिन सत्र स्थापित करता है
3. **Cookie Cache** - प्रमाणीकरण जानकारी कैश करता है
4. **API Request** - API अनुरोध भेजता है
5. **YiDa Backend** - Yida बैकएंड प्रसंस्करण
6. **Page/Form Output** - पृष्ठ/फॉर्म आउटपुट

## CLI आर्किटेक्चर

![CLI आर्किटेक्चर](/images/architecture-cli.png)

OpenYida CLI और Yida प्लेटफॉर्म का पारस्परिक संवाद:

- **AI Coding Tool** → **OpenYida CLI**
  - Login (लॉगिन)
  - Create App (ऐप बनाएं)
  - Create Page (पृष्ठ बनाएं)
  - Publish (प्रकाशित करें)
  - Get Schema (स्कीमा प्राप्त करें)
- **YiDa Low-Code Platform**
  - App Management (ऐप प्रबंधन)
  - Form Pages (फॉर्म पृष्ठ)
  - Custom Pages (कस्टम पृष्ठ)

## स्किल सिस्टम

![स्किल सिस्टम](/images/architecture-skills.png)

OpenYida की स्किल तीन श्रेणियों में विभाजित हैं:

### लॉगिन स्किल
- `login` - Yida प्लेटफॉर्म में लॉगिन
- `logout` - लॉगआउट

### क्रिएट स्किल
- `create-app` - ऐप बनाएं
- `create-page` - पृष्ठ बनाएं
- `create-form` - फॉर्म बनाएं
- `get-schema` - डेटा संरचना प्राप्त करें

### पब्लिश स्किल
- `publish` - ऐप प्रकाशित करें

## कोर डिज़ाइन सिद्धांत

### 1. मॉड्यूलर डिज़ाइन

प्रत्येक कार्यात्मक मॉड्यूल स्वतंत्र रूप से समाहित है और स्पष्ट इंटरफेस के माध्यम से परस्पर क्रिया करता है:

- **कमांड लेयर**: CLI इनपुट संभालता है, कोर सेवाओं को कॉल करता है
- **सर्विस लेयर**: व्यावसायिक तर्क समाहित करता है, एकीकृत API प्रदान करता है
- **एडाप्टर लेयर**: बाहरी सिस्टम (AI, Yida प्लेटफॉर्म) के साथ इंटरफेस

### 2. प्लगइन एक्सटेंशन

स्किल सिस्टम गतिशील लोडिंग और विस्तार का समर्थन करता है:

```typescript
// स्किल इंटरफेस परिभाषा
interface Skill {
  name: string;
  version: string;
  activate(context: SkillContext): void;
  deactivate(): void;
}
```

### 3. कॉन्फ़िगरेशन-संचालित

मल्टी-एनवायरनमेंट कॉन्फ़िगरेशन का समर्थन, लचीला स्विचिंग:

- लोकल ग्लोबल कॉन्फ़िगरेशन (~/.openyida/)
- प्रोजेक्ट-लेवल कॉन्फ़िगरेशन (.openyidarc.json)
- एनवायरनमेंट वेरिएबल ओवरराइड

### 4. टाइप सुरक्षा

पूर्ण-चेन TypeScript टाइप समर्थन, रनटाइम त्रुटियों को कम करता है।

## डेटा प्रवाह

### ऐप निर्माण प्रवाह

```
User Input → AI Service → Code Generation → Yida API → Application
                ↑                              ↓
           PRD Analysis                  Validation
                ↑                              ↓
           Template Selection           Deployment
```

### डेटा ऑपरेशन प्रवाह

```
User Command → Config Load → Auth Check → Yida API → Data Response
                                              ↓
                                         Cache/Store
```

## सुरक्षा डिज़ाइन

### प्रमाणीकरण सुरक्षा

- OAuth 2.0 मानक प्रवाह
- टोकन स्वचालित रिफ्रेश तंत्र
- संवेदनशील जानकारी का स्थानीय एन्क्रिप्टेड भंडारण

### डेटा संचरण

- पूर्ण-चेन HTTPS
| API हस्ताक्षर सत्यापन
| अनुरोध पुन:प्रयास और टाइमआउट नियंत्रण

## प्रदर्शन अनुकूलन

- **कनेक्शन पूल**: HTTP कनेक्शन पुन:उपयोग
- **स्मार्ट कैशिंग**: कॉन्फ़िगरेशन और मेटाडेटा का स्थानीय कैशिंग
- **वृद्धिशील अपडेट**: केवल बदली गई सामग्री प्रसारित करें
- **समवर्तन नियंत्रण**: एक साथ API कॉल सीमित करें

## विस्तार बिंदु

डेवलपर्स निम्नलिखित स्तरों पर विस्तार कर सकते हैं:

1. **नए कमांड**: कस्टम CLI कमांड जोड़ें
2. **नई स्किल**: AI क्षमताओं का विस्तार
3. **नए टेम्पलेट**: कस्टम कोड जनरेशन टेम्पलेट
4. **नए AI प्रोवाइडर**: अन्य LLM को एकीकृत करें
