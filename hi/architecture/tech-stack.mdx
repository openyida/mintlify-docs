## तकनीकी स्टैक आर्किटेक्चर

![तकनीकी स्टैक आर्किटेक्चर](/images/architecture-tech-stack.png)

OpenYida का तकनीकी स्टैक दो भागों में विभाजित है: विकास वातावरण और क्लाउड प्लेटफ़ॉर्म:

### विकास वातावरण (Development Environment)
- **Node.js** - रनटाइम वातावरण
- **Git** - संस्करण नियंत्रण

### स्वचालित परीक्षण (Playwright)
- ब्राउज़र स्वचालन
- एंड-टू-एंड परीक्षण
- पेज स्क्रीनशॉट तुलना

### Yida क्लाउड प्लेटफ़ॉर्म (Yida Cloud Platform)
- **App** - एप्लिकेशन प्रबंधन
- **Form** - फ़ॉर्म पेज
- **Custom Page** - कस्टम पेज

## कोर मॉड्यूल विवरण

### अंतर्राष्ट्रीयकरण (i18n.js)

12 भाषाओं का समर्थन करने वाला अंतर्राष्ट्रीयकरण सिस्टम:

```javascript
const { t, getLanguage, setLanguage } = require('./i18n');

// अनुवाद उपयोग करें
console.log(t('login.success'));
console.log(t('create_app.usage', 'appName'));

// भाषा प्राप्त/सेट करें
console.log(getLanguage()); // 'zh'
setLanguage('en');
```

**भाषा पहचान प्राथमिकता:**
1. `OPENYIDA_LANG` पर्यावरण चर
2. सिस्टम पर्यावरण चर `LANG` / `LC_ALL`
3. डिफ़ॉल्ट: सरलीकृत चीनी

### वातावरण पहचान (env.js)

स्वचालित रूप से चल रहे AI टूल वातावरण का पता लगाता है:

| पहचान विधि | टूल |
|----------|------|
| डायरेक्टरी अस्तित्व पहचान | Claude Code, Cursor, OpenCode, Aone Copilot, Qoder |
| विशेष फ़ाइल पहचान | Wukong (悟空) |

### निदान इंजन (doctor.js)

तीन-परत आर्किटेक्चर डिज़ाइन:

1. **DiagnosticEngine** - निदान शेड्यूलर
2. **EnvironmentChecker** - वातावरण चेकर
3. **ApplicationChecker** - एप्लिकेशन चेकर
4. **FixEngine** - स्मार्ट मरम्मत इंजन

## फंक्शनल मॉड्यूल विवरण

### प्रमाणीकरण मॉड्यूल (auth/)

```
auth/
├.js          # Yida लॉगिन (Cookie कैश + QR कोड स्कैन)
├.js           # लॉगिन स्थिति प्रबंधन
├.js            # संगठन प्रबंधन
└-login.js       # टर्मिनल QR कोड लॉगिन
```

**लॉगिन प्रक्रिया:**
1. वर्तमान AI टूल वातावरण का पता लगाएं
2. वातावरण के अनुसार लॉगिन विधि चुनें (QR कोड स्कैन / CDP / फ़ाइल पढ़ना)
3. Cookie प्राप्त करें और स्थानीय एन्क्रिप्टेड कैश में स्टोर करें
4. बाद के कमांड स्वचालित रूप से कैश किए गए लॉगिन स्थिति का उपयोग करते हैं

### कनेक्टर मॉड्यूल (connector/)

```
connector/
├.js                    # Yida कनेक्टर API एनकैप्सुलेशन
├-parser.js            # curl कमांड पार्सर
├-generator.js       # एक्शन जनरेटर
├-generator.js         # विवरण जनरेटर
├-smart-create.js # स्मार्ट क्रिएशन एंट्री
└
```

**स्मार्ट क्रिएशन प्रक्रिया:**
```
curl कमांड → पार्सर → API जानकारी निकालना → प्रमाणीकरण पहचान → मौजूदा कनेक्टर से मिलान → कॉन्फ़िगरेशन जनरेट → क्रिएट/अपडेट
```

### डेटा प्रबंधन मॉड्यूल (data-management.js)

एकीकृत डेटा ऑपरेशन इंटरफ़ेस, समर्थन:

| संसाधन प्रकार | ऑपरेशन |
|----------|------|
| form | query, get, create, update |
| process | query, get, create |
| task | query, execute |
| subform | query |

const BASE_URL = 'https://www.aliwork.com';

| एंडपॉइंट | उपयोग |
|------|------|
| `/dingtalk/web/APP_xxx/xxx` | एप्लिकेशन के भीतर इंटरफ़ेस |

## सुरक्षा डिज़ाइन

1. **Cookie एन्क्रिप्टेड स्टोरेज** - स्थानीय कैश एन्क्रिप्टेड स्टोर
2. **CSRF Token सत्यापन** - प्रत्येक अनुरोध Token ले जाता है
3. **कोई हार्डकोडेड क्रेडेंशियल नहीं** - सभी क्रेडेंशियल गतिशील रूप से प्राप्त
4. **वातावरण अलगाव** - विभिन्न AI टूल वातावरण स्वतंत्र

---

## प्रदर्शन अनुकूलन

1. **अनुवाद फ़ाइलें लेज़ी लोडिंग** - मांग पर भाषा पैकेज लोड
2. **Schema स्थानीय कैश** - API कॉल कम करें
3. **समवर्ती अनुरोध नियंत्रण** - अत्यधिक अनुरोधों से सीमित होने से बचें
4. **वृद्धिशील अपडेट** - फ़ॉर्म अपडेट ओवरराइट के बजाय मर्ज का उपयोग करते हैं

## विस्तार तंत्र

### नया कमांड जोड़ें

1. `lib/` में `new-feature.js` बनाएं
2. `bin/yida.js` में कमांड रूटिंग पंजीकृत करें
3. `README.md` और दस्तावेज़ अपडेट करें
4. अंतर्राष्ट्रीयकरण कुंजी जोड़ें

### नई भाषा जोड़ें

1. `lib/core/locales/` में `xx.js` बनाएं
2. सभी कुंजियों का अनुवाद करें (`zh.js` को आधार के रूप में उपयोग करें)
3. `i18n.js` में `SUPPORTED_LANGUAGES` अपडेट करें
4. पूर्णता जांचने के लिए `npm run validate-i18n` चलाएं
