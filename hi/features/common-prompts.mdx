OpenYida प्राकृतिक भाषा बातचीत के लिए डिज़ाइन किया गया है। निम्नलिखित प्रॉम्प्ट्स का सत्यापन किया गया है और वे सभी समर्थित AI टूल में काम करते हैं।

---

## ऐप निर्माण

```
मुझे एक xxx एप्लिकेशन बनाने में मदद करें
```
```
मुझे एक CRM बनाने में मदद करें
```
```
मुझे एक व्यक्तिगत वेतन कैलकुलेटर ऐप बनाने में मदद करें
```
```
मुझे Yida पर एक IPD सिस्टम बनाने में मदद करें ताकि चिप उत्पादन की पूरी प्रक्रिया का प्रबंधन किया जा सके
```
```
आवश्यकताओं के दस्तावेज़ के आधार पर एक ऐप जनरेट करें: [PRD पेस्ट करें]
```
```
Yida पर एक जन्मदिन की बधाई मिनी-गेम बनाएं
```
```
एक लालटेन फेस्टिवल पहेली गेम बनाएं
```

## फॉर्म और पेज संचालन

### फॉर्म पेज बनाएं

```
मुझे एक xxx फॉर्म पेज बनाने में मदद करें
```
```
एक छुट्टी का आवेदन फॉर्म बनाएं जिसमें शामिल हों: कर्मचारी का नाम, विभाग, प्रारंभ तिथि, समाप्ति तिथि, छुट्टी का कारण
```

### फॉर्म फील्ड जोड़ें

```
[पेज नाम] में एक [फील्ड नाम] फील्ड जोड़ें, फील्ड प्रकार: [TextField/NumberField/DateField/SelectField/...]
```
```
कर्मचारी जानकारी फॉर्म में एक "विभाग" ड्रॉपडाउन फील्ड जोड़ें, विकल्प: इंजीनियरिंग, मार्केटिंग, बिक्री, मानव संसाधन
```

<Note>
केवल [समर्थित फील्ड प्रकार](/hi/features/skills#समर्थित-फील्ड-प्रकार) स्वीकार किए जाते हैं। असमर्थित प्रकार (जैसे `MobileField`, `EmailField`) अस्वीकार कर दिए जाएंगे।
</Note>

### फील्ड गुण संशोधित करें

```
[पेज नाम] में [फील्ड नाम] फील्ड को अनिवार्य बनाएं
```
```
[पेज नाम] में [फील्ड नाम] फील्ड को केवल-पढ़ने योग्य बनाएं
```
```
[पेज नाम] में [फील्ड नाम] फील्ड का लेबल [नया लेबल] में बदलें
```

### कस्टम डिस्प्ले पेज

```
[पेज नाम] के लिए एक कस्टम डिस्प्ले पेज बनाएं
```
```
एक एंटरप्राइज़ सहयोग उत्पाद के लिए एक लैंडिंग पेज जनरेट करें
```

## Flash Note से PRD

```
DingTalk Flash Notes को PRD दस्तावेज़ों में बदलें
```

```
इस बैठक के ट्रांसक्रिप्ट से एक PRD जनरेट करें: [नोट्स पेस्ट करें]
```

```
संलग्न Flash Note से एक प्रोजेक्ट आवश्यकताओं दस्तावेज़ बनाएं
```

```
बैठक के नोट्स को एक संरचित उत्पाद आवश्यकताओं दस्तावेज़ में बदलें
```

<Note>
लॉगिन आवश्यक है। यह कमांड बैठक के ट्रांसक्रिप्ट पढ़ता है और AI का उपयोग करके संरचित PRD दस्तावेज़ जनरेट करता है। आउटपुट `prd/` निर्देशिका में सहेजा जाता है।
</Note>

## एकीकरण और स्वचालन

### एकीकरण प्रवाह बनाएं

```
एक एकीकरण प्रवाह बनाएं जो नए रिकॉर्ड सबमिट होने पर सूचनाएं भेजता है
```

```
[फॉर्म नाम] पर एक स्वचालन सेट करें जो नए सबमिशन पर [टीम/उपयोगकर्ता] को सूचित करे
```

```
एक [फॉर्म नाम] वर्कफ़्लो बनाएं जो [प्राप्तकर्ताओं] को ईमेल सूचनाएं ट्रिगर करता है
```

### स्वचालित कार्रवाइयां जोड़ें

```
[प्रवाह नाम] एकीकरण में एक डेटा लुकअप कार्रवाई जोड़ें
```

```
[लक्ष्य फॉर्म] में स्वचालित रूप से प्रविष्टियां बनाने के लिए एक रिकॉर्ड निर्माण कार्रवाई जोड़ें
```

<Note>
एकीकरण प्रवाह फॉर्म ईवेंट (सम्मिलित, अद्यतन, हटाएं) पर ट्रिगर होते हैं और सूचनाएं, डेटा लुकअप और रिकॉर्ड निर्माण निष्पादित कर सकते हैं। प्रवाह बनाने और कॉन्फ़िगर करने के लिए CLI का उपयोग करें।
</Note>

## DingTalk एकीकरण

### खोजें और क्वेरी करें

```
[नाम] नामक एक संपर्क खोजें
```

```
सभी कैलेंडर ईवेंट सूचीबद्ध करें
```

```
मेरे अनुमोदन अनुरोध क्वेरी करें
```

### बनाएं और प्रबंधित करें

```
एक टू-डू कार्य बनाएं: [कार्य विवरण]
```

```
एक संदेश भेजें: [संदेश सामग्री]
```

### उपस्थिति क्वेरी करें

```
उपस्थिति रिकॉर्ड सूचीबद्ध करें
```

```
मेरी उपस्थिति स्थिति जांचें
```

<Info>
`dws` कमांड के लिए DingTalk CLI को अलग से इंस्टॉल करने की आवश्यकता होती है। यदि पहले से इंस्टॉल नहीं है तो `openyida dws install` चलाएं। आउटपुट डिफ़ॉल्ट रूप से तालिका प्रारूप में होता है; JSON आउटपुट के लिए `-f json` जोड़ें।
</Info>

<Tip>
**सुझाव:** जितना अधिक संदर्भ आप प्रदान करेंगे, परिणाम उतने ही बेहतर होंगे। "एक फॉर्म बनाएं" कहने के बजाय, कहें "एक छुट्टी अनुमोदन फॉर्म बनाएं जिसमें कर्मचारी का नाम, विभाग, छुट्टी का प्रकार (ड्रॉपडाउन: वार्षिक/बीमार/व्यक्तिगत), प्रारंभ और समाप्ति तिथियां, कारण शामिल हों — कर्मचारी का नाम और तिथियां अनिवार्य करें"।
</Tip>
