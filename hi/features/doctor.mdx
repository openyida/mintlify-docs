## बुनियादी उपयोग

### निदान चलाएं

```bash
openyida doctor
```

### स्वचालित मरम्मत

```bash
openyida doctor --fix
```

### रिपोर्ट बनाएं

```bash
# Markdown प्रारूप
openyida doctor --report markdown

# JSON प्रारूप
openyida doctor --report json

# HTML प्रारूप
openyida doctor --report html
```

## पर्यावरण जांच आइटम

| आइटम | विवरण | स्वचालित मरम्मत |
|--------|------|----------|
| CLI संस्करण | जांचें कि openyida संस्करण SKILL.md से मेल खाता है | ✅ |
| project/ प्रारंभ | जांचें कि project/ कार्य निर्देशिका मौजूद है | ✅ |
| Node.js संस्करण | जांचें यदि ≥ 18 | ❌ |
| Playwright | जांचें यदि स्थापित है | ✅ |
| config.json | कॉन्फ़िगरेशन फ़ाइल जांचें | ✅ |
| Yida लॉगिन | लॉगिन स्थिति जांचें | ✅ |
| नेटवर्क कनेक्टिविटी | aliwork.com कनेक्टिविटी जांचें | ❌ |

## एप्लिकेशन निदान आइटम

| आइटम | विवरण |
|--------|------|
| PRD फ़ाइल | जांचें यदि आवश्यकता दस्तावेज़ मौजूद है |
| पेज सोर्स कोड | सोर्स कोड फ़ाइलें, console.log, खाली फ़ाइलें जांचें |
| Schema कैश | कैश फ़ाइल प्रारूप जांचें |
| React Hooks | Hooks उपयोग मानक जांचें |

## स्वास्थ्य निगरानी मोड

निरंतर निगरानी शुरू करें, हर मिनट जांचें:

```bash
openyida doctor --monitor
```

<Warning>
स्वास्थ्य निगरानी सुविधा वर्तमान में बीटा में है। API और आउटपुट प्रारूप भविष्य के रिलीज में बदल सकते हैं।
</Warning>

आउटपुट उदाहरण:

```
[14:32:01] स्वास्थ्य: 95/100  पास: 18/19 | त्रुटि: 0 | चेतावनी: 1

[14:33:01] स्वास्थ्य: 100/100➡
  पास: 19/19 | त्रुटि: 0 | चेतावनी: 0
```

निगरानी रोकने के लिए `Ctrl+C` दबाएं।

## ऑनलाइन निदान

पहले से प्रकाशित एप्लिकेशन के लिए ऑनलाइन निदान:

```bash
openyida doctor --production --app <appId>
```

जांच सामग्री:
- एप्लिकेशन ID वैधता
- ऑनलाइन त्रुटि लॉग
- प्रदर्शन मेट्रिक्स

---

## टिकट और VOC

### टिकट बनाएं (Bug)

```bash
openyida doctor --create-ticket
```

शीर्षक और विवरण इंटरैक्टिव इनपुट, स्वचालित रूप से GitHub Issues में सबमिट करें।

### VOC बनाएं (आवश्यकता)

```bash
openyida doctor --create-voc
```

### स्मार्ट सबमिशन

सिस्टम स्वचालित रूप से निर्णय लेता है कि Bug है या आवश्यकता:

```bash
openyida doctor --auto-submit
```

## पूर्व-जांच

महत्वपूर्ण संचालन से पहले स्वचालित रूप से जांच चलाएं:

### प्रकाशन से पहले जांच

```bash
# स्वचालित रूप से चलाएं
openyida publish से पहले स्वचालित रूप से पूर्व-जांच चलाता है
```

### निर्माण से पहले जांच

```bash
# स्वचालित रूप से चलाएं
openyida create-app से पहले स्वचालित रूप से पूर्व-जांच चलाता है
```

## मरम्मत प्रकार

| प्रकार | विवरण | उदाहरण |
|------|------|------|
| `AUTO` | स्वचालित मरम्मत | config.json बनाएं, क्षतिग्रस्त कैश हटाएं |
| `COMMAND` | कमांड मरम्मत | `npx playwright install` चलाएं |
| `MANUAL` | मैन्युअल मरम्मत | gh CLI स्थापित करें, PRD लिखें |

## स्वास्थ्य गणना

स्वास्थ्य स्कोर (0-100) गणना सूत्र:

```
स्वास्थ्य = पास दर × 100 - त्रुटि × 10 - चेतावनी × 3
```

| स्कोर | स्थिति |
|------|------|
| 90-100 | उत्कृष्ट
| 70-89 | अच्छा
| 50-69 | सामान्य
| < 50 | सुधार आवश्यक🔴
