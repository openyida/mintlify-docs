OpenYida समृद्ध CLI कमांड प्रदान करता है, जो एप्लिकेशन के पूर्ण जीवनचक्र प्रबंधन का समर्थन करता है।

---

## वातावरण और प्रमाणीकरण

```bash
# वातावरण जांच
openyida env

# लॉगिन
openyida login [--qr]              # --qr टर्मिनल में QR कोड का उपयोग करें
openyida login --wukong            # Wukong वातावरण के लिए विशेष

# लॉगआउट
openyida logout

# लॉगिन स्थिति प्रबंधन
openyida auth status               # लॉगिन स्थिति देखें
openyida auth login                # लॉगिन करें
openyida auth refresh              # सत्र रीफ्रेश करें
openyida auth logout               # लॉगआउट करें

# संगठन प्रबंधन
openyida org list                  # सुलभ संगठनों की सूची
openyida org switch --corp-id <id> # संगठन बदलें

# कार्य निर्देशिका आरंभ करें
openyida copy [--force]            # project निर्देशिका कॉपी करें

# निदान उपकरण
openyida doctor [--fix]            # वातावरण निदान
openyida doctor --monitor          # स्वास्थ्य निगरानी
openyida doctor --report <format>  # रिपोर्ट बनाएं (json|markdown|html)
```

## एप्लिकेशन प्रबंधन

```bash
# एप्लिकेशन बनाएं
openyida create-app "<नाम>" [desc] [icon] [color] [themeColor]

# एप्लिकेशन निर्यात करें
openyida export <appType> [output] # माइग्रेशन पैकेज बनाएं

# एप्लिकेशन आयात करें
openyida import <file> [name]      # पैकेज से एप्लिकेशन पुनर्निर्माण करें
```

## फॉर्म प्रबंधन

```bash
# फॉर्म बनाएं
openyida create-form create <appType> "<नाम>" <फ़ील्डJSON> \
  [--layout <लेआउट>] \
  [--theme <थीम>] \
  [--label-align <संरेखण>]

# फॉर्म अपडेट करें
openyida create-form update <appType> <formUuid> <संशोधनJSON>

# फॉर्म Schema प्राप्त करें
openyida get-schema <appType> <formUuid>

# फॉर्म कॉन्फ़िगरेशन अपडेट करें
openyida update-form-config <appType> <formUuid> <isRenderNav> <शीर्षक>
```

### समर्थित फ़ील्ड प्रकार

`create-form` कमांड केवल निम्नलिखित फ़ील्ड प्रकारों को स्वीकार करती है। असमर्थित प्रकार का उपयोग करने पर त्रुटि लौटाई जाती है।

#### बुनियादी फ़ील्ड

अंतर्निहित फ़ील्ड जिनके लिए किसी प्लगइन की आवश्यकता नहीं है।

| प्रकार | विवरण |
|--------|--------|
| `TextField` | एक पंक्ति का टेक्स्ट |
| `TextareaField` | बहु-पंक्ति टेक्स्ट |
| `NumberField` | संख्या |
| `RateField` | रेटिंग |
| `DateField` | दिनांक |
| `RadioField` | रेडियो बटन |
| `SelectField` | ड्रॉपडाउन (एकल चयन) |
| `CheckboxField` | चेकबॉक्स |
| `MultiSelectField` | ड्रॉपडाउन (बहु चयन) |
| `CascadeSelectField` | कैस्केड चयन |
| `CascadeDateField` | कैस्केड दिनांक |
| `EmployeeField` | सदस्य चयनकर्ता |
| `DepartmentSelectField` | विभाग चयनकर्ता |
| `AssociationFormField` | संबद्ध फ़ॉर्म |
| `AssociationQuery` | संबद्ध क्वेरी |
| `AttachmentField` | अनुलग्नक |
| `ImageField` | छवि |
| `AddressField` | पता |
| `CountrySelectField` | देश/क्षेत्र चयनकर्ता |
| `TableField` | उप-फ़ॉर्म / विवरण |
| `SerialNumberField` | क्रम संख्या |

#### उन्नत फ़ील्ड

विस्तारित कार्यक्षमता वाले अंतर्निहित फ़ील्ड। कुछ सुविधाओं को सक्रियण की आवश्यकता हो सकती है।

| प्रकार | विवरण |
|--------|--------|
| `EditorField` | रिच टेक्स्ट संपादक |
| `DigitalSignatureField` | डिजिटल हस्ताक्षर |
| `LocationField` | GPS स्थान |

#### प्लगइन फ़ील्ड

इन फ़ील्ड को उपयोग से पहले प्लगइन केंद्र से संबंधित प्लगइन स्थापित करना आवश्यक है।

| प्रकार | आवश्यक प्लगइन |
|--------|----------------|
| `TimeZoneDateField` | अंतर्राष्ट्रीय समय क्षेत्र घटक |
| `CC_AIButtonField` | AI घटक |
| `CC_PG_ESignField` | स्मार्ट वित्त घटक |
| `CC_TeambitionProjectLinksField` | Teambition प्लगइन |
| `CC_TeambitionTaskLinksField` | Teambition प्लगइन |
| `CC_MoneyField` | स्मार्ट वित्त घटक |
| `CC_PhoneNumberField` | विस्तार घटक |
| `CC_ChineseIdField` | विस्तार घटक |
| `YidaDingtalkAgentField` | DingTalk एजेंट प्लगइन |

<Warning>
प्लगइन फ़ील्ड विफल हो जाएंगे यदि आवश्यक प्लगइन आपके संगठन में स्थापित नहीं है। इन फ़ील्ड प्रकारों का उपयोग करने से पहले प्लगइन केंद्र से प्लगइन स्थापित करें।
</Warning>

<Warning>
`MobileField`, `EmailField` जैसे फ़ील्ड प्रकार समर्थित नहीं हैं और अस्वीकार कर दिए जाएंगे। केवल ऊपर सूचीबद्ध प्रकारों का उपयोग करें।
</Warning>

## पेज प्रबंधन

```bash
# कस्टम पेज बनाएं
openyida create-page <appType> "<नाम>" [--datasource <jsonOrFile>]

# पेज संकलित और प्रकाशित करें
openyida publish <स्रोत_फ़ाइल_पथ> <appType> <formUuid> [--skip-lint]

# पेज शेयर कॉन्फ़िगरेशन
openyida save-share-config <appType> <formUuid> <url> <isOpen> [openAuth]
openyida get-page-config <appType> <formUuid>
openyida verify-short-url <appType> <formUuid> <url>
```

## डेटा प्रबंधन

```bash
# एकीकृत डेटा प्रबंधन
openyida data <कार्य> <संसाधन> [args]

# फॉर्म डेटा
openyida data query form <appType> <formUuid> [--page N] [--size N] [--search-json JSON]
openyida data get form <appType> --inst-id <id>
openyida data create form <appType> <formUuid> --data <JSON>  # JSON कुंजियाँ फ़ील्ड ID या फ़ील्ड लेबल हो सकती हैं
openyida data update form <appType> --inst-id <id> --data <JSON>

# प्रक्रिया डेटा
openyida data query process <appType> <formUuid> [--instance-status स्थिति]
openyida data get process <appType> --process-inst-id <id>
openyida data create process <appType> <formUuid> --process-code <कोड> --data <JSON>

# कार्य प्रबंधन
openyida data query tasks <appType> --type <todo|done|submitted|cc>
openyida data execute task <appType> --task-id <id> --process-inst-id <id> --out-result <AGREE|DISAGREE>

# उप-फॉर्म
openyida data query subform <appType> <formUuid> --inst-id <id> --table-field-id <fieldId>
```

## कनेक्टर प्रबंधन

```bash
# कनेक्टर सूची
openyida connector list [--page N] [--size N]

# कनेक्टर बनाएं
openyida connector create "<नाम>" "<डोमेन>" --operations <फ़ाइल> [विकल्प]

# कनेक्टर विवरण देखें
openyida connector detail <connector-id>

# कनेक्टर हटाएं
openyida connector delete <connector-id> [--force]

# एक्शन प्रबंधन
openyida connector add-action --connector-id <id> --operations <फ़ाइल> [--confirm]
openyida connector list-actions <connector-id>
openyida connector delete-action <connector-id> <operation-id> [--force]
openyida connector test --connector-id <id> --action <actionId> [विकल्प]

# प्रमाणीकरण खाता प्रबंधन
openyida connector list-connections <connector-id>
openyida connector create-connection <connector-id> "<नाम>" [विकल्प]

# स्मार्ट निर्माण
openyida connector smart-create --curl "<curl_कमांड>" [--name <नाम>] [--desc <विवरण>]

# सहायक उपकरण
openyida connector parse-api --curl "<curl_कमांड>" --output <फ़ाइल>
openyida connector gen-template [आउटपुट_पथ]
```

## वैश्विक विकल्प

| विकल्प | विवरण |
|------|------|
| `--help`, `-h` | सहायता जानकारी दिखाएं |
| `--version`, `-v` | संस्करण संख्या दिखाएं |
| `--qr` | लॉगिन के लिए टर्मिनल में QR कोड का उपयोग करें |
| `--force` | ऑपरेशन फोर्स करें, पुष्टि छोड़ें |
| `--skip-lint` | `publish` के दौरान Yida कोडिंग मानक पूर्व-जांच छोड़ें |
