## 1. विस्तृत प्रॉम्प्ट लिखें

जनरेट किए गए ऐप्स की गुणवत्ता प्रॉम्प्ट के विस्तार स्तर के समानुपाती होती है। तुलना:

<CodeGroup>

```text अस्पष्ट (बचें)
एक CRM बनाएं
```

```text विस्तृत (अनुशंसित)
तीन संबंधित फॉर्म के साथ एक CRM बनाएं:
1. लीड फॉर्म — फ़ील्ड: नाम, कंपनी, फोन, ईमेल, स्रोत (ड्रॉपडाउन: रेफरल/वेबसाइट/इवेंट/कोल्ड कॉल), स्थिति (ड्रॉपडाउन: नया लीड/संपर्क किया/योग्य/अयोग्य), सौंपा गया (सदस्य चयनकर्ता)
2. ग्राहक फॉर्म — लीड से रूपांतरित, फ़ील्ड: कंपनी का नाम, उद्योग, वार्षिक राजस्व, प्रमुख संपर्क, खाता प्रबंधक
3. अवसर फॉर्म — ग्राहकों से लिंक, फ़ील्ड: अवसर का नाम, राशि, चरण (ड्रॉपडाउन: प्रस्ताव/वार्ता/जीता/हारा), अनुमानित बंद होने की तारीख, नोट्स
```

</CodeGroup>

## 2. आवश्यकता दस्तावेज़ों का उपयोग करें

जटिल ऐप्स के लिए, पहले एक PRD लिखें, फिर इसे AI को दें:

```
निम्नलिखित आवश्यकता दस्तावेज़ के आधार पर Yida ऐप जनरेट करें:

[PRD पेस्ट करें — फ़ील्ड नाम, प्रकार, सत्यापन नियम, वर्कफ़्लो आदि]
```

यह दृष्टिकोण पुनरावृत्ति प्रॉम्प्टिंग की तुलना में अधिक सटीक परिणाम देता है।

---

## 3. वृद्धिशील पुनरावृत्ति

एक प्रॉम्प्ट में सब कुछ बनाने का प्रयास न करें। पहले कोर संरचना बनाएं, फिर धीरे-धीरे जटिलता जोड़ें:

```
चरण 1: मूल कर्मचारी जानकारी फॉर्म बनाएं
चरण 2: कर्मचारियों से लिंक किए गए अवकाश अनुरोध फॉर्म जोड़ें
चरण 3: अवकाश अनुमोदन वर्कफ़्लो कॉन्फ़िगर करें
चरण 4: लंबित अनुमोदन दिखाने वाला मैनेजर डैशबोर्ड जोड़ें
चरण 5: डैशबोर्ड को सार्वजनिक पृष्ठ के रूप में प्रकाशित करें
```

## 4. प्रत्येक चरण सत्यापित करें

AI द्वारा प्रत्येक घटक बनाने के बाद, इसे सत्यापित करें:

```
[फॉर्म नाम] फॉर्म का वर्तमान Schema दिखाएं
जांचें कि [पृष्ठ नाम] पृष्ठ प्रकाशित और सुलभ है या नहीं
```

समस्याओं को जल्दी पकड़ें इससे पहले कि वे बड़ी समस्याओं में जमा हो जाएं।

---

## 5. प्रोजेक्ट वर्किंग डायरेक्टरी का पुन: उपयोग करें

प्रति प्रोजेक्ट `openyida copy` केवल एक बार चलाएं, हर सत्र में नहीं। `project/config.json` फ़ाइल `appId` और `corpId` संग्रहीत करती है — AI इसका उपयोग लक्ष्य ऐप को जानने के लिए करता है। प्रोजेक्ट के बीच में इस फ़ाइल को हटाएं नहीं।

## 6. पर्यावरण-विशिष्ट लॉगिन

विभिन्न AI टूल अलग-अलग लॉगिन विधियों की आवश्यकता होती है:

| AI टूल | लॉगिन कमांड |
|---------|----------|
| Claude Code, Cursor, VSCode | `openyida login` |
| वुकोंग | `openyida login --wukong` |
| OpenCode, Aone Copilot | `openyida login` |

टूल स्विच करते समय, सक्रिय पर्यावरण की पुष्टि करने के लिए पहले `openyida env` चलाएं।

---

## 7. कस्टम पेज इमेज के लिए CDN का उपयोग करें

जब कस्टम पेज में इमेज होती हैं, तो विश्वसनीय रेंडरिंग सुनिश्चित करने के लिए पहले उन्हें CDN पर अपलोड करें:

```bash
openyida cdn-config   # Alibaba Cloud OSS + CDN कॉन्फ़िगर करें (एक बार)
openyida cdn-upload ./my-image.png   # स्थिर CDN URL प्राप्त करें
```

फिर अपने पेज प्रॉम्प्ट में CDN URL का संदर्भ दें:
```
इस CDN URL का उपयोग करके लैंडिंग पेज में हीरो बैनर इमेज जोड़ें: https://cdn.example.com/banner.png
```

## 8. मल्टी-ऑर्गनाइज़ेशन सेटअप संभालें

जब आप कई Yida संगठनों तक पहुंच सकते हैं:

```bash
openyida org list          # सभी उपलब्ध संगठन देखें
openyida org switch --corp-id <id>   # लक्ष्य संगठन पर स्विच करें
openyida env               # स्विच की पुष्टि करें
```

ऐप बनाने से पहले हमेशा सक्रिय संगठन सत्यापित करें ताकि गलत स्थान पर निर्माण से बचा जा सके।

---

## 9. डिबगिंग से पहले Doctor चलाएं

विफल कमांड्स को डिबग करने में समय बिताने से पहले, चलाएं:

```bash
openyida doctor --fix
```

स्वचालित रूप से सबसे सामान्य समस्याओं का पता लगाता है और हल करता है: समाप्त सत्र, लापता कॉन्फ़िगरेशन फ़ाइलें, पर्यावरण बेमेल।

## 10. AI-सहायता प्राप्त PR का स्वागत है

OpenYida स्वयं में योगदान करते समय, AI-जनरेटेड कोड को पूरी तरह से स्वीकार करें। PR विवरण में उपयोग किए गए AI टूल का उल्लेख करें — यह मेंटेनर्स को जनरेशन संदर्भ को समझने और अधिक कुशलता से समीक्षा करने में मदद करता है।
