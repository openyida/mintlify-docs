# アーキテクチャ概要

OpenYidaはNode.jsベースのCLIツールで、階層アーキテクチャ設計を採用し、AI能力、Yidaプラットフォーム統合、開発者エクスペリエンスを有機的に結合しています。

## 全体ワークフロー

![OpenYida ワークフロー](/images/architecture-workflow.png)

OpenYidaのワークフローは6つの段階に分かれています：

1. **User Requirement** - ユーザーが自然言語で要件を記述
2. **AI Understanding** - AIがユーザーの要件を理解・分析
3. **PRD Document** - 製品要件ドキュメントを生成
4. **CLI Execution** - CLIコマンドを実行してアプリを作成
5. **YiDa Platform** - Yidaプラットフォームでアプリを構築
6. **Online Access** - アプリをオンラインでアクセス可能に

## 認証フロー

![認証フロー](/images/architecture-auth.png)

OpenYidaの認証フロー：

1. **config.json** - 設定情報を保存
2. **Login Session** - ログインセッションを確立
3. **Cookie Cache** - 認証情報をキャッシュ
4. **API Request** - APIリクエストを送信
5. **YiDa Backend** - Yidaバックエンドで処理
6. **Page/Form Output** - ページ/フォームを出力

## CLIアーキテクチャ

![CLIアーキテクチャ](/images/architecture-cli.png)

OpenYida CLIとYidaプラットフォームの対話：

- **AI Coding Tool** → **OpenYida CLI**
  - Login（ログイン）
  - Create App（アプリ作成）
  - Create Page（ページ作成）
  - Publish（公開）
  - Get Schema（構造取得）
- **YiDa Low-Code Platform**
  - App Management（アプリ管理）
  - Form Pages（フォームページ）
  - Custom Pages（カスタムページ）

## スキル体系

![スキル体系](/images/architecture-skills.png)

OpenYidaのスキルは3つのカテゴリに分類されます：

### Login Skills（ログインスキル）
- `login` - Yidaプラットフォームにログイン
- `logout` - ログアウト

### Create Skills（作成スキル）
- `create-app` - アプリを作成
- `create-page` - ページを作成
- `create-form` - フォームを作成
- `get-schema` - データ構造を取得

### Publish Skills（公開スキル）
- `publish` - アプリを公開

## コア設計原則

### 1. モジュール設計

各機能モジュールは独立してカプセル化され、明確なインターフェースを通じて対話します：

- **コマンド層**: CLI入力を処理し、コアサービスを呼び出す
- **サービス層**: ビジネスロジックをカプセル化し、統一APIを提供
- **アダプタ層**: 外部システム（AI、Yidaプラットフォーム）と対話

### 2. プラグイン拡張

Skillシステムは動的ロードと拡張をサポートします：

```typescript
// Skillインターフェース定義
interface Skill {
  name: string;
  version: string;
  activate(context: SkillContext): void;
  deactivate(): void;
}
```

### 3. 設定駆動

マルチ環境設定をサポートし、柔軟に切り替え可能：

- ローカルグローバル設定（~/.openyida/）
- プロジェクトレベル設定（.openyidarc.json）
- 環境変数による上書き

### 4. タイプセーフティ

全チェーンTypeScriptタイプサポート、ランタイムエラーを削減。

## セキュリティ設計

### 認証セキュリティ

- OAuth 2.0標準フロー
- Token自動更新メカニズム
- 機密情報のローカル暗号化保存

### データ転送

- 全チェーンHTTPS
- API署名検証
- リクエスト再試行とタイムアウト制御

## パフォーマンス最適化

- **接続プール**: HTTP接続を再利用
- **スマートキャッシュ**: 設定とメタデータをローカルキャッシュ
- **増分更新**: 変更内容のみを転送
- **並行制御**: 同時API呼び出しを制限

## 拡張ポイント

開発者は以下のレベルで拡張できます：

1. **新コマンド**: カスタムCLIコマンドを追加
2. **新Skill**: AI能力を拡張
3. **新テンプレート**: カスタムコード生成テンプレート
4. **新AI Provider**: 他の大規模モデルに接続
