# Architecture Overview

OpenYida is a Node.js-based CLI tool with a layered architecture design that organically combines AI capabilities, Yida platform integration, and developer experience.

## Overall Workflow

![OpenYida Workflow](/images/architecture-workflow.png)

OpenYida's workflow consists of 6 stages:

1. **User Requirement** - Users describe requirements in natural language
2. **AI Understanding** - AI understands and analyzes user requirements
3. **PRD Document** - Generate product requirement document
4. **CLI Execution** - Execute CLI commands to create applications
5. **YiDa Platform** - Build applications on Yida platform
6. **Online Access** - Application goes live for access

## Authentication Flow

![Authentication Flow](/images/architecture-auth.png)

OpenYida's authentication flow:

1. **config.json** - Store configuration information
2. **Login Session** - Establish login session
3. **Cookie Cache** - Cache authentication information
4. **API Request** - Send API requests
5. **YiDa Backend** - Yida backend processing
6. **Page/Form Output** - Output page/form

![CLI Architecture](/images/architecture-cli.png)

## Skill System

![Skill System](/images/architecture-skills.png)

OpenYida's skills are divided into three categories:

### Login Skills
- `login` - Login to Yida platform
- `logout` - Logout

### Create Skills
- `create-app` - Create application
- `create-page` - Create page
- `create-form` - Create form
- `get-schema` - Get data structure

### Publish Skills
- `publish` - Publish application

## Core Design Principles

### 1. Modular Design

Each functional module is independently encapsulated and interacts through clear interfaces:

- **Command Layer**: Handles CLI input, calls core services
- **Service Layer**: Encapsulates business logic, provides unified API
- **Adapter Layer**: Interfaces with external systems (AI, Yida platform)

### 2. Plugin Extension

Skill system supports dynamic loading and extension:

```typescript
// Skill interface definition
interface Skill {
  name: string;
  version: string;
  activate(context: SkillContext): void;
  deactivate(): void;
}
```

### 3. Configuration Driven

Supports multi-environment configuration with flexible switching:

- Local global configuration (~/.openyida/)
- Project-level configuration (.openyidarc.json)
- Environment variable override

### 4. Type Safety

Full-chain TypeScript type support to reduce runtime errors.

## Security Design

### Authentication Security

- OAuth 2.0 standard flow
- Token auto-refresh mechanism
- Local encrypted storage of sensitive information

### Data Transmission

- Full-chain HTTPS
- API signature verification
- Request retry and timeout control

## Performance Optimization

- **Connection Pool**: Reuse HTTP connections
- **Smart Caching**: Local caching of configuration and metadata
- **Incremental Updates**: Only transmit changed content
- **Concurrency Control**: Limit simultaneous API calls

## Extension Points

Developers can extend at the following levels:

1. **New Commands**: Add custom CLI commands
2. **New Skills**: Extend AI capabilities
3. **New Templates**: Custom code generation templates
4. **New AI Providers**: Integrate other LLMs
