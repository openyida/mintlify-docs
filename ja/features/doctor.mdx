OpenYidaには強力な診断ツール`doctor`が組み込まれており、開発者が環境をチェックし、問題を診断し、自動修復し、アプリの健全性を継続的に監視するのに役立ちます。

---

## 機能概要

<CardGroup cols={2}>
  <Card title="環境チェック" icon="laptop-code">
    Node.js、Python、Playwright、ログイン状態などの環境依存関係をチェックします。
  </Card>
  <Card title="アプリ診断" icon="magnifying-glass">
    PRD、ページソースコード、Schemaキャッシュ、React Hooks規範をチェックします。
  </Card>
  <Card title="スマート修復" icon="wand-magic">
    修復可能な問題を自動的に修復し、手動修復ガイダンスを提供します。
  </Card>
  <Card title="ヘルスモニタリング" icon="heart-pulse">
    アプリの健全性を継続的に監視し、トレンドレポートを生成します。
  </Card>
</CardGroup>

## 基本的な使用法

### 診断の実行

```bash
openyida doctor
```

### 自動修復

```bash
openyida doctor --fix
```

### レポートの生成

```bash
# Markdown形式
openyida doctor --report markdown

# JSON形式
openyida doctor --report json

# HTML形式
openyida doctor --report html
```

## 環境チェック項目

| チェック項目 | 説明 | 自動修復 |
|--------|------|----------|
| Node.jsバージョン | ≥ 16かどうかをチェック | ❌ |
| Pythonバージョン | ≥ 3.10かどうかをチェック | ❌ |
| Playwright | インストールされているかどうかをチェック | ✅ |
| Chromium | ブラウザがインストールされているかどうかをチェック | ✅ |
| gh CLI | GitHub CLIをチェック | ❌ |
| ghログイン | gh CLIログイン状態をチェック | ✅ |
| config.json | 設定ファイルをチェック | ✅ |
| Skills | スキルパッケージのインストールをチェック | ❌ |
| Yidaログイン | ログイン状態をチェック | ✅ |
| ネットワーク接続 | aliwork.com接続性をチェック | ❌ |

---

## アプリ診断項目

| チェック項目 | 説明 |
|--------|------|
| PRDファイル | 要件ドキュメントが存在するかどうかをチェック |
| ページソースコード | ソースコードファイル、console.log、空ファイルをチェック |
| Schemaキャッシュ | キャッシュファイル形式をチェック |
| React Hooks | Hooks使用規範をチェック |

## ヘルスモニタリングモード

継続監視を開始し、1分ごとにチェックします：

```bash
openyida doctor --monitor
```

<Warning>
ヘルスモニタリング機能は現在ベータ版です。APIと出力形式は将来のリリースで変更される可能性があります。
</Warning>

出力例：

```
[14:32:01] 健全性: 95/100 📈
  通過: 18/19 | エラー: 0 | 警告: 1

[14:33:01] 健全性: 100/100 ➡️
  通過: 19/19 | エラー: 0 | 警告: 0
```

`Ctrl+C`でモニタリングを停止します。

## 本番診断

公開済みアプリの本番診断：

```bash
openyida doctor --production --app <appId>
```

チェック内容：
- アプリIDの有効性
- 本番エラーログ
- パフォーマンス指標

## チケットとVOC

### チケット作成（Bug）

```bash
openyida doctor --create-ticket
```

タイトルと説明を対話的に入力し、GitHub Issuesに自動的に送信します。

### VOC作成（要件）

```bash
openyida doctor --create-voc
```

### スマート送信

システムがBugか要件かを自動的に判断します：

```bash
openyida doctor --auto-submit
```

## 修復タイプ

| タイプ | 説明 | 例 |
|------|------|------|
| `AUTO` | 自動修復 | config.jsonの作成、破損したキャッシュの削除 |
| `COMMAND` | コマンド修復 | `pip install playwright`の実行 |
| `MANUAL` | 手動修復 | gh CLIのインストール、PRDの作成 |

## 健全性スコアの計算

健全性スコア（0-100）の計算式：

```
健全性 = 通過率 × 100 - エラー数 × 10 - 警告数 × 3
```

| スコア | ステータス |
|------|------|
| 90-100 | 優秀 🟢 |
| 70-89 | 良好 🟡 |
| 50-69 | 一般 🟠 |
| < 50 | 改善が必要 🔴 |
