OpenYidaは豊富なCLIコマンドを提供し、アプリの全ライフサイクル管理をサポートします。

---

## 環境と認証

```bash
# 環境検出
openyida env

# ログイン
openyida login [--qr]              # --qr はターミナルQRコードを使用
openyida login --wukong            # 悟空環境専用

# ログアウト
openyida logout

# ログイン状態管理
openyida auth status               # ログイン状態を確認
openyida auth login                # ログインを実行
openyida auth refresh              # ログイン状態を更新
openyida auth logout               # ログアウト

# 組織管理
openyida org list                  # アクセス可能な組織を一覧表示
openyida org switch --corp-id <id> # 組織を切り替え

# 作業ディレクトリの初期化
openyida copy [--force]            # projectディレクトリをコピー

# 診断ツール
openyida doctor [--fix]            # 環境診断
openyida doctor --monitor          # ヘルスモニタリング
openyida doctor --report <format>  # レポート生成 (json|markdown|html)
```

## アプリ管理

```bash
# アプリ作成
openyida create-app "<名称>" [desc] [icon] [color] [themeColor]

# アプリエクスポート
openyida export <appType> [output] # 移行パッケージを生成

# アプリインポート
openyida import <file> [name]      # 移行パッケージからアプリを再構築
```

## フォーム管理

```bash
# フォーム作成
openyida create-form create <appType> "<表单名>" <字段JSON> \
  [--layout <布局>] \
  [--theme <主题>] \
  [--label-align <对齐>]

# フォーム更新
openyida create-form update <appType> <formUuid> <修改JSON>

# フォームSchema取得
openyida get-schema <appType> <formUuid>

# フォーム設定更新
openyida update-form-config <appType> <formUuid> <isRenderNav> <title>
```

### サポートされているフィールドタイプ

`create-form` コマンドは以下のフィールドタイプのみ受け付けます。サポートされていないタイプを使用するとエラーが返されます。

#### 基本フィールド

組み込みフィールドで、プラグインのインストールは不要です。

| タイプ | 説明 |
|--------|------|
| `TextField` | 1行テキスト |
| `TextareaField` | 複数行テキスト |
| `NumberField` | 数値 |
| `RateField` | 評価 |
| `DateField` | 日付 |
| `RadioField` | ラジオボタン |
| `SelectField` | ドロップダウン（単一選択） |
| `CheckboxField` | チェックボックス |
| `MultiSelectField` | ドロップダウン（複数選択） |
| `CascadeSelectField` | カスケード選択 |
| `CascadeDateField` | カスケード日付 |
| `EmployeeField` | メンバー選択 |
| `DepartmentSelectField` | 部門選択 |
| `AssociationFormField` | 関連フォーム |
| `AssociationQuery` | 関連クエリ |
| `AttachmentField` | 添付ファイル |
| `ImageField` | 画像 |
| `AddressField` | 住所 |
| `CountrySelectField` | 国/地域選択 |
| `TableField` | サブフォーム/明細 |
| `SerialNumberField` | シリアル番号 |

#### 高度なフィールド

組み込みフィールドで、一部の機能はアクティベーションが必要な場合があります。

| タイプ | 説明 |
|--------|------|
| `EditorField` | リッチテキストエディタ |
| `DigitalSignatureField` | デジタル署名 |
| `LocationField` | GPS位置情報 |

#### プラグインフィールド

使用する前にプラグインセンターから対応するプラグインのインストールが必要です。

| タイプ | 必要なプラグイン |
|--------|------------------|
| `TimeZoneDateField` | 国際化タイムゾーンコンポーネント |
| `CC_AIButtonField` | AIコンポーネント |
| `CC_PG_ESignField` | スマート財務コンポーネント |
| `CC_TeambitionProjectLinksField` | Teambitionプラグイン |
| `CC_TeambitionTaskLinksField` | Teambitionプラグイン |
| `CC_MoneyField` | スマート財務コンポーネント |
| `CC_PhoneNumberField` | 拡張コンポーネント |
| `CC_ChineseIdField` | 拡張コンポーネント |
| `YidaDingtalkAgentField` | DingTalk関連プラグイン |

<Warning>
プラグインフィールドは、対応するプラグインがインストールされていない場合、作成に失敗します。これらのフィールドタイプを使用する前に、プラグインセンターから必要なプラグインをインストールしてください。
</Warning>

<Warning>
`MobileField`、`EmailField` などのフィールドタイプはサポートされておらず、使用すると拒否されます。上記の表に記載されているタイプのみを使用してください。
</Warning>

## ページ管理

```bash
# カスタムページ作成
openyida create-page <appType> "<ページ名>" [--datasource <jsonOrFile>]

# ページのコンパイルと公開
openyida publish <源文件路径> <appType> <formUuid>

# ページ共有設定
openyida save-share-config <appType> <formUuid> <url> <isOpen> [openAuth]
openyida get-page-config <appType> <formUuid>
openyida verify-short-url <appType> <formUuid> <url>
```

## データ管理

```bash
# 統一データ管理
openyida data <action> <resource> [args]

# フォームデータ
openyida data query form <appType> <formUuid> [--page N] [--size N] [--search-json JSON]
openyida data get form <appType> --inst-id <id>
openyida data create form <appType> <formUuid> --data <JSON>
openyida data update form <appType> --inst-id <id> --data <JSON>

# プロセスデータ
openyida data query process <appType> <formUuid> [--instance-status STATUS]
openyida data get process <appType> --process-inst-id <id>
openyida data create process <appType> <formUuid> --process-code <code> --data <JSON>

# タスク管理
openyida data query tasks <appType> --type <todo|done|submitted|cc>
openyida data execute task <appType> --task-id <id> --process-inst-id <id> --out-result <AGREE|DISAGREE>

# サブフォーム
openyida data query subform <appType> <formUuid> --inst-id <id> --table-field-id <fieldId>
```

## プロセス管理

```bash
# プロセスフォーム作成（新規フォーム + プロセス設定）
openyida create-process <appType> "<フォーム名>" <フィールドJSONファイル> <プロセス定義ファイル>

# プロセスフォーム作成（既存フォーム再利用 + プロセス設定）
openyida create-process <appType> --formUuid <formUuid> <プロセス定義ファイル>

# 定義ファイルからプロセスルールを設定
openyida configure-process <appType> <formUuid> <プロセス定義ファイル> [processCode]
```

## コネクタ管理

```bash
# コネクタ一覧
openyida connector list [--page N] [--size N]

# コネクタ作成
openyida connector create "<名称>" "<域名>" --operations <file> [选项]

# コネクタ詳細
openyida connector detail <connector-id>

# コネクタ削除
openyida connector delete <connector-id> [--force]

# アクション管理
openyida connector add-action --connector-id <id> --operations <file> [--confirm]
openyida connector list-actions <connector-id>
openyida connector delete-action <connector-id> <operation-id> [--force]
openyida connector test --connector-id <id> --action <actionId> [选项]

# 認証アカウント管理
openyida connector list-connections <connector-id>
openyida connector create-connection <connector-id> "<名称>" [选项]

# スマート作成
openyida connector smart-create --curl "<curl命令>" [--name <name>] [--desc <desc>]

# 補助ツール
openyida connector parse-api --curl "<curl命令>" --output <file>
openyida connector gen-template [输出路径]
```
