## 環境と認証

```bash
# 環境検出
openyida env

# ログイン
openyida login [--qr]              # --qr はターミナルQRコードを使用
openyida login --check-only        # ログイン状態の確認のみ（ログインは実行しない）

# ログアウト
openyida logout

# ログイン状態管理
openyida auth status               # ログイン状態を確認
openyida auth login                # ログインを実行
openyida auth refresh              # ログイン状態を更新
openyida auth logout               # ログアウト

# 組織管理
openyida org list                  # アクセス可能な組織を一覧表示
openyida org switch --corp-id <id> # 組織を切り替え

# 作業ディレクトリの初期化
openyida copy [--force]            # projectディレクトリをコピー（--forceで上書き）
openyida copy -skills              # yida-skillsをAIツールのskillsディレクトリにコピー
openyida copy -project [--force]   # projectディレクトリを明示的にコピー（デフォルトと同じ）

# 診断ツール
openyida doctor [--fix]            # 環境診断（--repairは--fixのエイリアス）
openyida doctor --production --app <appId>  # 本番アプリ診断
openyida doctor --monitor          # ヘルスモニタリング
openyida doctor --report <format>  # レポート生成 (json|markdown|html)
openyida doctor --create-ticket    # 診断結果からチケットを作成
openyida doctor --create-voc       # VOC（機能リクエスト）を作成
openyida doctor --auto-submit      # 自動判定してチケットまたはVOCを送信
```

## フォーム管理

```bash
# フォーム作成
openyida create-form create <appType> "<表单名>" <字段JSON> \
  [--layout <布局>] \
  [--theme <主题>] \
  [--label-align <对齐>]

# フォーム更新
openyida create-form update <appType> <formUuid> <修改JSON>

# フォームSchema取得
openyida get-schema <appType> <formUuid>  # フィールドIDを使用する操作の前に必ず実行

# フォーム設定更新
openyida update-form-config <appType> <formUuid> <isRenderNav> <title>
```


<Warning>
  YidaのフィールドIDはプラットフォームによりランダムに生成されます（例: `textField_eftt1aa5m`、`selectField_fix024y92`）。フィールド名からは推測できません。フィールドIDを使用する操作（データの作成・更新、クエリ条件の設定、カスタムページの作成など）の前に、**必ず** `openyida get-schema` を実行して実際のフィールドIDを取得してください。
</Warning>

### サポートされているフィールドタイプ

`create-form` コマンドは以下のフィールドタイプのみ受け付けます。サポートされていないタイプを使用するとエラーが返されます。

#### 基本フィールド

組み込みフィールドで、プラグインのインストールは不要です。

| タイプ | 説明 |
|--------|------|
| `TextField` | 1行テキスト |
| `TextareaField` | 複数行テキスト |
| `NumberField` | 数値 |
| `RateField` | 評価 |
| `DateField` | 日付 |
| `RadioField` | ラジオボタン |
| `SelectField` | ドロップダウン（単一選択） |
| `CheckboxField` | チェックボックス |
| `MultiSelectField` | ドロップダウン（複数選択） |
| `CascadeSelectField` | カスケード選択 |
| `CascadeDateField` | カスケード日付 |
| `EmployeeField` | メンバー選択 |
| `DepartmentSelectField` | 部門選択 |
| `AssociationFormField` | 関連フォーム |
| `AssociationQuery` | 関連クエリ |
| `AttachmentField` | 添付ファイル |
| `ImageField` | 画像 |
| `AddressField` | 住所 |
| `CountrySelectField` | 国/地域選択 |
| `TableField` | サブフォーム/明細 |
| `SerialNumberField` | シリアル番号 |

#### 高度なフィールド

組み込みフィールドで、一部の機能はアクティベーションが必要な場合があります。

| タイプ | 説明 |
|--------|------|
| `EditorField` | リッチテキストエディタ |
| `DigitalSignatureField` | デジタル署名 |
| `LocationField` | GPS位置情報 |

#### プラグインフィールド

使用する前にプラグインセンターから対応するプラグインのインストールが必要です。

| タイプ | 必要なプラグイン |
|--------|------------------|
| `TimeZoneDateField` | 国際化タイムゾーンコンポーネント |
| `CC_AIButtonField` | AIコンポーネント |
| `CC_PG_ESignField` | スマート財務コンポーネント |
| `CC_TeambitionProjectLinksField` | Teambitionプラグイン |
| `CC_TeambitionTaskLinksField` | Teambitionプラグイン |
| `CC_MoneyField` | スマート財務コンポーネント |
| `CC_PhoneNumberField` | 拡張コンポーネント |
| `CC_ChineseIdField` | 拡張コンポーネント |
| `YidaDingtalkAgentField` | DingTalk関連プラグイン |

<Warning>
プラグインフィールドは、対応するプラグインがインストールされていない場合、作成に失敗します。これらのフィールドタイプを使用する前に、プラグインセンターから必要なプラグインをインストールしてください。
</Warning>

<Warning>
`MobileField`、`EmailField` などのフィールドタイプはサポートされておらず、使用すると拒否されます。上記の表に記載されているタイプのみを使用してください。
</Warning>

## データ管理

<Warning>
  データの作成・更新前に、**必ず** `openyida get-schema` を実行して実際のフィールドIDを取得してください。データ書き込み後、`openyida data query form <appType> <formUuid> --size 1` で1件抽出し、`formData` のフィールドに実際の値があることを確認してください。すべてのフィールド値が空の場合、フィールドIDが正しくないため `get-schema` で再取得が必要です。
</Warning>

```bash
# 統一データ管理
openyida data <action> <resource> [args]

# フォームデータ
openyida data query form <appType> <formUuid> [--page N] [--size N] [--search-json JSON]
openyida data get form <appType> --inst-id <id>
openyida data create form <appType> <formUuid> --data-json <JSON>  # JSONキーはフィールドIDまたはフィールドラベル
openyida data update form <appType> --inst-id <id> --data-json <JSON>

# プロセスデータ
openyida data query process <appType> <formUuid> [--instance-status STATUS]
openyida data get process <appType> --process-inst-id <id>
openyida data create process <appType> <formUuid> --process-code <code> --data-json <JSON>

# タスク管理
openyida data query tasks <appType> --type <todo|done|submitted|cc>
openyida data execute task <appType> --task-id <id> --process-inst-id <id> --out-result <AGREE|DISAGREE>

# サブフォーム
openyida data query subform <appType> <formUuid> --inst-id <id> --table-field-id <fieldId>
```

## グローバルタスクセンター

`appType` を指定せずに、現在のユーザーのグローバルタスクセンターデータを照会します。

```bash
# 未処理タスク
openyida task-center todo [--page N] [--size N] [--keyword TEXT]

# 自分が作成した
openyida task-center created [--page N] [--size N] [--keyword TEXT] [--no-detail]

# 処理済み
openyida task-center done [--page N] [--size N] [--keyword TEXT]

# CC されたタスク
openyida task-center cc [--page N] [--size N] [--keyword TEXT]

# 代理提出
openyida task-center proxy [--page N] [--size N] [--keyword TEXT]
```

### タスクタイプ

| タイプ | 説明 | API エンドポイント |
|--------|------|-------------------|
| `todo` | 未処理タスク（対応が必要なもの） | `getTodoTasksInCorp` |
| `created` | 自分が作成した（自分が開始したプロセス） | `getMyCreateInCorp` |
| `done` | 処理済み（既に処理したもの） | `getDoneTasksInCorp` |
| `cc` | CC されたタスク | `getNotifyMeInCorp` |
| `proxy` | 代理提出（代理で提出したプロセス） | `getSubmitAgentInCorp` |

### オプションパラメータ

| パラメータ | 説明 | デフォルト |
|-----------|------|-----------|
| `--page N` | ページ番号 | `1` |
| `--size N` | 1ページあたりの件数（最大 100） | `20` |
| `--keyword TEXT` | 検索キーワード | — |
| `--no-detail` | 詳細を返さない（`created` タイプのみ対応） | — |

<Info>
  `openyida data query tasks` とは異なり、`task-center` コマンドはグローバルタスクセンターデータを照会し、`appType` パラメータは不要です。
</Info>

## フラッシュノートからPRD変換

AIを使用してDingTalkフラッシュノート（会議録音テキスト）を構造化されたPRDドキュメントに変換します。

```bash
# ファイルから読み取り
openyida flash-to-prd --file <ファイルパス> [--name <プロジェクト名>] [--max-tokens <数>]

# 標準入力から読み取り
cat meeting.txt | openyida flash-to-prd --name <プロジェクト名> [--max-tokens <数>]
```

### パラメータ

| パラメータ | 短縮形 | 説明 | デフォルト |
|------|------|------|------|
| `--file` | `-f` | フラッシュノートファイルのパス | — |
| `--name` | `-n` | プロジェクト名（省略時はコンテンツから自動検出） | — |
| `--max-tokens` | — | AI応答の最大トークン数 | `8000` |

このコマンドは指定されたファイルまたは標準入力からフラッシュノートの内容を読み取り、テキストの前処理（会議メタデータの抽出や発言者の識別を含む）を行い、Yida AIサービスを呼び出して構造化されたPRDを生成します。出力されたPRDファイルはプロジェクトの `prd/` ディレクトリに保存されます。

### 出力

コマンドはstdoutにJSON形式の結果を出力します：

```json
{
  "success": true,
  "projectName": "プロジェクト名",
  "prdFile": "/path/to/prd/プロジェクト名.md",
  "contentLength": 4096,
  "meetingRecognition": {
    "metaFields": 3,
    "a1Sections": 2,
    "speakers": 5
  }
}
```

<Info>
  このコマンドを実行する前にログインが必要です。まだ認証されていない場合は、先に `openyida login` を実行してください。
</Info>

## インテグレーションと自動化

フォームイベントでトリガーされ、通知、データ検索、データ作成などの自動アクションを実行するインテグレーションロジックフローを作成します。

```bash
openyida integration create <appType> <formUuid> "<flowName>" [options]
```

### 必須引数

| 引数 | 説明 |
|------|------|
| `<appType>` | アプリケーション ID |
| `<formUuid>` | トリガーフォーム UUID |
| `<flowName>` | インテグレーションフローの名前 |

### オプション

| オプション | 説明 | デフォルト |
|-----------|------|----------|
| `--process-code <code>` | 更新する既存のプロセスコード（省略で新規作成） | — |
| `--receivers <id1,id2>` | 通知受信者のユーザー ID（カンマ区切り） | — |
| `--title <text>` | 通知タイトル | フロー名 |
| `--content <text>` | 通知本文 | — |
| `--events <types>` | トリガーイベント（カンマ区切り）：`insert`、`update`、`delete` | `insert` |
| `--data-form-uuid <uuid>` | 「単一レコード取得」データノードのフォーム UUID | — |
| `--data-condition <mapping>` | データノードのフィールドマッピング（`bFieldId:bFieldName:aFieldId[:componentType]`）。複数回指定可能。 | — |
| `--add-data-form-uuid <uuid>` | 「レコード追加」データノードのフォーム UUID | — |
| `--add-data-assignment <mapping>` | データ追加ノードのフィールド割り当て（`targetFieldId:valueType:value`）。`valueType` は `processVar`、`literal`、`column` のいずれか。複数回指定可能。 | — |
| `--publish` | 作成後すぐにフローを公開 | ドラフトのみ |

### 例

```bash
# 新しい送信でトリガーされる通知フローを作成
openyida integration create APP_XXX FORM-YYY "New Order Alert" \
  --receivers "user1,user2" \
  --events insert \
  --title "New Order" \
  --content "A new order was submitted"

# 作成してすぐに公開
openyida integration create APP_XXX FORM-YYY "Auto Notify" \
  --receivers "user1" \
  --publish
```

## DingTalk CLI (dws)

連絡先、カレンダー、ToDo、承認、勤怠などの DingTalk Workspace CLI へのプロキシコマンド。

```bash
openyida dws <command> [args]
```

<Info>
  `dws` コマンドには DingTalk CLI の別途インストールが必要です。インストールされていない場合、OpenYida は初回使用時にインストールを促します。
</Info>

### よく使うコマンド

| コマンド | 説明 |
|---------|------|
| `contact user search --keyword "<name>"` | 連絡先を検索 |
| `calendar event list` | カレンダーイベントを一覧表示 |
| `todo task create --title "<title>" --executors "<userId>"` | ToDo タスクを作成 |
| `approval instance list` | 承認インスタンスを一覧表示 |
| `attendance record list` | 勤怠記録を照会 |
| `chat robot send --content "<message>"` | チャットロボットメッセージを送信 |

### 特殊コマンド

```bash
# DingTalk CLI をインストール
openyida dws install

# ヘルプを表示
openyida dws help
```

### 出力形式

デフォルトでは、出力は人間が読みやすいテーブル形式でフォーマットされます。AI エージェントに適した JSON 出力には `-f json` を追加してください。

```bash
openyida dws contact user search --keyword "name" -f json
```

## 会話のエクスポート

現在の AI ツール環境から AI 会話記録を Markdown ドキュメントとしてエクスポートします。

```bash
openyida export-conversation [options]
```

### オプション

| オプション | エイリアス | 説明 | デフォルト |
|-----------|----------|------|----------|
| `--output` | `-o` | 出力ファイルパス | 自動生成パス |
| `--input` | `-i` | 入力会話ファイル（JSON/JSONL） | — |
| `--latest` | — | 最新の会話のみエクスポート | 有効 |
| `--list` | — | 利用可能な会話記録を一覧表示 | — |

### 例

```bash
# 現在の会話をエクスポート
openyida export-conversation

# 特定のファイルにエクスポート
openyida export-conversation -o output.md

# 利用可能な会話を一覧表示
openyida export-conversation --list

# 特定の入力ファイルからエクスポート
openyida export-conversation -i conversation.json -o result.md
```
