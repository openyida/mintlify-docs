OpenYidaは多様なビジネスシーンに適用でき、以下は一般的なアプリケーションタイプの例です。

---

## 🏢 エンタープライズビジネスシステム

複数モジュールのビジネスプラットフォームを構築し、複雑なフォーム、ワークフロー、データ関連付けをサポートします。

<AccordionGroup>
  <Accordion title="IPD（統合製品開発）">
    **プロンプト：**
    ```
    Yidaを使用してIPDシステムを作成し、チップ生産の全プロセスを管理してください。
    プロジェクト立ち上げ、設計レビュー、テープアウト、量産などの段階を含みます
    ```
    **出力：** 複数フォームアプリ。段階追跡、レビューチェックリスト、マイルストーン管理、チーム間の可視性を含みます。
  </Accordion>

  <Accordion title="CRM（顧客関係管理）">
    **プロンプト：**
    ```
    CRMを構築してください。リード、顧客、商談、フォローアップ記録などのモジュールを含みます
    ```
    **出力：** 関連フォーム：リード → 顧客 → 商談 → 契約。ステータス追跡と割り当てフィールドを含みます。
  </Accordion>

  <Accordion title="人事と休暇管理">
    **プロンプト：**
    ```
    人事休暇管理システムを作成してください。休暇申請、承認プロセス、休暇残高統計を含みます
    ```
    **出力：** 承認プロセスを含む休暇申請フォーム、残高計算機、マネージャーダッシュボード。
  </Accordion>

  <Accordion title="プロジェクト管理">
    **プロンプト：**
    ```
    プロジェクト管理アプリを構築してください。タスク、マイルストーン、チームメンバー、進行状況追跡を含みます
    ```
    **出力：** 優先度/ステータス/担当者フィールドを含むタスクリスト、マイルストーントラッカー、ガントチャートビュー。
  </Accordion>
</AccordionGroup>

## 💰 実用ツールと計算機

単一用途の計算またはデータ収集ツール。

<AccordionGroup>
  <Accordion title="個人給与計算機">
    **プロンプト：**
    ```
    個人給与計算機アプリを構築してください。税込給与を入力すると、所得税と手取り給与が自動計算されます
    ```
    **出力：** 税引前控除、所得税計算、手取り集計の数式フィールドを含むフォーム。
  </Accordion>

  <Accordion title="予算トラッカー">
    **プロンプト：**
    ```
    個人予算トラッカーを構築してください。収入、支出分類、月次集計を記録します
    ```
    **出力：** 分類ドロップダウンと自動計算される月次残高を含む収入/支出入力フォーム。
  </Accordion>
</AccordionGroup>

## 🔌 API統合（コネクタ）

HTTPコネクタを通じてYidaアプリと外部システムを接続します。

<AccordionGroup>
  <Accordion title="Webhook通知">
    **プロンプト：**
    ```
    DingTalk webhookのHTTPコネクタを作成し、フォーム提交通知を送信するPOSTアクションを追加してください
    ```
    **出力：** テスト機能を含む設定済みコネクタ。フォームイベント時にトリガーできます。
  </Accordion>

  <Accordion title="curlからスマート作成">
    **プロンプト：**
    ```
    このcurlコマンドからスマートにコネクタを作成してください：curl -X POST https://api.example.com/v1/users -H "Authorization: Bearer TOKEN" -d '{"name":"test"}'
    ```
    **出力：** 自動設定されたコネクタ。ベースURL、リクエストヘッダー、アクションパラメータを抽出します。
  </Accordion>
</AccordionGroup>

---

<Note>
これらは出発点です。出力はYidaプラットフォームで完全に編集可能なアプリ（ブラックボックス構成ではない）であるため、いつでもYidaビジュアルエディターまたはAIへの後続指示を通じて生成されたアプリを最適化できます。
</Note>
