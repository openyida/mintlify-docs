## 1. 詳細なプロンプトを作成する

生成アプリの品質はプロンプトの詳細度に比例します。比較：

<CodeGroup>

```text 曖昧（避ける）
CRM を作成する
```

```text 詳細（推奨）
3 つの関連フォームを含む CRM を作成する：
1. リードフォーム — フィールド：名前、会社、電話、メール、ソース（ドロップダウン：紹介/ウェブサイト/イベント/コールドコール）、ステータス（ドロップダウン：新規リード/連絡済み/適格/失格）、担当者（メンバーセレクター）
2. 顧客フォーム — リードから転換、フィールド：会社名、業界、年間収入、主要連絡先、アカウントマネージャー
3. 機会フォーム — 顧客にリンク、フィールド：機会名、金額、ステージ（ドロップダウン：提案/交渉/受注/失注）、予想クローズ日、備考
```

</CodeGroup>

## 2. 要件ドキュメントを使用する

複雑なアプリはまず PRD を作成し、AI に渡す：

```
以下の要件ドキュメントに基づいて宜搭アプリを生成する：

[PRD を貼り付け — フィールド名、タイプ、検証ルール、ワークフローなど]
```

このアプローチは反復プロンプトよりも正確な結果を生み出します。

---

## 3. 増分反復

1 つのプロンプトですべてを構築しようとしないでください。まずコア構造を構築し、徐々に複雑さを追加します：

```
ステップ 1：基本的な従業員情報フォームを作成する
ステップ 2：従業員にリンクされた休暇申請フォームを追加する
ステップ 3：休暇承認ワークフローを設定する
ステップ 4：保留中の承認を表示するマネージャーダッシュボードを追加する
ステップ 5：ダッシュボードを公開ページとして公開する
```

---

## 4. 各ステップを検証する

AI が各コンポーネントを作成した後、検証させます：

```
[フォーム名] フォームの現在の Schema を表示する
[ページ名] ページが公開されてアクセス可能か確認する
```

早期に問題を発見し、大きな問題に累積するのを防ぎます。

---

## 5. プロジェクト作業ディレクトリを再利用する

プロジェクトごとに `openyida copy` を 1 回だけ実行し、セッションごとに実行しないでください。`project/config.json` ファイルは `appId` と `corpId` を保存します — AI はこれを使用してターゲットアプリを認識します。プロジェクト途中でこのファイルを削除しないでください。

---

## 6. 環境固有のログイン

異なる AI ツールは異なるログイン方法を必要とします：

| AI ツール | ログインコマンド |
|---------|----------|
| Claude Code、Cursor、VSCode | `openyida login` |
| 悟空 | `openyida login --wukong` |
| OpenCode、Aone Copilot | `openyida login` |

ツールを切り替えるときは、まず `openyida env` を実行してアクティブ環境を確認します。

---

## 7. カスタムページ画像に CDN を使用する

カスタムページに画像が含まれる場合、信頼性の高いレンダリングを確保するためにまず CDN にアップロードします：

```bash
openyida cdn-config   # Alibaba Cloud OSS + CDN を設定（1 回限り）
openyida cdn-upload ./my-image.png   # 安定した CDN URL を取得
```

その後、ページプロンプトで CDN URL を参照します：
```
この CDN URL を使用してランディングページにヒーローバナー画像を追加する：https://cdn.example.com/banner.png
```

---

## 8. 複数組織設定を処理する

複数の宜搭組織にアクセスできる場合：

```bash
openyida org list          # 利用可能なすべての組織を表示
openyida org switch --corp-id <id>   # ターゲット組織に切り替え
openyida env               # 切り替えを確認
```

アプリを作成する前に常にアクティブ組織を検証し、間違ったスペースで構築するのを防ぎます。

---

## 9. デバッグ前に Doctor を実行する

失敗したコマンドのデバッグに時間を費やす前に、実行します：

```bash
openyida doctor --fix
```

最も一般的な問題を自動的に検出して解決します：期限切れセッション、設定ファイルの欠落、環境の不一致。

---

## 10. AI 支援 PR を歓迎

OpenYida 自体に貢献する場合、AI 生成コードを完全に受け入れます。PR 説明に使用した AI ツールを記載します — メンテナーが生成コンテキストを理解し、より効率的にレビューするのに役立ちます。
