# 아키텍처 개요

OpenYida는 Node.js 기반 CLI 도구로 계층형 아키텍처 설계를 채택하여 AI 기능, Yida 플랫폼 통합, 개발자 경험을 유기적으로 결합합니다.

## 전체 워크플로우

![OpenYida 워크플로우](/images/architecture-workflow.png)

OpenYida의 워크플로우는 6단계로 나뉩니다：

1. **User Requirement** - 사용자가 자연어로 요구사항을 설명
2. **AI Understanding** - AI가 사용자 요구사항을 이해하고 분석
3. **PRD Document** - 제품 요구사항 문서 생성
4. **CLI Execution** - CLI 명령을 실행하여 앱 생성
5. **YiDa Platform** - Yida 플랫폼에서 앱 구축
6. **Online Access** - 앱 온라인 액세스

## 인증 흐름

![인증 흐름](/images/architecture-auth.png)

OpenYida의 인증 흐름：

1. **config.json** - 구성 정보 저장
2. **Login Session** - 로그인 세션 설정
3. **Cookie Cache** - 인증 정보 캐시
4. **API Request** - API 요청 전송
5. **YiDa Backend** - Yida 백엔드 처리
6. **Page/Form Output** - 페이지/양식 출력

## CLI 아키텍처

![CLI 아키텍처](/images/architecture-cli.png)

OpenYida CLI와 Yida 플랫폼의 상호 작용：

- **AI Coding Tool** → **OpenYida CLI**
  - Login（로그인）
  - Create App（앱 생성）
  - Create Page（페이지 생성）
  - Publish（게시）
  - Get Schema（구조 가져오기）
- **YiDa Low-Code Platform**
  - App Management（앱 관리）
  - Form Pages（양식 페이지）
  - Custom Pages（사용자 정의 페이지）

## 스킬 시스템

![스킬 시스템](/images/architecture-skills.png)

OpenYida의 스킬은 3가지 카테고리로 분류됩니다：

### Login Skills（로그인 스킬）
- `login` - Yida 플랫폼에 로그인
- `logout` - 로그아웃

### Create Skills（생성 스킬）
- `create-app` - 앱 생성
- `create-page` - 페이지 생성
- `create-form` - 양식 생성
- `get-schema` - 데이터 구조 가져오기

### Publish Skills（게시 스킬）
- `publish` - 앱 게시

## 핵심 설계 원칙

### 1. 모듈 설계

각 기능 모듈은 독립적으로 캡슐화되고 명확한 인터페이스를 통해 상호 작용합니다：

- **명령 계층**: CLI 입력을 처리하고 코어 서비스 호출
- **서비스 계층**: 비즈니스 로직을 캡슐화하고 통일 API 제공
- **어댑터 계층**: 외부 시스템（AI, Yida 플랫폼）과 상호 작용

### 2. 플러그인 확장

스킬 시스템은 동적 로딩 및 확장을 지원합니다：

```typescript
// 스킬 인터페이스 정의
interface Skill {
  name: string;
  version: string;
  activate(context: SkillContext): void;
  deactivate(): void;
}
```

### 3. 구성 기반

다중 환경 구성을 지원하고 유연하게 전환 가능：

- 로컬 전역 구성（~/.openyida/）
- 프로젝트 수준 구성（.openyidarc.json）
- 환경 변수로 덮어쓰기

### 4. 타입 안전성

전체 체인 TypeScript 타입 지원, 런타임 오류 감소.

## 데이터 흐름

### 앱 생성 흐름

```
User Input → AI Service → Code Generation → Yida API → Application
                ↑                              ↓
           PRD Analysis                  Validation
                ↑                              ↓
           Template Selection           Deployment
```

### 데이터 작업 흐름

```
User Command → Config Load → Auth Check → Yida API → Data Response
                                              ↓
                                         Cache/Store
```

## 기술 스택

| 레벨 | 기술 |
|------|------|
| 런타임 | Node.js 18+ |
| 언어 | TypeScript 5.x |
| CLI 프레임워크 | Commander.js |
| HTTP 클라이언트 | Axios |
| AI SDK | OpenAI SDK, Anthropic SDK |
| 구성 관리 | Cosmiconfig |
| 로그 | Winston |
| 테스트 | Jest |

## 보안 설계

### 인증 보안

- OAuth 2.0 표준 흐름
- 토큰 자동 갱신 메커니즘
- 민감 정보 로컬 암호화 저장

### 데이터 전송

- 전체 체인 HTTPS
- API 서명 검증
- 요청 재시도 및 시간 초과 제어

## 성능 최적화

- **연결 풀**: HTTP 연결 재사용
- **스마트 캐시**: 구성 및 메타데이터 로컬 캐시
- **증분 업데이트**: 변경 내용만 전송
- **동시 제어**: 동시 API 호출 제한

## 확장 지점

개발자는 다음 레벨에서 확장할 수 있습니다：

1. **새 명령**: 사용자 정의 CLI 명령 추가
2. **새 스킬**: AI 기능 확장
3. **새 템플릿**: 사용자 정의 코드 생성 템플릿
4. **새 AI Provider**: 다른 대규모 모델 연결
