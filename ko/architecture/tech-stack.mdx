## 기술 스택 아키텍처

![기술 스택 아키텍처](/images/architecture-tech-stack.png)

OpenYida의 기술 스택은 개발 환경과 클라우드 플랫폼의 두 부분으로 나뉩니다：

### 개발 환경 (Development Environment)
- **Node.js** - 런타임 환경
- **Python** - Playwright 자동화 테스트
- **Git** - 버전 관리

### 자동화 테스트 (Playwright)
- 브라우저 자동화
- 엔드 투 엔드 테스트
- 페이지 스크린샷 비교

### Yida 클라우드 플랫폼 (YiDa Cloud Platform)
- **App** - 앱 관리
- **Form** - 양식 페이지
- **Custom Page** - 사용자 정의 페이지

## 핵심 모듈 상세

### 국제화 (i18n.js)

12개 언어의 국제화 시스템을 지원합니다：

```javascript
const { t, getLanguage, setLanguage } = require('./i18n');

// 번역 사용
console.log(t('login.success'));
console.log(t('create_app.usage', 'appName'));

// 언어 가져오기/설정
console.log(getLanguage()); // 'zh'
setLanguage('en');
```

**언어 감지 우선순위：**
1. `OPENYIDA_LANG` 환경 변수
2. `LANG` / `LC_ALL` 시스템 환경 변수
3. 기본값：간체 중국어

### 환경 감지 (env.js)

현재 실행 중인 AI 도구 환경을 자동으로 감지합니다：

| 감지 방법 | 도구 |
|----------|------|
| 디렉터리 존재 감지 | Claude Code, Cursor, OpenCode, Aone Copilot, Qoder |
| 특수 파일 감지 | Wukong |

### 진단 엔진 (doctor.js)

3계층 아키텍처 설계：

1. **DiagnosticEngine** - 진단 스케줄러
2. **EnvironmentChecker** - 환경 체커
3. **ApplicationChecker** - 애플리케이션 체커
4. **FixEngine** - 스마트 복구 엔진

## 기능 모듈 상세

### 인증 모듈 (auth/)

```
auth/
├── login.js          # Yida 로그인（Cookie 캐시 + QR 코드）
├── auth.js           # 로그인 상태 관리
├── org.js            # 조직 관리
└── qr-login.js       # 터미널 QR 코드 로그인
```

**로그인 흐름：**
1. 현재 AI 도구 환경 감지
2. 환경에 따라 로그인 방식 선택（QR 코드 / CDP / 파일 읽기）
3. Cookie를 가져와 로컬에서 암호화하여 캐시
4. 후속 명령에서 캐시된 로그인 상태를 자동으로 사용

### 커넥터 모듈 (connector/)

```
connector/
├── api.js                    # Yida 커넥터 API 캡슐화
├── curl-parser.js            # curl 명령 파서
├── action-generator.js       # 작업 생성기
├── desc-generator.js         # 설명 생성기
├── connector-smart-create.js # 스마트 생성 진입점
└── ...
```

**스마트 생성 흐름：**
```
curl 명령 → 파서 → API 정보 추출 → 인증 감지 → 기존 커넥터 일치 → 구성 생성 → 생성/업데이트
```

### 데이터 관리 모듈 (data-management.js)

통합 데이터 작업 인터페이스, 다음을 지원합니다：

| 리소스 유형 | 작업 |
|----------|------|
| form | query, get, create, update |
| process | query, get, create |
| task | query, execute |
| subform | query |

## Yida API 호출

### 기본 설정

```javascript
const BASE_URL = 'https://www.aliwork.com';

// 요청 헤더
const headers = {
  'Content-Type': 'application/json',
  'X-Csrf-Token': csrfToken,
  'Cookie': cookies,
};
```

### 핵심 API

| 엔드포인트 | 용도 |
|------|------|
| `/dingtalk/web/APP_xxx/xxx` | 앱 내 인터페이스 |
| `/dingtalk/web/APP_xxx/form/getFormDataList` | 양식 데이터 쿼리 |
| `/dingtalk/web/APP_xxx/form/saveFormData` | 양식 데이터 저장 |
| `/dingtalk/web/APP_xxx/proc/getProcessInstances` | 프로세스 인스턴스 쿼리 |

## 보안 설계

1. **Cookie 암호화 저장** - 로컬 캐시 암호화 저장
2. **CSRF Token 검증** - 각 요청에서 Token 휴대
3. **하드코딩된 자격 증명 없음** - 모든 자격 증명은 동적으로 가져옴
4. **환경 격리** - 다른 AI 도구 환경은 독립적

---

## 성능 최적화

1. **지연 로드 번역 파일** - 언어 팩을 온디맨드로 로드
2. **Schema 로컬 캐시** - API 호출 감소
3. **병렬 요청 제어** - 과도한 요청으로 인한 제한 방지
4. **증분 업데이트** - 양식 업데이트는 덮어쓰기가 아닌 병합 채택

## 확장 메커니즘

### 새 명령 추가

1. `lib/`에서 `new-feature.js` 생성
2. `bin/yida.js`에서 명령 라우트 등록
3. `README.md` 및 문서 업데이트
4. 국제화 키 추가

### 새 언어 추가

1. `lib/core/locales/`에서 `xx.js` 생성
2. 모든 키 번역（`zh.js` 기준）
3. `i18n.js`의 `SUPPORTED_LANGUAGES` 업데이트
4. `npm run validate-i18n` 실행하여 완전성 확인
