## 기여 방법

1. **버그 보고** → 재현 단계와 환경 정보를 포함하여 Issue 열기
2. **기능 제안** → 먼저 Discussion 또는 Issue 를 시작한 후 구현
3. **문서 개선** → 문서 관련 PR 은 항상 환영
4. **스킬 추가** → `yida-skills/` 아래에서 확장
5. **버그 수정 / 새 기능** → 아래 개발 워크플로우 따르기

## 개발 환경 설정

```bash
# 1. 저장소 Fork 및 클론
git clone git@github.com:your-username/openyida.git
cd openyida

# 2. 의존성 설치
npm install

# 3. 로컬 디버깅 위한 전역 링크
npm link

# 4. 테스트npm test
```

---

## PR 체크리스트

풀 리퀘스트를 제출하기 전에 확인:

- [ ] 실제 Yida 계정으로 로컬에서 관련 기능 테스트 완료
- [ ] 모든 테스트 통과: `npm test`
- [ ] JS 구문 검사 통과:
  ```bash
  node --check bin/yida.js && for f in lib/*.js lib/locales/*.js; do node --check "$f"; done
  ```
- [ ] PR 설명에 변경 사항과 이유가 명확히 설명됨
- [ ] UI/동작 변경 사항이 있는 경우샷 녹화 첨부

---

## PR 가이드라인

- **하나의 PR, 하나의 일** — 관련 없는 변경 사항을 섞지 않음
- **PR 제목 형식:** `feat: add xxx` / `fix: fix xxx` / `docs: update xxx`
- **설명** 에 포함: 무엇, 왜, 테스트 방법
- PR 이 Issue 를 닫는 경우, 설명에 `Closes #123` 추가
