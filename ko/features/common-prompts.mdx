## 양식 및 페이지 작업

### 양식 페이지 생성

```
xxx 양식 페이지를 만드세요
```
```
휴가 신청 양식을 만드세요. 포함할 필드：직원 이름, 부서, 시작일, 종료일, 휴가 사유
```

### 양식 필드 추가

```
[페이지 이름]에 [필드 이름] 필드를 추가하세요. 필드 유형：[TextField/NumberField/DateField/SelectField/...]
```
```
직원 정보 양식에 "부서" 드롭다운 필드를 추가하세요. 옵션：엔지니어링 부서, 마케팅 부서, 영업 부서, 인사 부서
```

<Note>
[지원되는 필드 유형](/ko/features/skills#지원되는-필드-유형)만 허용됩니다. 지원되지 않는 유형(예: `MobileField`, `EmailField`)은 거부됩니다.
</Note>

### 필드 속성 변경

```
[페이지 이름]의 [필드 이름] 필드를 필수로 설정하세요
```
```
[페이지 이름]의 [필드 이름] 필드를 읽기 전용으로 설정하세요
```
```
[페이지 이름]의 [필드 이름] 필드 라벨을 [새 라벨]로 변경하세요
```

### 사용자 정의 표시 페이지

```
```


## 통합 및 자동화

### 통합 플로우 생성

```
새 레코드가 제출될 때 알림을 복내는 통합 플로우를 생성하세요
```

```
[양식 이름]에서 새 제출 시 [팀/사용자]에게 알림을 복내는 자동화를 설정하세요
```

```
[양식 이름] 워크플로우를 생성하여 [수신자]에게 이메일 알림을 복내세요
```

### 자동화 액션 추가

```
[플로우 이름] 통합에 데이터 조회 액션을 추가하세요
```

```
레코드 생성 액션을 추가하여 [대상 양식]에 자동으로 항목을 생성하세요
```

<Note>
통합 플로우는 양식 이벤트(삽입, 업데이트, 삭제)에서 트리거되며 알림, 데이터 조회 및 레코드 생성을 실행할 수 있습니다. CLI를 사용하여 플로우를 생성하고 구성하세요.
</Note>
