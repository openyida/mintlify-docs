## 스마트 커넥터 생성

curl 명령에서 원클릭으로 커넥터 생성：

```bash
openyida connector smart-create \
  --curl "curl 'https://api.example.com/v1/users' -H 'Authorization: Bearer xxx'" \
  --name "사용자 관리 커넥터" \
  --desc "사용자 데이터를 관리하기 위한 HTTP 커넥터"
```

스마트 생성 흐름：

1. **인터페이스 정보 구문 분석** - 프로토콜, Host, 메서드, 경로, 인증 유형 추출
2. **기존 커넥터와 일치** - 동일한 Host와 인증의 기존 커넥터 검색
3. **커넥터 구성 생성** - OpenAPI 형식의 커넥터 정의 자동 구축
4. **생성 또는 업데이트** - 새 커넥터 생성 또는 기존 커넥터에 작업 추가

## 커넥터 관리 명령

### 커넥터 나열

```bash
openyida connector list
openyida connector list --page 1 --size 20
```

### 커넥터 생성

```bash
openyida connector create "커넥터 이름" "api.example.com" \
  --operations ./operations.json \
  --protocol https \
  --desc "커넥터 설명"
```

<Note>
operations JSON 파일의 `method` 필드는 자동으로 소문자로 정규화됩니다 (예: `"POST"`는 `"post"`로 변환). `method`를 지정하지 않으면 기본값은 `"get"`입니다.
</Note>

### 커넥터 세부 정보

```bash
openyida connector detail <connector-id>
```

### 커넥터 삭제

```bash
openyida connector delete <connector-id> --force
```

## 작업 관리

### 실행 작업 추가

```bash
openyida connector add-action \
  --connector-id <id> \
  --operations ./new-action.json \
  --confirm
```

<Note>
operations 설정의 `method` 필드는 자동으로 소문자로 정규화됩니다 (예: `"POST"`는 `"post"`로 변환). 대소문자 구분 없이 지정할 수 있습니다. 미지정 시 기본값은 `"get"`입니다.
</Note>

### 기존 작업 나열

```bash
openyida connector list-actions <connector-id>
```

### 작업 테스트

```bash
openyida connector test \
  --connector-id <id> \
  --action <actionId> \
  --body '{"key": "value"}'
```

### 작업 삭제

```bash
openyida connector delete-action <connector-id> <operation-id> --force
```

## 인증 계정 관리

### 인증 계정 나열

```bash
openyida connector list-connections <connector-id>
```

### 인증 계정 생성

```bash
openyida connector create-connection <connector-id> "프로덕션 환경 계정" \
  --token "Bearer xxx" \
  --type bearer
```


## 모범 사례

### 1. 환경 변수를 사용하여 민감 정보 관리

```bash
export API_TOKEN="your-secret-token"
openyida connector smart-create --curl "curl -H 'Authorization: Bearer $API_TOKEN' ..."
```

### 2. 복잡한 커넥터를 단계적으로 생성

```bash
# 1단계：기본 커넥터 생성
openyida connector create "DingTalk API" "oapi.dingtalk.com"

# 2단계：작업 순차 추가
openyida connector add-action --connector-id <id> --operations ./action1.json
openyida connector add-action --connector-id <id> --operations ./action2.json
```

### 3. 테스트 후 활성화

```bash
# 생성만 하고 확인
openyida connector add-action --connector-id <id> --operations ./action.json

# 작업 테스트
openyida connector test --connector-id <id> --action <actionId>

# Yida 백엔드에서 최종 확인
```

## 자주 묻는 질문

<AccordionGroup>
  <Accordion title="curl 명령 구문 분석에 실패하면 어떻게 하나요?">
    curl 명령 형식이 올바르고 완전한 URL과 필요한 헤더가 포함되어 있는지 확인하세요. 특수 문자가 포함된 경우 따옴표로 묶는 것을 권장합니다.
  </Accordion>

  <Accordion title="기존 커넥터의 작업을 업데이트하려면 어떻게 하나요?">
    `connector add-action` 명령을 사용하세요. 작업 ID가 이미 존재하는 경우 시스템에서 업데이트할지 묻습니다.
  </Accordion>

  <Accordion title="어떤 인증 방식이 지원되나요?">
    현재 Bearer Token, API Key, Basic Auth, OAuth2를 지원합니다. curl 명령 내의 인증 정보는 자동으로 인식됩니다.
  </Accordion>
</AccordionGroup>
