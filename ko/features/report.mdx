## 보고서 생성

### 기본 사용법

```bash
openyida create-report <appType> "보고서 이름" '<차트-정의-JSON-또는-파일>'
```

세 번째 인수는 인라인 JSON 문자열 또는 JSON 파일 경로일 수 있습니다.

### 예: 판매 보고서 생성

```bash
openyida create-report APP_XXXXXX "Sales Dashboard" '{
  "charts": [
    {
      "type": "bar",
      "title": "Monthly Sales",
      "dataSource": "sales_form",
      "xField": "month",
      "yField": "amount"
    }
  ]
}'
```

### 파일에서 차트 정의 로드

```bash
openyida create-report APP_XXXXXX "Annual Report" ./charts-config.json
```

## 보고서에 차트 추가

```bash
openyida append-chart <appType> <reportId> '<차트-정의-JSON-또는-파일>'
```

### 예

```bash
openyida append-chart APP_XXXXXX REPORT_XXXXX '{
  "type": "line",
  "title": "Growth Trend",
  "dataSource": "form_sales",
  "xField": "month",
  "yField": "growth_rate"
}'
```

---

## 차트 유형

| 유형 | 설명 | 적합한 용도 |
|------|-------------|----------|
| **bar** | 막대형 차트 | 카테고리 비교 |
| **line** | 꺾은선형 차트 | 시간에 따른 추세 |
| **pie** | 원형 차트 | 부분 대 전체 관계 |
| **table** | 데이터 테이블 | 상세 데이터 보기 |
| **area** | 영역형 차트 | 누적 추세 |
| **scatter** | 산점도 | 상관 분석 |

---

## 차트 정의 형식

### 막대형 차트

```json
{
  "type": "bar",
  "title": "Sales by Region",
  "dataSource": "form_uuid",
  "xField": "region",
  "yField": "amount",
  "aggregation": "sum"
}
```

### 꺾은선형 차트

```json
{
  "type": "line",
  "title": "Traffic Trend",
  "dataSource": "form_uuid",
  "xField": "date",
  "yField": "visitors",
  "aggregation": "sum"
}
```

### 원형 차트

```json
{
  "type": "pie",
  "title": "Market Share",
  "dataSource": "form_uuid",
  "categoryField": "product",
  "valueField": "share"
}
```

### 데이터 테이블

```json
{
  "type": "table",
  "title": "Detail View",
  "dataSource": "form_uuid",
  "columns": [
    { "field": "name", "title": "Name" },
    { "field": "amount", "title": "Amount" },
    { "field": "date", "title": "Date" }
  ]
}
```

---

## 집계 함수

| 함수 | 설명 |
|----------|-------------|
| **sum** | 값의 합계 |
| **avg** | 평균 값 |
| **count** | 레코드 수 |
| **max** | 최대 값 |
| **min** | 최소 값 |

## 완전한 설정 예

### sales-report-charts.json

```json
{
  "charts": [
    {
      "type": "bar",
      "title": "Monthly Sales",
      "dataSource": "form_monthly_sales",
      "xField": "month",
      "yField": "sales_amount",
      "aggregation": "sum"
    },
    {
      "type": "pie",
      "title": "Product Categories",
      "dataSource": "form_sales",
      "categoryField": "product_category",
      "valueField": "sales_amount"
    },
    {
      "type": "line",
      "title": "Sales Trend",
      "dataSource": "form_daily_sales",
      "xField": "date",
      "yField": "sales_amount",
      "aggregation": "sum"
    },
    {
      "type": "table",
      "title": "Sales Detail",
      "dataSource": "form_sales",
      "columns": [
        { "field": "order_no", "title": "Order No.", "width": 150 },
        { "field": "customer_name", "title": "Customer", "width": 200 },
        { "field": "product_name", "title": "Product", "width": 200 },
        { "field": "amount", "title": "Amount", "width": 120 },
        { "field": "sales_date", "title": "Date", "width": 150 }
      ],
      "pageSize": 10
    }
  ]
}
```

보고서 생성:

```bash
openyida create-report APP_XXXXXX "Sales Analysis Report" ./sales-report-charts.json
```

## 모범 사례

1. **적절한 차트 유형 선택** - 추세에는 꺾은선형, 비교에는 막대형, 비율에는 원형 차트 사용
2. **차트 수 제한** - 보고서당 6개 이하의 차트 유지
3. **데이터 테이블 포함** - 상세 데이터 검사를 위해 차트와 테이블을 함께 사용
4. **적절한 페이지 크기 설정** - 데이터 테이블은 페이지당 10-20행 사용
