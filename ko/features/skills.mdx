OpenYida는 풍부한 CLI 명령을 제공하며 앱의 전체 수명 주기 관리를 지원합니다.

---

## 환경 및 인증

```bash
# 환경 감지
openyida env

# 로그인
openyida login [--qr]              # --qr은 터미널 QR 코드 사용
openyida login --wukong            # Wukong 환경 전용

# 로그아웃
openyida logout

# 로그인 상태 관리
openyida auth status               # 로그인 상태 확인
openyida auth login                # 로그인 실행
openyida auth refresh              # 로그인 상태 갱신
openyida auth logout               # 로그아웃

# 조직 관리
openyida org list                  # 액세스 가능한 조직 나열
openyida org switch --corp-id <id> # 조직 전환

# 작업 디렉터리 초기화
openyida copy [--force]            # project 디렉터리 복사

# 진단 도구
openyida doctor [--fix]            # 환경 진단
openyida doctor --monitor          # 헬스 모니터링
openyida doctor --report <format>  # 보고서 생성 (json|markdown|html)
```

## 앱 관리

```bash
# 앱 생성
openyida create-app "<名称>" [desc] [icon] [color] [themeColor]

# 앱 내보내기
openyida export <appType> [output] # 마이그레이션 패키지 생성

# 앱 가져오기
openyida import <file> [name]      # 마이그레이션 패키지에서 앱 재구축
```

## 양식 관리

```bash
# 양식 생성
openyida create-form create <appType> "<表单名>" <字段JSON> \
  [--layout <布局>] \
  [--theme <主题>] \
  [--label-align <对齐>]

# 양식 업데이트
openyida create-form update <appType> <formUuid> <修改JSON>

# 양식 Schema 가져오기
openyida get-schema <appType> <formUuid>

# 양식 구성 업데이트
openyida update-form-config <appType> <formUuid> <isRenderNav> <title>
```

### 지원되는 필드 유형

`create-form` 명령은 다음 필드 유형만 허용합니다. 지원되지 않는 유형을 사용하면 오류가 반환됩니다.

#### 기본 필드

내장 필드로, 플러그인 설치가 필요하지 않습니다.

| 유형 | 설명 |
|------|------|
| `TextField` | 한 줄 텍스트 |
| `TextareaField` | 여러 줄 텍스트 |
| `NumberField` | 숫자 |
| `RateField` | 평점 |
| `DateField` | 날짜 |
| `RadioField` | 라디오 버튼 |
| `SelectField` | 드롭다운 (단일 선택) |
| `CheckboxField` | 체크박스 |
| `MultiSelectField` | 드롭다운 (다중 선택) |
| `CascadeSelectField` | 계단식 선택 |
| `CascadeDateField` | 계단식 날짜 |
| `EmployeeField` | 구성원 선택 |
| `DepartmentSelectField` | 부서 선택 |
| `AssociationFormField` | 연관 양식 |
| `AssociationQuery` | 연관 쿼리 |
| `AttachmentField` | 첨부 파일 |
| `ImageField` | 이미지 |
| `AddressField` | 주소 |
| `CountrySelectField` | 국가/지역 선택 |
| `TableField` | 하위 양식/상세 |
| `SerialNumberField` | 일련 번호 |

#### 고급 필드

내장 필드로, 일부 기능은 활성화가 필요할 수 있습니다.

| 유형 | 설명 |
|------|------|
| `EditorField` | 리치 텍스트 편집기 |
| `DigitalSignatureField` | 디지털 서명 |
| `LocationField` | GPS 위치 |

#### 플러그인 필드

사용하기 전에 플러그인 센터에서 해당 플러그인을 설치해야 합니다.

| 유형 | 필요한 플러그인 |
|------|-----------------|
| `TimeZoneDateField` | 국제화 시간대 컴포넌트 |
| `CC_AIButtonField` | AI 컴포넌트 |
| `CC_PG_ESignField` | 스마트 재무 컴포넌트 |
| `CC_TeambitionProjectLinksField` | Teambition 플러그인 |
| `CC_TeambitionTaskLinksField` | Teambition 플러그인 |
| `CC_MoneyField` | 스마트 재무 컴포넌트 |
| `CC_PhoneNumberField` | 확장 컴포넌트 |
| `CC_ChineseIdField` | 확장 컴포넌트 |
| `YidaDingtalkAgentField` | DingTalk 에이전트 플러그인 |

<Warning>
플러그인 필드는 해당 플러그인이 설치되지 않은 경우 생성에 실패합니다. 이러한 필드 유형을 사용하기 전에 플러그인 센터에서 필요한 플러그인을 설치하세요.
</Warning>

<Warning>
`MobileField`, `EmailField` 등의 필드 유형은 지원되지 않으며 거부됩니다. 위 표에 나열된 유형만 사용하세요.
</Warning>

## 페이지 관리

```bash
# 사용자 정의 페이지 생성
openyida create-page <appType> "<페이지명>" [--datasource <jsonOrFile>]

# 페이지 컴파일 및 게시
openyida publish <源文件路径> <appType> <formUuid> [--skip-lint]

# 페이지 공유 구성
openyida save-share-config <appType> <formUuid> <url> <isOpen> [openAuth]
openyida get-page-config <appType> <formUuid>
openyida verify-short-url <appType> <formUuid> <url>
```

## 데이터 관리

```bash
# 통합 데이터 관리
openyida data <action> <resource> [args]

# 양식 데이터
openyida data query form <appType> <formUuid> [--page N] [--size N] [--search-json JSON]
openyida data get form <appType> --inst-id <id>
openyida data create form <appType> <formUuid> --data <JSON>  # JSON 키는 필드 ID 또는 필드 레이블 가능
openyida data update form <appType> --inst-id <id> --data <JSON>

# 프로세스 데이터
openyida data query process <appType> <formUuid> [--instance-status STATUS]
openyida data get process <appType> --process-inst-id <id>
openyida data create process <appType> <formUuid> --process-code <code> --data <JSON>

# 작업 관리
openyida data query tasks <appType> --type <todo|done|submitted|cc>
openyida data execute task <appType> --task-id <id> --process-inst-id <id> --out-result <AGREE|DISAGREE>

# 하위 양식
openyida data query subform <appType> <formUuid> --inst-id <id> --table-field-id <fieldId>
```

## 커넥터 관리

```bash
# 커넥터 나열
openyida connector list [--page N] [--size N]

# 커넥터 생성
openyida connector create "<名称>" "<域名>" --operations <file> [选项]

# 커넥터 세부 정보
openyida connector detail <connector-id>

# 커넥터 삭제
openyida connector delete <connector-id> [--force]

# 작업 관리
openyida connector add-action --connector-id <id> --operations <file> [--confirm]
openyida connector list-actions <connector-id>
openyida connector delete-action <connector-id> <operation-id> [--force]
openyida connector test --connector-id <id> --action <actionId> [选项]

# 인증 계정 관리
openyida connector list-connections <connector-id>
openyida connector create-connection <connector-id> "<名称>" [选项]

# 스마트 생성
openyida connector smart-create --curl "<curl命令>" [--name <name>] [--desc <desc>]

# 보조 도구
openyida connector parse-api --curl "<curl命令>" --output <file>
openyida connector gen-template [输出路径]
```

## DingTalk CLI 통합

```bash
# DingTalk CLI (dws) 명령어 패스스루
openyida dws <command> [args]

# DingTalk CLI 설치
openyida dws install

# dws 도움말 표시
openyida dws help

# 예시
openyida dws contact user search --keyword "이름"    # 연락처 검색
openyida dws calendar event list                      # 캘린더 이벤트 목록
openyida dws todo task create --title "작업"          # 할 일 생성
openyida dws approval instance list                   # 결재 인스턴스 목록
openyida dws attendance record list                   # 근태 기록 조회
openyida dws chat robot send --content "메시지"       # 로봇 메시지 전송
```

`-f json`을 추가하면 JSON 출력을 사용합니다 (AI 에이전트용).

<Note>
DingTalk CLI (`dws`)는 별도 설치가 필요합니다. `openyida dws install`로 안내에 따라 설치하거나, [DingTalk CLI 리포지토리](https://github.com/DingTalk-Real-AI/dingtalk-workspace-cli)에서 수동 설치하세요.
</Note>

## 작업 센터

```bash
# 작업 센터 조회
openyida task-center <type> [--page N] [--size N] [--keyword TEXT]
```

지원되는 유형: `todo` (할 일), `created` (내가 생성한), `done` (처리 완료), `cc` (참조), `pre-submit` (대리 제출).

## 대화 내보내기

```bash
# AI 대화 기록 내보내기
openyida export-conversation [옵션]

# 옵션
openyida export-conversation --output <path>       # 출력 파일 경로 지정
openyida export-conversation --input <file>        # 입력 대화 파일 지정
openyida export-conversation --latest              # 최신 대화만 내보내기 (기본값)
openyida export-conversation --list                # 사용 가능한 대화 목록 표시
```

## 전역 옵션

| 옵션 | 설명 |
|------|------|
| `--help`, `-h` | 도움말 정보 표시 |
| `--version`, `-v` | 버전 표시 |
| `--qr` | 터미널 QR 코드로 로그인 |
| `--force` | 강제 실행, 확인 건너뛰기 |
| `--skip-lint` | `publish` 시 Yida 코딩 표준 사전 검사 건너뛰기 |
