OpenYida는 풍부한 CLI 명령을 제공하며 앱의 전체 수명 주기 관리를 지원합니다.

---

## 환경 및 인증

```bash
# 환경 감지
openyida env

# 로그인
openyida login [--qr]              # --qr은 터미널 QR 코드 사용
openyida login --wukong            # Wukong 환경 전용

# 로그아웃
openyida logout

# 로그인 상태 관리
openyida auth status               # 로그인 상태 확인
openyida auth login                # 로그인 실행
openyida auth refresh              # 로그인 상태 갱신
openyida auth logout               # 로그아웃

# 조직 관리
openyida org list                  # 액세스 가능한 조직 나열
openyida org switch --corp-id <id> # 조직 전환

# 작업 디렉터리 초기화
openyida copy [--force]            # project 디렉터리 복사 (빈 디렉터리에는 내용을 직접 복사)
openyida copy -skills              # yida-skills를 AI 도구 skills 디렉터리에 복사

# 진단 도구
openyida doctor [--fix]            # 환경 진단
openyida doctor --monitor          # 헬스 모니터링
openyida doctor --report <format>  # 보고서 생성 (json|markdown|html)
```

## 양식 관리

```bash
# 양식 생성
openyida create-form create <appType> "<表单名>" <字段JSON> \
  [--layout <布局>] \
  [--theme <主题>] \
  [--label-align <对齐>]

# 양식 업데이트
openyida create-form update <appType> <formUuid> <修改JSON>

# 양식 Schema 가져오기
openyida get-schema <appType> <formUuid>  # 필드 ID를 사용하는 모든 작업 전에 반드시 실행

# 양식 구성 업데이트
openyida update-form-config <appType> <formUuid> <isRenderNav> <title>
```

<Warning>
  Yida 필드 ID는 플랫폼에서 무작위로 생성됩니다(예: `textField_eftt1aa5m`, `selectField_fix024y92`). 필드 이름으로는 추측할 수 없습니다. 필드 ID를 사용하는 모든 작업(데이터 생성/수정, 쿼리 조건 설정, 커스텀 페이지 작성 등) 전에 **반드시** `openyida get-schema`를 실행하여 실제 필드 ID를 가져와야 합니다.
</Warning>

### 지원되는 필드 유형

`create-form` 명령은 다음 필드 유형만 허용합니다. 지원되지 않는 유형을 사용하면 오류가 반환됩니다.

#### 기본 필드

내장 필드로, 플러그인 설치가 필요하지 않습니다.

| 유형 | 설명 |
|------|------|
| `TextField` | 한 줄 텍스트 |
| `TextareaField` | 여러 줄 텍스트 |
| `NumberField` | 숫자 |
| `RateField` | 평점 |
| `DateField` | 날짜 |
| `RadioField` | 라디오 버튼 |
| `SelectField` | 드롭다운 (단일 선택) |
| `CheckboxField` | 체크박스 |
| `MultiSelectField` | 드롭다운 (다중 선택) |
| `CascadeSelectField` | 계단식 선택 |
| `CascadeDateField` | 계단식 날짜 |
| `EmployeeField` | 구성원 선택 |
| `DepartmentSelectField` | 부서 선택 |
| `AssociationFormField` | 연관 양식 |
| `AssociationQuery` | 연관 쿼리 |
| `AttachmentField` | 첨부 파일 |
| `ImageField` | 이미지 |
| `AddressField` | 주소 |
| `CountrySelectField` | 국가/지역 선택 |
| `TableField` | 하위 양식/상세 |
| `SerialNumberField` | 일련 번호 |

#### 고급 필드

내장 필드로, 일부 기능은 활성화가 필요할 수 있습니다.

| 유형 | 설명 |
|------|------|
| `EditorField` | 리치 텍스트 편집기 |
| `DigitalSignatureField` | 디지털 서명 |
| `LocationField` | GPS 위치 |

#### 플러그인 필드

사용하기 전에 플러그인 센터에서 해당 플러그인을 설치해야 합니다.

| 유형 | 필요한 플러그인 |
|------|-----------------|
| `TimeZoneDateField` | 국제화 시간대 컴포넌트 |
| `CC_AIButtonField` | AI 컴포넌트 |
| `CC_PG_ESignField` | 스마트 재무 컴포넌트 |
| `CC_TeambitionProjectLinksField` | Teambition 플러그인 |
| `CC_TeambitionTaskLinksField` | Teambition 플러그인 |
| `CC_MoneyField` | 스마트 재무 컴포넌트 |
| `CC_PhoneNumberField` | 확장 컴포넌트 |
| `CC_ChineseIdField` | 확장 컴포넌트 |
| `YidaDingtalkAgentField` | DingTalk 에이전트 플러그인 |

<Warning>
플러그인 필드는 해당 플러그인이 설치되지 않은 경우 생성에 실패합니다. 이러한 필드 유형을 사용하기 전에 플러그인 센터에서 필요한 플러그인을 설치하세요.
</Warning>

<Warning>
`MobileField`, `EmailField` 등의 필드 유형은 지원되지 않으며 거부됩니다. 위 표에 나열된 유형만 사용하세요.
</Warning>

## 데이터 관리

<Warning>
  데이터 생성 또는 수정 전에 **반드시** `openyida get-schema`를 실행하여 실제 필드 ID를 가져와야 합니다. 데이터 작성 후 `openyida data query form <appType> <formUuid> --size 1`로 1건을 조회하여 `formData` 필드에 실제 값이 있는지 확인하세요. 모든 필드 값이 비어 있으면 필드 ID가 잘못된 것이므로 `get-schema`를 다시 실행해야 합니다.
</Warning>

```bash
# 통합 데이터 관리
openyida data <action> <resource> [args]

# 양식 데이터
openyida data query form <appType> <formUuid> [--page N] [--size N] [--search-json JSON]
openyida data get form <appType> --inst-id <id>
openyida data create form <appType> <formUuid> --data-json <JSON>  # JSON 키는 필드 ID 또는 필드 레이블 가능
openyida data update form <appType> --inst-id <id> --data-json <JSON>

# 프로세스 데이터
openyida data query process <appType> <formUuid> [--instance-status STATUS]
openyida data get process <appType> --process-inst-id <id>
openyida data create process <appType> <formUuid> --process-code <code> --data-json <JSON>

# 작업 관리
openyida data query tasks <appType> --type <todo|done|submitted|cc>
openyida data execute task <appType> --task-id <id> --process-inst-id <id> --out-result <AGREE|DISAGREE>

# 하위 양식
openyida data query subform <appType> <formUuid> --inst-id <id> --table-field-id <fieldId>
```

## 글로벌 작업 센터

`appType`을 지정하지 않고 현재 사용자의 글로벌 작업 센터 데이터를 조회합니다.

```bash
# 할 일 작업
openyida task-center todo [--page N] [--size N] [--keyword TEXT]

# 내가 생성한 작업
openyida task-center created [--page N] [--size N] [--keyword TEXT] [--no-detail]

# 처리 완료
openyida task-center done [--page N] [--size N] [--keyword TEXT]

# 참조로 받은 작업
openyida task-center cc [--page N] [--size N] [--keyword TEXT]

# 대리 제출
openyida task-center proxy [--page N] [--size N] [--keyword TEXT]
```

### 작업 유형

| 유형 | 설명 | API 엔드포인트 |
|------|------|---------------|
| `todo` | 처리 대기 작업 | `getTodoTasksInCorp` |
| `created` | 내가 시작한 프로세스 | `getMyCreateInCorp` |
| `done` | 이미 처리한 작업 | `getDoneTasksInCorp` |
| `cc` | 참조로 받은 작업 | `getNotifyMeInCorp` |
| `proxy` | 대리 제출한 프로세스 | `getSubmitAgentInCorp` |

### 선택적 매개변수

| 매개변수 | 설명 | 기본값 |
|----------|------|--------|
| `--page N` | 페이지 번호 | `1` |
| `--size N` | 페이지당 항목 수 (최대 100) | `20` |
| `--keyword TEXT` | 검색 키워드 | — |
| `--no-detail` | 상세 정보 생략 (`created` 유형만 지원) | — |

<Info>
  `openyida data query tasks`와 달리 `task-center` 명령은 글로벌 작업 센터 데이터를 조회하며 `appType` 매개변수가 필요하지 않습니다.
</Info>

## 플래시 노트를 PRD로 변환

AI를 사용하여 DingTalk 플래시 노트(회의 녹음 텍스트)를 구조화된 PRD 문서로 변환합니다.

```bash
# 파일에서 읽기
openyida flash-to-prd --file <파일경로> [--name <프로젝트명>] [--max-tokens <숫자>]

# 표준 입력에서 읽기
cat meeting.txt | openyida flash-to-prd --name <프로젝트명> [--max-tokens <숫자>]
```

### 매개변수

| 매개변수 | 단축형 | 설명 | 기본값 |
|------|------|------|------|
| `--file` | `-f` | 플래시 노트 파일 경로 | — |
| `--name` | `-n` | 프로젝트 이름 (생략 시 내용에서 자동 감지) | — |
| `--max-tokens` | — | AI 응답의 최대 토큰 수 | `8000` |

이 명령은 지정된 파일 또는 표준 입력에서 플래시 노트 내용을 읽고, 텍스트 전처리(회의 메타데이터 추출 및 발화자 식별 포함)를 수행한 후 Yida AI 서비스를 호출하여 구조화된 PRD를 생성합니다. 출력된 PRD 파일은 프로젝트의 `prd/` 디렉토리에 저장됩니다.

### 출력

명령은 stdout에 다음 형태의 JSON 결과를 출력합니다:

```json
{
  "success": true,
  "projectName": "프로젝트명",
  "prdFile": "/path/to/prd/프로젝트명.md",
  "contentLength": 4096,
  "meetingRecognition": {
    "metaFields": 3,
    "a1Sections": 2,
    "speakers": 5
  }
}
```

<Info>
  이 명령을 실행하기 전에 로그인이 필요합니다. 아직 인증하지 않았다면 먼저 `openyida login`을 실행하세요.
</Info>

## 통합 및 자동화

폼 이벤트에서 트리거되어 알림, 데이터 조회, 데이터 생성 등의 자동화된 작업을 실행하는 통합 로직 플로우를 생성합니다.

```bash
openyida integration create <appType> <formUuid> "<flowName>" [options]
```

### 필수 인수

| 인수 | 설명 |
|------|------|
| `<appType>` | 애플리케이션 ID |
| `<formUuid>` | 트리거 폼 UUID |
| `<flowName>` | 통합 플로우 이름 |

### 옵션

| 옵션 | 설명 | 기본값 |
|------|------|--------|
| `--process-code <code>` | 업데이트할 기존 프로세스 코드 (생략하면 새로 생성) | — |
| `--receivers <id1,id2>` | 알림 수신자 사용자 ID (쉼표로 구분) | — |
| `--title <text>` | 알림 제목 | 플로우 이름 |
| `--content <text>` | 알림 본문 | — |
| `--events <types>` | 트리거 이벤트 (쉼표로 구분): `insert`, `update`, `delete` | `insert` |
| `--data-form-uuid <uuid>` | "단일 레코드 가져오기" 데이터 노드의 폼 UUID | — |
| `--data-condition <mapping>` | 데이터 노드의 필드 매핑 (`bFieldId:bFieldName:aFieldId[:componentType]`). 여러 번 지정 가능. | — |
| `--add-data-form-uuid <uuid>` | "레코드 추가" 데이터 노드의 폼 UUID | — |
| `--add-data-assignment <mapping>` | 데이터 추가 노드의 필드 할당 (`targetFieldId:valueType:value`). `valueType`은 `processVar`, `literal` 또는 `column`일 수 있음. 여러 번 지정 가능. | — |
| `--publish` | 생성 후 즉시 플로우 게시 | 초안만 |

### 예시

```bash
# 새 제출 시 트리거되는 알림 플로우 생성
openyida integration create APP_XXX FORM-YYY "New Order Alert" \
  --receivers "user1,user2" \
  --events insert \
  --title "New Order" \
  --content "A new order was submitted"

# 생성 후 즉시 게시
openyida integration create APP_XXX FORM-YYY "Auto Notify" \
  --receivers "user1" \
  --publish
```

## DingTalk CLI (dws)

연락처, 캘린더, 할 일, 결재, 근태 등을 위한 DingTalk Workspace CLI 프록시 명령입니다.

```bash
openyida dws <command> [args]
```

<Info>
  `dws` 명령은 DingTalk CLI를 별도로 설치해야 합니다. 설치되어 있지 않으면 OpenYida가 처음 사용 시 설치를 안내합니다.
</Info>

### 자주 사용하는 명령

| 명령 | 설명 |
|------|------|
| `contact user search --keyword "<name>"` | 연락처 검색 |
| `calendar event list` | 캘린더 이벤트 목록 |
| `todo task create --title "<title>" --executors "<userId>"` | 할 일 작업 생성 |
| `approval instance list` | 결재 인스턴스 목록 |
| `attendance record list` | 근태 기록 조회 |
| `chat robot send --content "<message>"` | 챗 로봇 메시지 전송 |

### 특수 명령

```bash
# DingTalk CLI 설치
openyida dws install

# 도움말 표시
openyida dws help
```

### 출력 형식

기본적으로 출력은 사람이 읽기 쉬운 테이블 형식으로 포맷됩니다. AI 에이전트에 적합한 JSON 출력을 위해 `-f json`을 추가하세요.

```bash
openyida dws contact user search --keyword "name" -f json
```

| 옵션 | 별칭 | 설명 | 기본값 |
|------|------|------|--------|
| `--output` | `-o` | 출력 파일 경로 | 자동 생성 경로 |
| `--input` | `-i` | 입력 대화 파일 (JSON/JSONL) | — |
| `--latest` | — | 최신 대화만 내보내기 | 활성화됨 |
| `--list` | — | 사용 가능한 대화 기록 목록 | — |

### 예시

```bash
# 현재 대화 내보내기
openyida export-conversation

# 특정 파일로 내보내기
openyida export-conversation -o output.md

# 사용 가능한 대화 목록
openyida export-conversation --list

# 특정 입력 파일에서 내보내기
openyida export-conversation -i conversation.json -o result.md
```
