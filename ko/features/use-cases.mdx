OpenYida는 다양한 비즈니스 시나리오에 적용할 수 있으며 다음은 일반적인 애플리케이션 유형의 예입니다.

---

## 🏢 엔터프라이즈 비즈니스 시스템

다중 모듈 비즈니스 플랫폼을 구축하고 복잡한 양식, 워크플로우, 데이터 연결을 지원합니다.

<AccordionGroup>
  <Accordion title="IPD（통합 제품 개발）">
    **프롬프트：**
    ```
    Yida를 사용하여 IPD 시스템을 만들고 칩 생산 전체 프로세스를 관리하세요.
    프로젝트 시작, 설계 검토, 테이프아웃, 양산 등 단계를 포함합니다
    ```
    **출력：** 다중 양식 앱. 단계 추적, 검토 체크리스트, 마일스톤 관리, 팀 간 가시성을 포함합니다.
  </Accordion>

  <Accordion title="CRM（고객 관계 관리）">
    **프롬프트：**
    ```
    CRM을 구축하세요. 리드, 고객, 기회, 팔로업 기록 등 모듈을 포함합니다
    ```
    **출력：** 관련 양식：리드 → 고객 → 기회 → 계약. 상태 추적 및 할당 필드를 포함합니다.
  </Accordion>

  <Accordion title="인사 및 휴가 관리">
    **프롬프트：**
    ```
    인사 휴가 관리 시스템을 만드세요. 휴가 신청, 승인 프로세스, 휴가 잔액 통계를 포함합니다
    ```
    **출력：** 승인 프로세스를 포함하는 휴가 신청 양식, 잔액 계산기, 관리자 대시보드.
  </Accordion>

  <Accordion title="프로젝트 관리">
    **프롬프트：**
    ```
    프로젝트 관리 앱을 구축하세요. 작업, 마일스톤, 팀원, 진행 상황 추적을 포함합니다
    ```
    **출력：** 우선순위/상태/담당자 필드를 포함하는 작업 목록, 마일스톤 추적기, 간트 차트 보기.
  </Accordion>
</AccordionGroup>

## 💰 실용 도구 및 계산기

단일 용도의 계산 또는 데이터 수집 도구.

<AccordionGroup>
  <Accordion title="개인 급여 계산기">
    **프롬프트：**
    ```
    개인 급여 계산기 앱을 구축하세요. 세전 급여를 입력하면 소득세와 실수령 급여가 자동 계산됩니다
    ```
    **출력：** 세전 공제, 소득세 계산, 실수령 합계의 수식 필드를 포함하는 양식.
  </Accordion>

  <Accordion title="예산 추적기">
    **프롬프트：**
    ```
    개인 예산 추적기를 구축하세요. 수입, 지출 분류, 월간 합계를 기록합니다
    ```
    **출력：** 분류 드롭다운과 자동 계산되는 월간 잔액을 포함하는 수입/지출 입력 양식.
  </Accordion>
</AccordionGroup>

## 🌐 랜딩 페이지 및 포털

마케팅 페이지, 제품 소개, 내부 포털.

<AccordionGroup>
  <Accordion title="엔터프라이즈 제품 랜딩 페이지">
    **프롬프트：**
    ```
    智联协同 기업 협업 제품의 랜딩 페이지를 생성하세요. 첫 화면, 기능 소개, 연락 양식을 포함합니다
    ```
    **출력：** 완전한 사용자 정의 표시 페이지. 첫 화면 배너, 기능 카드, 임베디드 연락 양식을 포함합니다.
  </Accordion>

  <Accordion title="이벤트 등록 페이지">
    **프롬프트：**
    ```
    연례 기술 컨퍼런스의 이벤트 등록 페이지를 만드세요. 참가자 정보 양식과 포럼 선택을 포함합니다
    ```
    **출력：** 브랜드화된 등록 페이지. 양식 검증 및 확인 로직을 포함합니다.
  </Accordion>
</AccordionGroup>

## 🎮 인터랙티브 이벤트 및 미니게임

내부 이벤트, 휴일, 마케팅 이벤트의 인터랙티브 도구.

<AccordionGroup>
  <Accordion title="원소절 수수께끼 맞추기">
    **프롬프트：**
    ```
    원소절 수수께끼 맞추기 게임을 만드세요. AI가 수수께끼 이미지를 생성하고 사용자가 답을 제출합니다. 틀리면 AI가 유머러스한 응답을 합니다
    ```
    **출력：** AI 생성 수수께끼 이미지, 답안 제출 양식, 조건부 응답 로직을 포함하는 게임 페이지.
  </Accordion>

  <Accordion title="생일 축하 미니게임">
    **프롬프트：**
    ```
    생일 축하 미니게임을 구축하세요. 사용자가 이름을 입력하면 개인화된 AI 축하 카드를 받을 수 있습니다
    ```
    **출력：** 인터랙티브 축하 카드 생성기. 응답 추적의 데이터 저장소를 포함합니다.
  </Accordion>
</AccordionGroup>

## 🔌 API 통합（커넥터）

HTTP 커넥터를 통해 Yida 앱과 외부 시스템을 연결합니다.

<AccordionGroup>
  <Accordion title="Webhook 알림">
    **프롬프트：**
    ```
    DingTalk webhook의 HTTP 커넥터를 만들고 양식 제출 알림을 보내는 POST 액션을 추가하세요
    ```
    **출력：** 테스트 기능을 포함하는 구성된 커넥터. 양식 이벤트시 트리거할 수 있습니다.
  </Accordion>

  <Accordion title="curl에서 스마트 생성">
    **프롬프트：**
    ```
    이 curl 명령에서 스마트하게 커넥터를 만드세요：curl -X POST https://api.example.com/v1/users -H "Authorization: Bearer TOKEN" -d '{"name":"test"}'
    ```
    **출력：** 자동 구성된 커넥터. 기본 URL, 요청 헤더, 액션 매개변수를 추출합니다.
  </Accordion>
</AccordionGroup>

---

<Note>
이것들은 시작점입니다. 출력은 Yida 플랫폼에서 완전히 편집 가능한 앱（블랙박스 구성이 아님）이므로 언제든지 Yida 비주얼 에디터나 AI에 대한 후속 지시를 통해 생성된 앱을 최적화할 수 있습니다.
</Note>
