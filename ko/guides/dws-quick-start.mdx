## 5분 빠른 시작

<Steps>
  <Step title="DingTalk CLI 설치">
    ```bash
    openyida dws install
    ```

    또는 수동 설치:

    ```bash
    curl -fsSL https://raw.githubusercontent.com/DingTalk-Real-AI/dingtalk-workspace-cli/main/scripts/install.sh | sh
    ```
  </Step>

  <Step title="DingTalk 앱 생성">
    1. [DingTalk 오픈 플랫폼](https://dev.dingtalk.com/) 접속
    2. "사내 애플리케이션" → "앱 생성"으로 이동
    3. 앱 기본 정보 입력
    4. "보안 설정"에서 리다이렉트 URL 추가: `http://127.0.0.1`
    5. 앱 버전 게시
  </Step>

  <Step title="자격 증명 획득">
    앱 상세 페이지에서 다음을 획득합니다:

    - **Client ID** (AppKey)
    - **Client Secret** (AppSecret)
  </Step>

  <Step title="자격 증명 구성">
    **방법 1: 환경 변수 (권장)**

    ```bash
    export DWS_CLIENT_ID="your-client-id"
    export DWS_CLIENT_SECRET="your-client-secret"
    ```

    **방법 2: 명령 매개변수**

    ```bash
    openyida dws auth login --client-id your-client-id --client-secret your-client-secret
    ```
  </Step>

  <Step title="로그인">
    ```bash
    openyida dws auth login
    ```

    안내에 따라 OAuth 디바이스 플로우 인증을 완료하세요.
  </Step>
</Steps>

## 주요 사용 시나리오

### 시나리오 1: 연락처 검색

```bash
# 기본 검색
openyida dws contact user search --keyword "張三"

# JSON 출력 (AI Agent에 적합)
openyida dws contact user search --keyword "張三" -f json

# 파일로 저장
openyida dws contact user search --keyword "張三" -o contacts.json
```

### 시나리오 2: 할 일 생성

```bash
# 개인 할 일 생성
openyida dws todo task create --title "准备周报" --executors "your-userid"

# 팀 할 일 생성
openyida dws todo task create --title "项目评审" --executors "userid1,userid2,userid3"

# 모든 할 일 목록 조회
openyida dws todo task list
```

### 시나리오 3: 근태 기록 조회

```bash
# 오늘의 출퇴근 기록 조회
openyida dws attendance record list

# 특정 사용자의 근태 조회
openyida dws attendance record list --userids "your-userid"
```

### 시나리오 4: 캘린더 일정 관리

```bash
# 오늘의 일정 목록 조회
openyida dws calendar event list

# 회의 생성
openyida dws calendar event create \
  --title "주간 회의" \
  --start-time "2026-03-29T10:00:00+08:00" \
  --end-time "2026-03-29T11:00:00+08:00" \
  --participants "userid1,userid2"

# 일정 여유/바쁨 상태 조회
openyida dws calendar free-busy query \
  --userids "userid1,userid2" \
  --start-time "2026-03-29T09:00:00+08:00" \
  --end-time "2026-03-29T18:00:00+08:00"
```

### 시나리오 5: 로봇 메시지 전송

```bash
# 텍스트 메시지 전송
openyida dws chat robot send --content "안녕하세요, 주간 보고서 제출을 잊지 마세요!"

# Markdown 메시지 전송
openyida dws chat robot send --content "## 회의 알림\n시간: 오늘 오후 3시\n장소: 회의실 A"
```

### 시나리오 6: 승인 프로세스 관리

```bash
# 모든 승인 인스턴스 목록 조회
openyida dws approval instance list

# 휴가 승인 생성
openyida dws approval instance create \
  --processCode "leave" \
  --formData '{"duration": "3", "reason": "事假"}'

# 승인 상세 정보 조회
openyida dws approval instance get --instanceId "INSTANCE-123"
```

## AI Agent 통합 예제

### Cursor/Claude Code에서 자동 호출

```javascript
// 코드에서 직접 사용
const { execSync } = require('child_process');

// 연락처 검색
const result = execSync('openyida dws contact user search --keyword "悟空" -f json', {
  encoding: 'utf8'
});
const users = JSON.parse(result);
console.log(users);

// 할 일 생성
execSync('openyida dws todo task create --title "代码审查" --executors "userId123"', {
  stdio: 'inherit'
});
```

### MCP Server 통합

DingTalk CLI는 MCP 프로토콜을 기본 지원하며, MCP Server로 사용할 수 있습니다:

```json
{
  "mcpServers": {
    "dws": {
      "command": "dws",
      "args": ["mcp"]
    }
  }
}
```

## 자주 발생하는 문제 해결

### Q1: "명령을 찾을 수 없음" 오류

```bash
# dws가 설치되어 있는지 확인
which dws

# 설치되지 않은 경우 실행
openyida dws install
```

### Q2: "인증 실패" 오류

```bash
# 로그인 상태 확인
openyida dws auth status

# 다시 로그인
openyida dws auth login
```

### Q3: "권한 부족" 오류

다음 사항을 확인하세요:

1. DingTalk 앱이 게시되어 있는지
2. 기업 관리자가 승인했는지
3. 자격 증명이 올바르게 구성되어 있는지

### Q4: JSON 파싱 실패

<Tip>
구조화된 출력을 얻으려면 항상 `-f json` 매개변수를 사용하세요.
</Tip>

```bash
openyida dws contact user search --keyword "張三" -f json
```

## 유용한 팁

### 팁 1: 별칭으로 명령 간소화

```bash
# ~/.zshrc 또는 ~/.bashrc에 추가
alias dws='openyida dws'

# 이제 바로 사용 가능
dws contact user search --keyword "張三"
```

### 팁 2: 파이프로 명령 조합

```bash
# 연락처를 검색하고 userId 추출
openyida dws contact user search --keyword "張三" -f json | jq '.[0].userId'
```

### 팁 3: 일괄 작업

```bash
# 할 일 일괄 생성 (파일에서 읽기)
cat tasks.json | while read task; do
  openyida dws todo task create --title "$task" --executors "userId"
done
```

### 팁 4: 예약 작업

```bash
# 매일 오전 9시에 일일 보고서 제출 알림 (crontab)
0 9 * * * openyida dws chat robot send --content "일일 보고서 제출을 잊지 마세요!" >> /tmp/daily-reminder.log
```
