<AccordionGroup>

  <Accordion title="OpenYida 를 사용하려면 Yida 계정이 필요합니까?">
    예. OpenYida 는 [Yida](https://www.aliwork.com) 로우코드 플랫폼에서 앱을 생성하고 관리하므로 Yida 가 활성화된 Alibaba/DingTalk 계정이 필요합니다. `openyida login` 을 인증하십시오.
  </Accordion>

  <Accordion title="어떤 AI 도구가 지원됩니까?">
    OpenYida 는 MCP(모델 컨텍스트 프로토콜) 를 지원하거나 markdown서를 읽을 수 있는 AI 코딩 도구와 호환됩니다. 확인된 지원: **Claude Code, Aone Copilot, OpenCode, Cursor, Visual Studio Code, Qoder,悟空**. 실제로는셸을 수 있는 모든 도구가 사용 가능합니다.
  </Accordion>

  <Accordion title="필요한 Node.js 버전은?">
    Node.js **≥ 18** 이 필요합니다. `node --version` 으로 버전을 확인하십시오. 최신 LTS 를 [nodejs.org](https://nodejs.org) 에서 다운로드하십시오.
  </Accordion>

  <Accordion title="로그인 QR 코드가 표시되지 않으면?">
    브라우저 지원하는 터미널에서 `openyida login` 을 보십시오. 헤드리스 환경(원격 서버 등) 의 경우 DingTalk/悟空 환경에서는 `openyida login --wukong` 을 사용하거나 터미널 출력의 로그인 URL 을 복사하여 로컬 브라우저에서 여십시오.
  </Accordion>

  <Accordion title="세션이 만료된 경우갱 방법은?">
    `openyida auth refresh` 를 세션을갱하고 전체 로그인 플로우를 피하십시오. `openyida login` 을 재인증하십시오.
  </Accordion>

  <Accordion title="여러 Yida 조직에 접근할 수 있는 경우 전환 방법은?">
    ```bash
    openyida org list       # 모든 조직 및 corpId 나열
    openyida org switch --corp-id <corpId>   # 대상 조직으로 전환
    openyida env            # 전환 확인
    ```
  </Accordion>

  <Accordion title="AI 가 생성한 양식에 필드가 누락된 경우 추가 방법은?">
    AI 에 직접 지시하십시오:
    ```
    [페이지명] 에 [필드명] 필드를 추가, 필드 유형: [TextField/NumberField/DateField/SelectField/...], 필수로 설정
    ```
    OpenYida 는 업데이트된 Schema 로 `openyida create-form` 을 호출하여 기존 필드를 덮어쓰지 않고 새 필드를 병합합니다. [지원되는 필드 유형](/ko/features/skills#지원되는-필드-유형)만 허용되며, 지원되지 않는 유형은 오류로 거부됩니다.
  </Accordion>

  <Accordion title="생성된 앱을 Yida 시각적 편집기에서 편집할 수 있습니까?">
    물론입니다. 출력은 표준 Yida 앱입니다 — Yida 플랫폼의 시각적 편집기에서 언제든지 열어 수동으로 수정할 수 있습니다. OpenYida 는 앱을 생성하며 설정을 잠그지 않습니다.
  </Accordion>

  <Accordion title="앱을 공개적으로 접근 가능하게 하는 방법은?">
    ```
    [페이지명] 페이지를 공개적으로 접근 가능하게 설정
    ```
    또는 직접
    ```bash
    openyida save-share-config
    ```
    페이지 공유 설정을 구성하여 로그인 없이 링크를 통해 누구나 접근할 수 있도록 합니다.
  </Accordion>

  <Accordion title="OpenYida 로 기존 Yida 앱을 마이그레이션할 수 있습니까?">
    예. `openyida export` 를 사용하여 기존 앱에서 마이그레이션 패키지를 생성하고 `openyida import` 를 사용하여 다른 조직이나 환경에서 재구축합니다.
  </Accordion>

  <Accordion title="`create-form` 과 `create-page` 의 차이점은?">
    - **`create-form`**: Yida **양식 페이지** 생성 — 데이터 입력 양식,底层에서 Yida 의 내장 데이터 저장소를 사용. 구조화된 데이터 수집(등록, 신청, 기록) 에 사용.
    - **`create-page`**: **사용자 정의 표시 페이지** 생성 — JSX 기반 페이지, 대시보드, 랜딩 페이지, 대화형 UI 에 사용. `openyida publish` 로 컴파일 및 게시.
  </Accordion>

  <Accordion title="OpenYida 는 프로세스/승인 워크플로우를 지원합니까?">
    예. `openyida create-process` 를 사용하여 승인 단계가 있는 프로세스 양식을 생성하고 `openyida configure-process` 로 워크플로우 규칙(승인자, 조건, 알림) 을 정의합니다. 프로세스 구성이 실패했지만 양식이 이미 생성된 경우, `--formUuid` 플래그로 양식을 다시 만들지 않고 재시도할 수 있습니다.
  </Accordion>

  <Accordion title="문제가 발생한 경우 디버그 방법은?">
    1. `openyida doctor --fix` 일반적인 문제를 자동 감지 및 수정
    2. `openyida env` 로그인 상태 및 환경 검증
    3. `openyida auth status` 세션 확인
    4. [GitHub Issues](https://github.com/openyida/openyida/issues) 확인하여 알려진 문제 파악
    5. 새로운 버그인 경우 Node.js 버전, 명령, 전체 오류 출력을 첨부하여 issue 제출
  </Accordion>

  <Accordion title="OpenYida 에 기여하는 방법은?">
    전체 프로세스에 대해서는 [기여 가이드](/ko/contributing) 를 읽으십시오. 간단히 말하면: 저장소 포크, 기능 브랜치에서 변경, `npm test` PR 제출. AI 지원 기여도 환영합니다!
  </Accordion>
