## 환경 요구사항

시작하기 전에 다음 종속성이 설치되어 있는지 확인하세요:

| 종속성 | 버전 | 확인 명령어 |
|--------|------|-------------|
| [Node.js](https://nodejs.org) | ≥ 18 | `node --version` |

<Tip>
  Node.js만 있으면 됩니다. Python, Playwright, 빌드 단계가 필요 없습니다 — 순수 Node.js 구현입니다.
</Tip>

---

## 1단계 — 설치

```bash
npm install -g openyida
```

설치 확인:

```bash
openyida --version
```

---

## 2단계 — Yida 로그인

```bash
openyida login
```

OpenYida는 브라우저 창(또는 QR 코드)을 열어 Yida/Alibaba 계정으로 인증합니다. 자격 증명은 로컬에 캐시되며 한 번만 로그인하면 됩니다.

<Info>
  **Wukong**을 AI 도구로 사용하는 경우 특수 로그인 플래그를 사용하세요:

  ```bash
  openyida login --wukong
  ```
</Info>

---

## 3단계 — 환경 확인

```bash
openyida env
```

현재 활성화된 AI 코딩 도구(Claude Code, Cursor, Wukong 등)를 감지하고 로그인 상태를 표시합니다. 환경을 전환할 때 언제든지 이 명령을 실행하세요.

---

## 4단계 — 프로젝트 작업 공간 초기화

```bash
openyida copy
```

`project/` 작업 공간 템플릿을 현재 AI 도구의 작업 디렉토리에 복사하여 AI가 앱 설정과 페이지 템플릿에 액세스할 수 있도록 합니다.

---

## 5단계 — 빌드 시작

AI 코딩 어시스턴트와 직접 대화하세요. 다음은 시작을 위한 프롬프트 예시입니다:

```
Yida에서 칩 생산 워크플로우 전체를 관리하는 IPD 시스템을 구축해주세요
```

```
리드, 고객, 거래를 포함한 CRM을 구축해주세요
```

```
개인 급여 계산기 앱을 만들어주세요
```

```
Yida에서 생일 축하 미니 게임을 만들어주세요
```

AI는 OpenYida의 스킬 패키지를 사용하여 올바른 순서로 CLI 명령을 호출합니다: `create-app` → `create-form` / `create-page` → `publish` → `save-share-config`.

---

## 6단계 — 앱 게시

AI가 앱을 생성한 후 게시 명령을 실행하세요:

```bash
openyida publish
```

또는 AI에게 물어보세요:
```
[페이지 이름] 페이지를 게시해주세요
페이지를 공개 접근 가능하게 설정해주세요
```

---

## 진단 실행

문제가 있는 경우 내장 진단 도구를 실행하세요:

```bash
openyida doctor
```

Node.js 버전, 로그인 상태, 환경 설정을 확인하고 `--fix`로 일반적인 문제를 자동으로 수정할 수 있습니다.

---

OpenClaw를 사용하는 경우 OpenYida 스킬 패키지를 직접 설치하세요:
