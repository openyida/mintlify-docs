## 환경 요구사항

시작하기 전에 다음 의존성이 설치되어 있는지 확인하세요:

| 의존성 | 버전 | 확인 명령어 |
|------|------|----------|
| [Node.js](https://nodejs.org) | ≥ 18 | `node --version` |

<Tip>
  이것이 전부입니다. OpenYida는 순수 Node.js 구현입니다 — Python, Playwright, 빌드 단계가 필요 없습니다.
</Tip>

---

## 1단계 — 설치

```bash
npm install -g openyida
```

설치 확인:

```bash
openyida --version
```

---

## 2단계 — Yida 로그인

```bash
openyida login
```

OpenYida는 브라우저 창을 열거나(또는 QR 코드를 표시하여) Yida/알리바바 계정으로 인증할 수 있도록 합니다. 자격 증명은 로컬에 캐시됩니다 — 한 번만 실행하면 됩니다.

<Info>
  **悟空**를 AI 도구로 사용하는 경우 특수 로그인 플래그를 사용하세요:

  ```bash
  openyida login --wukong
  ```
</Info>

---

## 3단계 — 환경 확인

```bash
openyida env
```

이는 현재 활성화된 AI 프로그래밍 도구(Claude Code, Cursor, 悟空 등)를 감지하고 현재 로그인 상태를 표시합니다. 환경을 전환할 때마다 이 명령을 실행하세요.

---

## 4단계 — 프로젝트 작업 공간 초기화

```bash
openyida copy
```

이는 `project/` 작업 공간 템플릿을 현재 AI 도구의 작업 디렉토리에 복사하여 AI가 애플리케이션 구성 및 페이지 템플릿에 액세스할 수 있도록 합니다.

---

## 5단계 — 구축 시작

이제 AI 프로그래밍 도우미와 대화하세요. 다음은 시작을 위한 몇 가지 프롬프트입니다:

```
Yida에서 칩 생산 워크플로우를 관리하는 IPD 시스템을 구축해줘
```

```
리드, 고객 및 거래가 포함된 CRM 시스템을 구축해줘
```

```
개인 급여 계산기 애플리케이션을 만들어줘
```

```
Yida에서 생일 축하 미니 게임을 생성해줘
```

AI는 OpenYida의 스킬 팩을 사용하여 올바른 CLI 명령을 순차적으로 호출합니다: `create-app` → `create-form` / `create-page` → `publish` → `save-share-config`.

---

## 6단계 — 애플리케이션 게시

AI가 애플리케이션을 생성한 후 다음 명령으로 게시하세요:

```bash
openyida publish
```

또는 AI에게 물어보세요:
```
[페이지 이름] 페이지를 게시해줘
페이지를 공개적으로 접근 가능하게 만들어줘
```

---

## 진단 실행

문제가 있는 것 같으면 내장 진단 도구를 실행하세요:

```bash
openyida doctor
```

Node.js 버전, 로그인 상태 및 환경 구성을 확인하고 `--fix`를 사용하여 일반적인 문제를 자동으로 해결할 수 있습니다.
