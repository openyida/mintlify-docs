## Key Files Explained

### `bin/yida.js`
The CLI entry point. All `openyida <command>` invocations are routed here. When adding a new command, register it in this file and update `README.md`.

### `lib/utils.js`
Shared utilities used across all modules: HTTP request helpers, cookie management, error formatting, and environment detection utilities.

### `project/config.json`
Stores the active app context: `appId`, `corpId`, `pageId`. The AI reads this file to know which app and page to target when executing commands.

### `yida-skills/SKILL.md`
The entry document for the AI skill pack. When an AI tool initializes OpenYiDA, it reads this file first to understand the available capabilities and how to invoke them.

## Code Conventions

| Convention | Rule |
|-----------|------|
| Module system | CommonJS (`require` / `module.exports`) — no ESM |
| Dependencies | Prefer Node.js native APIs; avoid new dependencies |
| Error handling | `console.error()` + `process.exit(1)` |
| Credentials | Never hardcode cookies, tokens, or secrets |
| New commands | Must update CLI command table in `README.md` |
| Branch policy | Never push directly to `main` — always use PRs |
