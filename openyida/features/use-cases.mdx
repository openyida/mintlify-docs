OpenYiDA is versatile. Below are the most common use cases, organized by app category.

---

## 🏢 Enterprise Business Systems

Build multi-module business platforms with complex forms, workflows, and data relationships.

<AccordionGroup>
  <Accordion title="IPD (Integrated Product Development)">
    **Prompt:**
    ```
    Build me an IPD system on Yida to manage the full chip production workflow,
    including project initiation, design review, tape-out, and mass production phases
    ```
    **What you get:** Multi-form app with phase tracking, review checklists, milestone management, and cross-team visibility.
  </Accordion>

  <Accordion title="CRM (Customer Relationship Management)">
    **Prompt:**
    ```
    Help me set up a CRM with modules for leads, customers, deals, and follow-up records
    ```
    **What you get:** Linked forms for leads → customers → opportunities → contracts, with status tracking and assignment fields.
  </Accordion>

  <Accordion title="HR & Leave Management">
    **Prompt:**
    ```
    Create an HR leave management system with leave requests, approval workflow, and leave balance tracking
    ```
    **What you get:** Leave request form with approval process, balance calculator, and manager dashboard.
  </Accordion>

  <Accordion title="Project Management">
    **Prompt:**
    ```
    Build a project management app with tasks, milestones, team members, and progress tracking
    ```
    **What you get:** Task list with priority/status/assignee fields, milestone tracker, and Gantt-style view.
  </Accordion>
</AccordionGroup>

## 🔌 API Integration (Connectors)

Connect Yida apps to external systems via HTTP connectors.

<AccordionGroup>
  <Accordion title="Webhook Notification">
    **Prompt:**
    ```
    Create an HTTP connector for our DingTalk webhook, add a POST action to send form submission notifications
    ```
    **What you get:** Configured connector with test capability, ready to trigger on form events.
  </Accordion>

  <Accordion title="Smart Connector from curl">
    **Prompt:**
    ```
    Smart-create a connector from this curl command: curl -X POST https://api.example.com/v1/users -H "Authorization: Bearer TOKEN" -d '{"name":"test"}'
    ```
    **What you get:** Auto-configured connector with extracted base URL, headers, and action parameters.
  </Accordion>
</AccordionGroup>

<Note>
These are starting points. Because the output is a fully editable Yida app (not a black-box config), you can always refine the generated app through the Yida visual editor or by giving the AI follow-up instructions.
</Note>
