  <Accordion title="Do I need a Yida account to use OpenYiDA?">
    Yes. OpenYiDA creates and manages apps on the [Yida (宜搭)](https://www.aliwork.com) low-code platform, which requires an Alibaba / DingTalk account with Yida access. Run `openyida login` to authenticate.
  </Accordion>

  <Accordion title="Which AI tools are supported?">
    OpenYiDA works with any AI coding tool that supports MCP (Model Context Protocol) or can read markdown skill documentation. Confirmed support: **Claude Code, Aone Copilot, OpenCode, Cursor, Visual Studio Code, Qoder, and Wukong (悟空)**. In practice, it works with any tool that can execute shell commands.
  </Accordion>

  <Accordion title="What Node.js version do I need?">
    Node.js **≥ 18** is required. Check your version with `node --version`. Download the latest LTS from [nodejs.org](https://nodejs.org).
  </Accordion>

  <Accordion title="The login QR code isn't showing up. What do I do?">
    Try running `openyida login` in a terminal that supports browser launch. If you're in a headless environment (e.g., a remote server), use `openyida login --wukong` if you're in the DingTalk/Wukong environment, or copy the login URL from the terminal output and open it in a local browser.
  </Accordion>

  <Accordion title="My session expired. How do I refresh it?">
    Run `openyida auth refresh` to renew your session without going through the full login flow again. If that fails, run `openyida login` to re-authenticate.
  </Accordion>

  <Accordion title="I have access to multiple Yida organizations. How do I switch?">
    ```bash
    openyida org list       # List all orgs with their corpId
    openyida org switch --corp-id <corpId>   # Switch to the target org
    openyida env            # Confirm the switch
    ```
  </Accordion>

  <Accordion title="The AI generated a form but it's missing some fields. How do I add them?">
    Just tell the AI:
    ```
    Add a [field name] field to [page name], field type: [text/number/date/dropdown/...], make it required
    ```
    OpenYiDA will call `openyida create-form` with the updated schema, which merges new fields into the existing form without overwriting existing ones.
  </Accordion>

  <Accordion title="Can I edit the generated app in the Yida visual editor?">
    Absolutely. The output is a standard Yida app — you can open it in the Yida platform's visual editor at any time and make changes manually. OpenYiDA generates apps, not locked configurations.
  </Accordion>

  <Accordion title="How do I make my app publicly accessible?">
    ```
    Make the [page name] page publicly accessible
    ```
    Or run directly:
    ```bash
    openyida save-share-config
    ```
    This configures the page's sharing settings so anyone with the link can view it without logging in.
  </Accordion>

  <Accordion title="Can I use OpenYiDA to migrate an existing Yida app?">
    Yes. Use `openyida export` to generate a migration package from an existing app, and `openyida import` to rebuild it in another organization or environment.
  </Accordion>

  <Accordion title="What's the difference between `create-form` and `create-page`?">
    - **`create-form`**: Creates a Yida **form page** — a data-entry form backed by Yida's built-in data storage. Use this for structured data collection (registrations, requests, records).
    - **`create-page`**: Creates a **custom display page** — a JSX-based page for dashboards, landing pages, and interactive UIs. These are compiled and published via `openyida publish`.
  </Accordion>

  <Accordion title="Does OpenYiDA support process/approval workflows?">
    Yes. Use `openyida create-process` to create a process form with approval steps, and `openyida configure-process` to define the workflow rules (approvers, conditions, notifications).
  </Accordion>

  <Accordion title="Something's broken. How do I debug?">
    1. Run `openyida doctor --fix` — auto-detects and fixes common issues
    2. Run `openyida env` — verify login status and active environment
    3. Run `openyida auth status` — check session validity
    4. Check the [GitHub Issues](https://github.com/openyida/openyida/issues) for known problems
    5. If it's a new bug, open an issue with: your Node.js version, the command you ran, and the full error output
  </Accordion>

  <Accordion title="How do I contribute to OpenYiDA?">
    Read the [Contributing Guide](/openyida/contributing) for the full workflow. TL;DR: fork the repo, make changes on a feature branch, run `npm test`, and open a PR. AI-assisted contributions are welcome!
  </Accordion>
