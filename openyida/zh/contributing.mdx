# 贡献指南

感谢您对 OpenYida 的兴趣！我们欢迎各种形式的贡献，包括代码、文档、Bug 报告和功能建议。

## 开发环境搭建

### 1. Fork 并克隆仓库

```bash
# Fork 仓库到您的 GitHub 账号
# 然后克隆到本地
git clone https://github.com/YOUR_USERNAME/openyida.git
cd openyida
```

### 2. 安装依赖

```bash
npm install
```

### 3. 运行测试

```bash
npm test
```

### 4. 本地调试

```bash
# 链接到全局
npm link

# 现在可以使用 openyida 命令测试
openyida --help
```

## 代码规范

### TypeScript

- 所有代码必须使用 TypeScript
- 启用严格模式 (`strict: true`)
- 避免使用 `any` 类型

### 代码风格

```bash
# 运行 ESLint
npm run lint

# 自动修复
npm run lint:fix

# 运行 Prettier
npm run format
```

### 提交规范

使用 [Conventional Commits](https://conventionalcommits.org/) 格式：

```
<type>(<scope>): <subject>

<body>

<footer>
```

类型说明：

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `docs` | 文档更新 |
| `style` | 代码格式（不影响功能）|
| `refactor` | 重构 |
| `test` | 测试相关 |
| `chore` | 构建/工具相关 |

示例：

```
feat(connector): add support for custom headers

Add ability to specify custom HTTP headers in connector configuration.
This enables authentication with APIs that require specific headers.

Closes #123
```

## 贡献流程

### 1. 创建分支

```bash
git checkout -b feat/your-feature-name
# 或
git checkout -b fix/issue-description
```

### 2. 开发和测试

- 编写代码
- 添加测试用例
- 确保所有测试通过
- 更新相关文档

### 3. 提交更改

```bash
git add .
git commit -m "feat(scope): description"
```

### 4. 推送到您的 Fork

```bash
git push origin feat/your-feature-name
```

### 5. 创建 Pull Request

- 在 GitHub 上创建 PR
- 填写 PR 模板
- 关联相关 Issue（如果有）
- 等待代码审查

## PR 审查流程

1. **自动化检查**: CI 会运行测试和代码检查
2. **代码审查**: 维护者会审查代码
3. **反馈处理**: 根据反馈修改代码
4. **合并**: 审查通过后合并到主分支

## 文档贡献

文档位于 `docs/` 目录，使用 MDX 格式。

### 编辑文档

1. 找到对应的 `.mdx` 文件
2. 进行编辑
3. 本地预览（如有 Mintlify 环境）
4. 提交 PR

### 添加新页面

1. 创建 `.mdx` 文件
2. 添加 frontmatter：

```yaml
---
title: 页面标题
description: 页面描述
---
```

3. 在 `docs.json` 中添加导航
4. 提交 PR

## 报告 Bug

### 提交前检查

- 搜索现有 Issue，避免重复
- 确认使用的是最新版本
- 尝试在干净环境中复现

### Bug 报告模板

```markdown
**描述**
清晰描述 Bug

**复现步骤**
1. 步骤一
2. 步骤二
3. ...

**期望行为**
描述期望的正确行为

**实际行为**
描述实际发生的行为

**环境信息**
- OS: [e.g. macOS 14.0]
- Node.js: [e.g. 18.12.0]
- OpenYida: [e.g. 1.2.3]

**附加信息**
截图、日志等
```

## 功能建议

### 提交前考虑

- 是否符合项目定位
- 是否对多数用户有用
- 是否有替代方案

### 功能建议模板

```markdown
**功能描述**
清晰描述建议的功能

**使用场景**
描述这个功能在什么场景下使用

**期望行为**
描述功能的具体行为

**替代方案**
是否有其他实现方式

**附加信息**
参考实现、设计稿等
```

## 社区规范

### 行为准则

- 尊重他人，保持友善
- 接受建设性批评
- 关注最有益于社区的内容
- 展现同理心

### 沟通渠道

- GitHub Issues: Bug 报告和功能讨论
- GitHub Discussions: 一般性讨论
- Pull Requests: 代码贡献
