# Estrutura do Projeto

OpenYida adota um design arquitetônico modular com uma organização clara do código, facilitando a manutenção e a extensão.

## Descrição dos Módulos Centrais

### src/commands/

Armazena a implementação de todos os comandos CLI, cada arquivo corresponde a um subcomando:

| Arquivo | Função |
|------|------|
| `create-app.ts` | Criar aplicativo do Yida |
| `create-form.ts` | Criar página de formulário |
| `create-page.ts` | Criar página personalizada |
| `publish.ts` | Publicar aplicativo |
| `share.ts` | Compartilhar aplicativo |
| `data.ts` | Operações de gerenciamento de dados |
| `connector.ts` | Gerenciamento de conectores |
| `report.ts` | Operações de relatórios |
| `doctor.ts` | Ferramenta de diagnóstico |
| `login.ts` | Autenticação de login |

### src/core/

Módulo de lógica de negócios central:

- **yida-api.ts**: Encapsula a API da plataforma aberta do Yida, fornece chamadas de interface unificadas
- **ai-service.ts**: Integra múltiplos serviços de IA (OpenAI, Claude, Tongyi Qianwen, etc.)
- **config.ts**: Gerencia arquivos de configuração local (~/.openyida/config.json)
- **auth.ts**: Lida com estado de login, atualização de Token, alternância entre múltiplas organizações

### src/utils/

Funções utilitárias gerais:

- **logger.ts**: Saída de log, suporta níveis e cores
- **file.ts**: Funções auxiliares de operação de arquivos
- **validator.ts**: Validação e verificação de entrada

### src/types/

Definições de tipos TypeScript, garantem segurança de tipos do código:

- **yida.ts**: Tipos relacionados à API do Yida
- **ai.ts**: Tipos relacionados a serviços de IA
- **index.ts**: Exportação de tipos gerais

## Sistema de Skills

```
skills/
├ builtin/               # Skills integrados
│   ├── code-generator/    # Geração de código
│   ├── form-builder/      # Construção de formulários
│   ├── page-designer/     # Design de páginas
│  └ data-analyzer/     # Análise de dados
└ custom/                # Skills personalizados por usuários
   └ (extensão de usuários)
```

Skills adotam arquitetura de plugins, suportam carregamento em tempo real e extensão dinâmica.

## Gerenciamento de Configuração

### Configuração Local

Local de armazenamento: `~/.openyida/config.json`

```json
{
  "defaultOrg": "org-default",
  "orgs": {
    "org-default": {
      "accessToken": "xxx",
      "refreshToken": "xxx",
      "expiresAt": 1234567890
    }
  },
  "aiProvider": "openai",
  "aiConfig": {
    "apiKey": "sk-xxx",
    "model": "gpt-4"
  }
}
```

### Configuração do Projeto

Pode criar `.openyidarc.json` no diretório raiz do projeto:

```json
{
  "appId": "APP-XXX",
  "defaultEnv": "dev",
  "environments": {
    "dev": { "appKey": "xxx", "appSecret": "xxx" },
    "prod": { "appKey": "xxx", "appSecret": "xxx" }
  }
}
```

## Mecanismo de Extensão

### Adicionar Novo Comando

1. Criar novo arquivo em `src/commands/`
2. Implementar função de processamento de comandos
3. Registrar comando em `src/index.ts`

### Adicionar Novo Skill

1. Criar diretório de Skill em `skills/custom/`
2. Implementar `skill.json` e lógica principal
3. Carregar usando `openyida skill load`

## Normas de Desenvolvimento

- Todas as novas funcionalidades devem incluir definições de tipos
- A implementação de comandos deve incluir documentação `--help`
- As funções centrais devem ter cobertura de testes unitários
- Seguir configuração do ESLint e Prettier
