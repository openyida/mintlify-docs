# Estrutura do projeto

OpenYida adota um design de arquitetura modular com organização de código clara, facilitando manutenção e extensão.

## Visão geral de diretórios

## Descrição dos módulos principais

### src/commands/

Contém a implementação de todos os comandos CLI, cada arquivo corresponde a um subcomando:

| Arquivo | Função |
|------|------|
| `create-app.ts` | Criar aplicativo Yida |
| `create-form.ts` | Criar página de formulário |
| `create-page.ts` | Criar página personalizada |
| `publish.ts` | Publicar aplicativo |
| `share.ts` | Compartilhar aplicativo |
| `data.ts` | Operações de gestão de dados |
| `connector.ts` | Gestão de conectores |
| `report.ts` | Operações de relatórios |
| `doctor.ts` | Ferramenta de diagnóstico |
| `login.ts` | Autenticação de login |

### src/core/

Módulos de lógica de negócios principal:

- **yida-api.ts**: Encapsula a API da plataforma aberta Yida, fornece interface de chamada unificada
- **ai-service.ts**: Integra múltiplos serviços de IA (OpenAI, Claude, Tongyi Qwen, etc.)
- **config.ts**: Gerencia arquivos de configuração local (~/.openyida/config.json)
- **auth.ts**: Trata estado de login, atualização de tokens, troca multi-organização

### src/utils/

Funções utilitárias gerais:

- **logger.ts**: Saída de logs, suporte a níveis e cores
- **file.ts**: Funções auxiliares de operações de arquivos
- **validator.ts**: Validação e verificação de entradas

### src/types/

Definições de tipos TypeScript, garantindo segurança de tipos do código:

- **yida.ts**: Tipos relacionados à API Yida
- **ai.ts**: Tipos relacionados a serviços de IA
- **index.ts**: Exportações de tipos gerais

## Sistema de habilidades

```
skills/
├── builtin/               # Habilidades integradas
│   ├── code-generator/    # Geração de código
│   ├── form-builder/      # Construção de formulários
│   ├── page-designer/     # Design de páginas
│   └── data-analyzer/     # Análise de dados
└── custom/                # Habilidades personalizadas do usuário
    └── (extensões do usuário)
```

As habilidades adotam uma arquitetura de plugins, suportando carregamento dinâmico e extensão.

## Gestão de configuração

### Configuração local

Local de armazenamento: `~/.openyida/config.json`

```json
{
  "defaultOrg": "org-default",
  "orgs": {
    "org-default": {
      "accessToken": "xxx",
      "refreshToken": "xxx",
      "expiresAt": 1234567890
    }
  },
  "aiProvider": "openai",
  "aiConfig": {
    "apiKey": "sk-xxx",
    "model": "gpt-4"
  }
}
```

### Configuração de projeto

Pode-se criar `.openyidarc.json` no diretório raiz do projeto:

```json
{
  "appId": "APP-XXX",
  "defaultEnv": "dev",
  "environments": {
    "dev": { "appKey": "xxx", "appSecret": "xxx" },
    "prod": { "appKey": "xxx", "appSecret": "xxx" }
  }
}
```

## Mecanismo de extensão

### Adicionar novo comando

1. Criar novo arquivo em `src/commands/`
2. Implementar função de tratamento de comando
3. Registrar comando em `src/index.ts`

### Adicionar nova habilidade

1. Criar diretório de habilidade em `skills/custom/`
2. Implementar `skill.json` e lógica principal
3. Carregar com `openyida skill load`

## Normas de desenvolvimento

- Todas as novas funcionalidades devem incluir definições de tipos
- Implementações de comandos devem incluir documentação `--help`
- Funcionalidades principais devem ter cobertura de testes unitários
- Seguir configurações de ESLint e Prettier
