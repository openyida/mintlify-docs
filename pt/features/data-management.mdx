## Formato do comando

```bash
openyida data <action> <resource> [args]
```

- **action**: `query` | `get` | `create` | `update` | `execute`
- **resource**: `form` | `process` | `task` | `subform` | `operation-records`

---

<Warning>
  Os IDs de campo no Yida sĂŁo gerados aleatoriamente pela plataforma (ex: `textField_eftt1aa5m`) e nĂŁo podem ser adivinhados. ### Formato de condiĂ§Ă”es de consulta

O parĂąmetro `--search-json` aceita um formato de consulta baseado em array. Cada condiĂ§ĂŁo Ă© um objeto com os seguintes campos:

<Note>
  O valor de `--search-json` deve ser um JSON vĂĄlido. Use aspas simples ao redor do JSON no terminal para evitar problemas de escape. VocĂȘ pode validar o JSON antes de usar com ferramentas como `jq` ou validadores online.

| Campo | DescriĂ§ĂŁo |
|-------|-----------|
| `key` | ID do campo (ex: `field_xxx`) |

| `type` | Tipo de dados: `TEXT`, `NUMBER`, `DATE`, etc. |

Exemplo:

```bash

```

### Exemplo de script de operaĂ§ĂŁo em lote
