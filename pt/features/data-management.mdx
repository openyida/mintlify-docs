## Formato do comando

```bash
openyida data <action> <resource> [args]
```

- **action**: `query` | `get` | `create` | `update` | `execute`
- **resource**: `form` | `process` | `task` | `subform` | `operation-records`

## Gerenciamento de dados do formulário

### Consultar dados do formulário

```bash
# Consulta básica
openyida data query form <appType> <formUuid>

# Consulta paginada
openyida data query form <appType> <formUuid> --page 1 --size 50

# Consulta condicional
openyida data query form <appType> <formUuid> \
  --search-json '[{"key":"field_xxx","value":"value","type":"TEXT","operator":"eq","componentName":"TextField"}]'

# Consultar instância específica
openyida data query form <appType> <formUuid> --inst-id <formInstId>

# Retornar apenas IDs
openyida data query form <appType> <formUuid> --ids-only

# Filtrar por criador e intervalo de datas
openyida data query form <appType> <formUuid> \
  --originator-id <userId> \
  --create-from "2026-01-01" --create-to "2026-12-31"
```

#### Parâmetros opcionais

| Parâmetro | Descrição |
|-----------|-----------|
| `--page N` | Número da página (padrão `1`) |
| `--size N` | Itens por página (padrão `20`, máx `100`) |
| `--search-json JSON` | Condições de pesquisa (deve ser JSON válido) |
| `--inst-id ID` | Consultar registro por ID de instância |
| `--ids-only` | Retornar apenas IDs de instância |
| `--originator-id ID` | Filtrar por criador |
| `--create-from DATE` / `--create-to DATE` | Filtrar por data de criação |
| `--modified-from DATE` / `--modified-to DATE` | Filtrar por data de modificação |
| `--dynamic-order FIELD` | Ordenar por campo |

### Obter um único registro

```bash
openyida data get form <appType> --inst-id <formInstId>
```

### Criar dados

```bash
openyida data create form <appType> <formUuid> \
  --data-json '{"field_xxx": "value", "field_yyy": 123}' \
  --dept-id <departmentId>
```

### Atualizar dados

```bash
openyida data update form <appType> \
  --inst-id <formInstId> \
  --data-json '{"field_xxx": "new value"}' \
  --use-latest-version y
```

## Gerenciamento de dados do fluxo

### Consultar instâncias de fluxo

```bash
# Consultar lista de fluxos
openyida data query process <appType> <formUuid> \
  --page 1 --size 20 \
  --instance-status RUNNING

# Status disponíveis: RUNNING (em andamento), COMPLETED (concluído), TERMINATED (encerrado)

# Retornar apenas IDs
openyida data query process <appType> <formUuid> --ids-only

# Filtrar por criador e status
openyida data query process <appType> <formUuid> \
  --originator-id <userId> \
  --instance-status COMPLETED \
  --create-from "2026-01-01" --create-to "2026-06-30"
```

#### Parâmetros opcionais

| Parâmetro | Descrição |
|-----------|-----------|
| `--page N` | Número da página (padrão `1`) |
| `--size N` | Itens por página (padrão `20`, máx `100`) |
| `--search-json JSON` | Condições de pesquisa (deve ser JSON válido) |
| `--instance-status STATUS` | Filtrar por status: `RUNNING`, `COMPLETED`, `TERMINATED` |
| `--task-id ID` | Filtrar por ID de tarefa |
| `--approved-result RESULT` | Filtrar por resultado de aprovação |
| `--ids-only` | Retornar apenas IDs de instância |
| `--originator-id ID` | Filtrar por criador |
| `--create-from DATE` / `--create-to DATE` | Filtrar por data de criação |
| `--modified-from DATE` / `--modified-to DATE` | Filtrar por data de modificação |

### Obter detalhes do fluxo

```bash
openyida data get process <appType> --process-inst-id <processInstanceId>
```

### Criar instância de fluxo

```bash
openyida data create process <appType> <formUuid> \
  --process-code <processCode> \
  --data-json '{"field_xxx": "value"}' \
  --dept-id <departmentId>
```

### Atualizar instância de fluxo

```bash
openyida data update process <appType> \
  --process-inst-id <processInstanceId> \
  --data-json '{"field_xxx": "updated value"}'
```

### Consultar registros de aprovação

```bash
openyida data query operation-records <appType> \
  --process-inst-id <processInstanceId>
```

## Gestão de tarefas

<Tip>
  Para consultar dados do centro de tarefas global sem especificar um `appType`, use o comando `openyida task-center`. Veja a [referência CLI](/pt/features/skills#centro-de-tarefas-global).
</Tip>

### Consultar lista de tarefas

```bash
# Tarefas pendentes
openyida data query tasks <appType> --type todo

# Tarefas concluídas
openyida data query tasks <appType> --type done

# Enviadas por mim
openyida data query tasks <appType> --type submitted

# CC para mim
openyida data query tasks <appType> --type cc

# Pesquisar tarefas por palavra-chave
openyida data query tasks <appType> --type todo \
  --keyword "aprovação" \
  --page 1 --size 50
```

#### Parâmetros opcionais

| Parâmetro | Descrição |
|-----------|-----------|
| `--type TYPE` | **(obrigatório)** Tipo de tarefa: `todo`, `done`, `submitted`, `cc` |
| `--page N` | Número da página (padrão `1`) |
| `--size N` | Itens por página (padrão `20`, máx `100`) |
| `--keyword TEXT` | Pesquisar tarefas por palavra-chave |
| `--process-codes CODES` | Filtrar por códigos de processo |
| `--instance-status STATUS` | Filtrar por status da instância |

### Executar tarefa

```bash
openyida data execute task <appType> \
  --task-id <taskId> \
  --process-inst-id <processInstanceId> \
  --out-result AGREE \
  --remark "Aprovado, por favor continuar" \
  --data-json '{"comment": "Aprovação aprovada"}'
```

#### Parâmetros

| Parâmetro | Obrigatório | Descrição |
|-----------|:-----------:|-----------|
| `--task-id ID` | Sim | ID da tarefa a executar |
| `--process-inst-id ID` | Sim | ID da instância de fluxo |
| `--out-result RESULT` | Sim | Resultado da execução: `AGREE` (aprovar) ou `DISAGREE` (não aprovar) |
| `--remark TEXT` | Sim | Comentário da execução |
| `--data-json JSON` | Não | Dados adicionais em formato JSON |
| `--no-execute-expressions` | Não | Não executar expressões ao salvar |

## Uso avançado

### Formato de condições de consulta

O parâmetro `--search-json` aceita um formato de consulta baseado em array. Cada condição é um objeto com os seguintes campos:

<Note>
  O valor de `--search-json` deve ser um JSON válido. Use aspas simples ao redor do JSON no terminal para evitar problemas de escape. Você pode validar o JSON antes de usar com ferramentas como `jq` ou validadores online.
</Note>

| Campo | Descrição |
|-------|-----------|
| `key` | ID do campo (ex: `field_xxx`) |
| `value` | Valor a comparar |
| `type` | Tipo de dados: `TEXT`, `NUMBER`, `DATE`, etc. |
| `operator` | Operador: `eq`, `like`, `gt`, `gte`, `lt`, `lte`, `in`, `range` |
| `componentName` | Tipo de componente: `TextField`, `NumberField`, `DateField`, etc. |

Exemplo:

```bash
openyida data query form <appType> <formUuid> \
  --search-json '[
    {"key":"field_status","value":"approved","type":"TEXT","operator":"eq","componentName":"TextField"},
    {"key":"field_amount","value":"1000","type":"NUMBER","operator":"gte","componentName":"NumberField"}
  ]'
```

### Exemplo de script de operação em lote

```bash
#!/bin/bash
# Exportar dados de formulário em lote

APP_TYPE="APP_XXXXXXXX"
FORM_UUID="FORM-XXXX-XXXX"
PAGE=1

while true; do
  RESULT=$(openyida data query form $APP_TYPE $FORM_UUID --page $PAGE --size 100)
  
  # Processar dados da página atual
  echo "$RESULT" >> ./export.json
  
  # Verificar se há próxima página
  HAS_MORE=$(echo "$RESULT" | jq '.hasMore')
  if [ "$HAS_MORE" != "true" ]; then
    break
  fi
  
  PAGE=$((PAGE + 1))
done
```

## Notas

<Warning>
  As operações de dados são permanentes, as operações de atualização não podem ser desfeitas. Recomenda-se realizar testes em pequena escala antes de executar operações em lote.
</Warning>

<Info>
  A consulta paginada tem um valor padrão de 20 registros por página, máximo 100. Para exportar grandes quantidades de dados, recomenda-se usar loop de paginação.
</Info>
