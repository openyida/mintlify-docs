O OpenYida fornece uma CLI unificada de gerenciamento de dados, suporta operações CRUD completas para dados de formulários, instâncias de fluxo, tarefas, subformulários, etc.

## Formato do comando

```bash
openyida data <action> <resource> [args]
```

- **action**: `query` | `get` | `create` | `update` | `execute`
- **resource**: `form` | `process` | `task` | `subform` | `operation-records`

## Gerenciamento de dados do formulário

### Consultar dados do formulário

```bash
# Consulta básica
openyida data query form <appType> <formUuid>

# Consulta paginada
openyida data query form <appType> <formUuid> --page 1 --size 50

# Consulta condicional
openyida data query form <appType> <formUuid> \
  --search-json '{"field_xxx": "value"}'

# Consultar instância específica
openyida data query form <appType> <formUuid> --inst-id <formInstId>
```

### Obter um único registro

```bash
openyida data get form <appType> --inst-id <formInstId>
```

### Criar dados

```bash
openyida data create form <appType> <formUuid> \
  --data '{"field_xxx": "value", "field_yyy": 123}' \
  --dept-id <departmentId>
```

Você também pode usar rótulos de campo em vez de IDs de campo no parâmetro `--data`. O CLI detecta automaticamente os rótulos de campo e os converte nos IDs de campo correspondentes obtendo o esquema do formulário.

```bash
# Usar rótulos de campo em vez de IDs de campo
openyida data create form <appType> <formUuid> \
  --data '{"Nome": "João", "Idade": 30, "Departamento": "Engenharia"}'
```

<Info>
  A resolução de rótulos de campo funciona fazendo a correspondência do texto do rótulo com o esquema do formulário. Tanto rótulos de campo quanto IDs de campo são aceitos no mesmo objeto `--data` — qualquer chave que não corresponda a um rótulo conhecido é passada como está.
</Info>

### Atualizar dados

```bash
openyida data update form <appType> \
  --inst-id <formInstId> \
  --data '{"field_xxx": "new value"}' \
  --use-latest-version y
```

## Gerenciamento de dados do fluxo

### Consultar instâncias de fluxo

```bash
# Consultar lista de fluxos
openyida data query process <appType> <formUuid> \
  --page 1 --size 20 \
  --instance-status RUNNING

# Status disponíveis: RUNNING (em andamento), COMPLETED (concluído), TERMINATED (encerrado)
```

### Obter detalhes do fluxo

```bash
openyida data get process <appType> --process-inst-id <processInstanceId>
```

### Criar instância de fluxo

```bash
openyida data create process <appType> <formUuid> \
  --process-code <processCode> \
  --data '{"field_xxx": "value"}' \
  --dept-id <departmentId>
```

### Consultar registros de aprovação

```bash
openyida data query operation-records <appType> \
  --process-inst-id <processInstanceId>
```

## Gestão de tarefas

### Consultar lista de tarefas

```bash
# Tarefas pendentes
openyida data query tasks <appType> --type todo

# Tarefas concluídas
openyida data query tasks <appType> --type done

# Enviadas por mim
openyida data query tasks <appType> --type submitted

# CC para mim
openyida data query tasks <appType> --type cc
```

### Executar tarefa

```bash
openyida data execute task <appType> \
  --task-id <taskId> \
  --process-inst-id <processInstanceId> \
  --out-result AGREE \
  --remark "Aprovado, por favor continuar" \
  --data '{"comment": "Aprovação aprovada"}'
```

**Opções de out-result:**
- `AGREE` - Aprovar
- `DISAGREE` - Não aprovar

## Gerenciamento de subformulários

### Consultar dados do subformulário

```bash
openyida data query subform <appType> <formUuid> \
  --inst-id <formInstId> \
  --table-field-id <fieldId> \
  --page 1 --size 20
```

## Uso avançado

### Condições de consulta complexas

```bash
# Consulta condicional de múltiplos campos
openyida data query form <appType> <formUuid> \
  --search-json '{
    "field_status": "approved",
    "field_amount": { "$gte": 1000 },
    "field_created_at": { "$gte": "2024-01-01" }
  }'
```

### Exemplo de script de operação em lote

```bash
#!/bin/bash
# Exportar dados de formulário em lote

APP_TYPE="APP_XXXXXXXX"
FORM_UUID="FORM-XXXX-XXXX"
PAGE=1

while true; do
  RESULT=$(openyida data query form $APP_TYPE $FORM_UUID --page $PAGE --size 100)
  
  # Processar dados da página atual
  echo "$RESULT" >> ./export.json
  
  # Verificar se há próxima página
  HAS_MORE=$(echo "$RESULT" | jq '.hasMore')
  if [ "$HAS_MORE" != "true" ]; then
    break
  fi
  
  PAGE=$((PAGE + 1))
done
```

## Exemplo de exportação de dados

```bash
# Exportar dados de um período específico
openyida data query form <appType> <formUuid> \
  --search-json '{
    "field_created_at": {
      "$gte": "2024-01-01T00:00:00",
      "$lte": "2024-12-31T23:59:59"
    }
  }' \
  --size 100 > ./data-export.json
```

## Notas

<Warning>
  As operações de dados são permanentes, as operações de exclusão e atualização não podem ser desfeitas. Recomenda-se realizar testes em pequena escala antes de executar operações em lote.
</Warning>

<Info>
  A consulta paginada tem um valor padrão de 20 registros por página, máximo 100. Para exportar grandes quantidades de dados, recomenda-se usar loop de paginação.
</Info>
