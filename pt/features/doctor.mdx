O OpenYida inclui uma poderosa ferramenta de diagnóstico `doctor` que ajuda os desenvolvedores a verificar o ambiente, diagnosticar problemas, reparar automaticamente e monitorar continuamente a saúde do aplicativo.

## Uso Básico

### Executar Diagnóstico

```bash
openyida doctor
```

### Reparo Automático

```bash
openyida doctor --fix
```

### Gerar Relatório

```bash
# Formato Markdown
openyida doctor --report markdown

# Formato JSON
openyida doctor --report json

# Formato HTML
openyida doctor --report html
```

## Itens de Verificação de Ambiente

| Item | Descrição | Reparo Automático |
|--------|------|----------|
| Versão CLI | Verifica se a versão do openyida corresponde ao SKILL.md | ✅ |
| Inicialização project/ | Verifica se o diretório de trabalho project/ existe | ✅ |
| Versão do Node.js | Verifica se ≥ 16 | ❌ |
| Playwright | Verifica se está instalado | ✅ |
| config.json | Verifica arquivo de configuração | ✅ |
| Login Yida | Verifica estado de login | ✅ |
| Conectividade de rede | Verifica conectividade de aliwork.com | ❌ |

---

## Itens de Diagnóstico de Aplicativo

| Item | Descrição |
|--------|------|
| Arquivo PRD | Verifica se existe documento de requisitos |
| Código fonte de páginas | Verifica arquivos de código fonte, console.log, arquivos vazios |
| Cache de Schema | Verifica formato de arquivo de cache |
| React Hooks | Verifica normas de uso de Hooks |

## Modo de Monitoramento de Saúde

Inicia monitoramento contínuo, verifica a cada minuto:

```bash
openyida doctor --monitor
```

<Warning>
A função de monitoramento de saúde está atualmente em versão beta. A API e os formatos de saída podem mudar em versões futuras.
</Warning>

Exemplo de saída:

```
[14:32:01] Saúde: 95/100  Aprovados: 18/19 | Erros: 0 | Avisos: 1

[14:33:01] Saúde: 100/100➡
  Aprovados: 19/19 | Erros: 0 | Avisos: 0
```

Pressione `Ctrl+C` para parar o monitoramento.

## Diagnóstico Online

Diagnóstico online para aplicativos já publicados:

```bash
openyida doctor --production --app <appId>
```

Conteúdo de verificação:
- Validade do ID do aplicativo
- Registros de erros online
- Métricas de desempenho

## Tickets e VOC

### Criar Ticket (Bug)

```bash
openyida doctor --create-ticket
```

Entrada interativa de título e descrição, envio automático para GitHub Issues.

### Criar VOC (Requisito)

```bash
openyida doctor --create-voc
```

### Envio Inteligente

O sistema julga automaticamente se é Bug ou requisito:

```bash
openyida doctor --auto-submit
```

## Pré-verificação

Executa verificação automaticamente antes de operações chave:

### Verificação Antes de Publicar

```bash
# Executa automaticamente
openyida publish executa automaticamente pré-verificação
```

### Verificação Antes de Criar

```bash
# Executa automaticamente
openyida create-app executa automaticamente pré-verificação
```

## Tipos de Reparo

| Tipo | Descrição | Exemplo |
|------|------|------|
| `AUTO` | Reparo automático | Criar config.json, excluir cache corrompida |
| `COMMAND` | Reparo por comando | Executar `pip install playwright` |
| `MANUAL` | Reparo manual | Instalar gh CLI, escrever PRD |

---

## Cálculo de Saúde

Fórmula de cálculo de pontuação de saúde (0-100):

```
Saúde = Taxa de aprovação × 100 - Erros × 10 - Avisos × 3
```

| Pontuação | Estado |
|------|------|
| 90-100 | Excelente
| 70-89 | Bom
| 50-69 | Regular
| < 50 | Precisa melhorar🔴
