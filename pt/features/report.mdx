## Criar relatórios

### Uso básico

```bash
openyida create-report <appType> "Nome do relatório" '<JSON-definição-ou-arquivo>'
```

O terceiro argumento pode ser uma string JSON inline ou um caminho para um arquivo JSON.

### Exemplo: criar um relatório de vendas

```bash
openyida create-report APP_XXXXXX "Sales Dashboard" '{
  "charts": [
    {
      "type": "bar",
      "title": "Monthly Sales",
      "cubeCode": "FORM_XXXXXX",
      "xField": "month",
      "yField": "amount"
    }
  ]
}'
```

### Carregar definição de gráfico de arquivo

```bash
openyida create-report APP_XXXXXX "Annual Report" ./charts-config.json
```

## Adicionar gráficos a um relatório

```bash
openyida append-chart <appType> <reportId> '<JSON-definição-ou-arquivo>'
```

### Exemplo

```bash
openyida append-chart APP_XXXXXX REPORT_XXXXX '{
  "type": "line",
  "title": "Growth Trend",
  "cubeCode": "FORM_XXXXXX",
  "xField": "month",
  "yField": "growth_rate"
}'
```

## Configuração de filtros

Os relatórios suportam filtros vinculados a gráficos, permitindo que os usuários filtrem dinamicamente os dados exibidos. Os filtros são definidos no nível do relatório e aplicados aos gráficos correspondentes.

```json
{
  "filters": [
    {
      "label": "Região",
      "field": "region",
      "type": "dropdown",
      "options": ["Norte", "Sul", "Leste", "Oeste"]
    },
    {
      "label": "Período",
      "field": "date",
      "type": "dateRange"
    }
  ],
  "charts": [
    {
      "type": "bar",
      "title": "Vendas por Região",
      "cubeCode": "FORM_XXXXXX",
      "xField": "region",
      "yField": "amount"
    }
  ]
}
```

| Campo | Descrição |
|-------|-----------|
| **label** | Rótulo exibido para o filtro |
| **field** | Campo do formulário vinculado ao filtro |
| **type** | Tipo de filtro (`dropdown`, `dateRange`, `text`, `number`) |
| **options** | Lista de opções disponíveis (apenas para `dropdown`) |

<Info>
Quando o tipo de filtro é `dropdown` e nenhuma opção é fornecida, as opções são geradas automaticamente a partir dos valores distintos do campo no formulário vinculado.
</Info>

## Tipos de gráficos

| Tipo | Descrição | Ideal para |
|------|-------------|----------|
| **bar** | Gráfico de barras | Comparar categorias |
| **line** | Gráfico de linhas | Tendências ao longo do tempo |
| **pie** | Gráfico circular | Relações parte-todo |
| **table** | Tabela de dados | Visualização detalhada de dados |
| **area** | Gráfico de área | Tendências cumulativas |
| **scatter** | Gráfico de dispersão | Análise de correlação |
| **indicator** | Cartão de indicador | Exibição de métricas-chave |
| **combo** | Gráfico combinado (barras + linhas) | Comparações de eixo duplo |
| **pivot** | Tabela dinâmica cruzada | Análise multidimensional |
| **gauge** | Gráfico de medidor | Acompanhamento de progresso ou metas |
| **radar** | Gráfico de radar | Comparações multidimensionais |
| **funnel** | Gráfico de funil | Conversão por etapas |

## Dimensionamento do layout de gráficos

Você pode controlar a largura e a altura de cada gráfico na grade do relatório usando as propriedades `w` e `h`. O relatório usa um sistema de grade de 6 colunas.

- **`w`** — largura da grade (1-6 colunas). Quando um gráfico excede o espaço restante na linha atual, ele é movido para a próxima linha.
- **`h`** — altura da grade (em unidades de grade).

Definir `w` ou `h` como `0` é válido e será respeitado. Se omitido, os valores padrão abaixo são usados.

| Tipo de gráfico | `w` padrão | `h` padrão |
|-----------------|------------|------------|
| **indicator** | 6 | 6 |
| **pie** | 3 | 22 |
| **bar** | 3 | 22 |
| **line** | 3 | 22 |
| **combo** | 6 | 22 |
| **table** | 6 | 38 |
| **pivot** | 6 | 30 |
| **gauge** | 2 | 18 |
| Outros tipos | 3 | 22 |

## Formato de definição de gráfico

### Gráfico de barras

```json
{
  "type": "bar",
  "title": "Sales by Region",
  "cubeCode": "FORM_XXXXXX",
  "xField": "region",
  "yField": "amount",
  "aggregation": "sum",
  "w": 6,
  "h": 22
}
```

### Gráfico de linhas

```json
{
  "type": "line",
  "title": "Traffic Trend",
  "cubeCode": "FORM_XXXXXX",
  "xField": "date",
  "yField": "visitors",
  "aggregation": "sum"
}
```

### Gráfico circular

```json
{
  "type": "pie",
  "title": "Market Share",
  "cubeCode": "FORM_XXXXXX",
  "categoryField": "product",
  "valueField": "share"
}
```

### Tabela de dados

```json
{
  "type": "table",
  "title": "Detail View",
  "cubeCode": "FORM_XXXXXX",
  "columns": [
    { "field": "name", "title": "Name" },
    { "field": "amount", "title": "Amount" },
    { "field": "date", "title": "Date" }
  ]
}
```

## Formato do cubeCode

O `cubeCode` identifica a fonte de dados do formulário vinculada ao gráfico. O formato segue o padrão:

```
FORM_<IDENTIFICADOR>
```

- O prefixo `FORM_` é obrigatório e indica que a fonte de dados é um formulário.
- O `<IDENTIFICADOR>` corresponde ao código único do formulário, geralmente em letras maiúsculas e separado por underscores (por exemplo, `FORM_MONTHLY_SALES`, `FORM_DAILY_SALES`).
- Para formulários genéricos ou quando o identificador exato não é relevante, utilize o placeholder `FORM_XXXXXX`.

Você pode encontrar o `cubeCode` de um formulário nas configurações do formulário ou na resposta da API ao listar formulários.

## Exemplo de configuração completa

### sales-report-charts.json

```json
{
  "charts": [
    {
      "type": "bar",
      "title": "Monthly Sales",
      "cubeCode": "FORM_MONTHLY_SALES",
      "xField": "month",
      "yField": "sales_amount",
      "aggregation": "sum"
    },
    {
      "type": "pie",
      "title": "Product Categories",
      "cubeCode": "FORM_SALES",
      "categoryField": "product_category",
      "valueField": "sales_amount"
    },
    {
      "type": "line",
      "title": "Sales Trend",
      "cubeCode": "FORM_DAILY_SALES",
      "xField": "date",
      "yField": "sales_amount",
      "aggregation": "sum"
    },
    {
      "type": "table",
      "title": "Sales Detail",
      "cubeCode": "FORM_SALES",
      "columns": [
        { "field": "order_no", "title": "Order No.", "width": 150 },
        { "field": "customer_name", "title": "Customer", "width": 200 },
        { "field": "product_name", "title": "Product", "width": 200 },
        { "field": "amount", "title": "Amount", "width": 120 },
        { "field": "sales_date", "title": "Date", "width": 150 }
      ],
      "pageSize": 10
    }
  ]
}
```

Criar o relatório:

```bash
openyida create-report APP_XXXXXX "Sales Analysis Report" ./sales-report-charts.json
```

## Melhores práticas

1. **Escolha o tipo de gráfico apropriado** - Gráficos de linhas para tendências, de barras para comparações, circulares para proporções
2. **Limite a quantidade de gráficos** - Mantenha 6 gráficos ou menos por relatório
3. **Inclua tabelas de dados** - Combine gráficos com tabelas para inspeção detalhada dos dados
4. **Configure tamanhos de página razoáveis** - Para tabelas de dados, use 10-20 linhas por página
