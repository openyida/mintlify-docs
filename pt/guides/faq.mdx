<AccordionGroup>

  <Accordion title="Preciso de uma conta YiDa para usar o OpenYida?">
    Sim. O OpenYida cria e gerencia aplicativos na plataforma low-code [YiDa](https://www.aliwork.com), exigindo uma conta Alibaba/DingTalk com YiDa habilitado. Execute `openyida login` para autenticar.
  </Accordion>

  <Accordion title="Quais ferramentas de IA são suportadas?">
    O OpenYida é compatível com qualquer ferramenta de codificação de IA que suporte MCP (Model Context Protocol) ou possa ler documentos de habilidades markdown. Suporte confirmado: **Claude Code, Aone Copilot, OpenCode, Cursor, Visual Studio Code, Qoder, Wukong**. Na prática, qualquer ferramenta que possa executar comandos shell funciona.
  </Accordion>

  <Accordion title="Qual versão do Node.js é necessária?">
    É necessário Node.js **≥ 18**. Verifique sua versão com `node --version`. Baixe a última LTS em [nodejs.org](https://nodejs.org).
  </Accordion>

  <Accordion title="E se o código QR de login não for exibido?">
    Tente executar `openyida login` em um terminal que suporte a inicialização do navegador. Se estiver em um ambiente headless (ex. servidor remoto), use `openyida login --wukong` em ambientes DingTalk/Wukong, ou copie a URL de login da saída do terminal e abra-a em seu navegador local.
  </Accordion>

  <Accordion title="Como atualizar uma sessão expirada?">
    Execute `openyida auth refresh` para renovar a sessão sem passar pelo fluxo de login completo. Se falhar, execute `openyida login` para reautenticar.
  </Accordion>

  <Accordion title="Como alternar entre várias organizações YiDa?">
    ```bash
    openyida org list       # Listar todas as organizações e corpIds
    openyida org switch --corp-id <corpId>   # Alternar para a organização alvo
    openyida env            # Confirmar a alternância
    ```
  </Accordion>

  <Accordion title="Como adicionar campos ausentes a formulários gerados por IA?">
    Diga diretamente à IA:
    ```
    Adicionar um campo [nome do campo] a [nome da página], tipo de campo: [TextField/NumberField/DateField/SelectField/...], torná-lo obrigatório
    ```
    O OpenYida chamará `openyida create-form` com o Schema atualizado, mesclando novos campos sem sobrescrever os existentes. Apenas [tipos de campo suportados](/pt/features/skills#tipos-de-campo-suportados) são aceitos — tipos não suportados serão rejeitados com erro.
  </Accordion>

  <Accordion title="Posso editar aplicativos gerados no editor visual do YiDa?">
    Absolutamente. A saída é um aplicativo YiDa padrão — você pode abri-lo no editor visual da plataforma YiDa a qualquer momento e fazer modificações manuais. O OpenYida gera aplicativos, não configurações bloqueadas.
  </Accordion>

  <Accordion title="Como tornar um aplicativo acessível publicamente?">
    ```
    Tornar a página [nome da página] acessível publicamente
    ```
    Ou executar diretamente:
    ```bash
    openyida save-share-config
    ```
    Configure as configurações de compartilhamento de página para que qualquer pessoa possa acessar via link sem login.
  </Accordion>

  <Accordion title="Posso migrar aplicativos YiDa existentes com o OpenYida?">
    Sim. Use `openyida export` para gerar um pacote de migração de um aplicativo existente, e `openyida import` para reconstruí-lo em outra organização ou ambiente.
  </Accordion>

  <Accordion title="Qual é a diferença entre `create-form` e `create-page`?">
    - **`create-form`**: Cria uma **página de formulário** YiDa — um formulário de entrada de dados usando o armazenamento de dados integrado do YiDa por baixo. Usado para coleta de dados estruturados (registros, solicitações, registros).
    - **`create-page`**: Cria uma **página de exibição personalizada** — uma página baseada em JSX para dashboards, landing pages, UI interativa. Compilada e publicada via `openyida publish`.
  </Accordion>

  <Accordion title="O OpenYida suporta fluxos de trabalho de processo/aprovação?">
    Sim. Use `openyida create-process` para criar formulários de processo com etapas de aprovação, e `openyida configure-process` para definir regras de fluxo de trabalho (aprovadores, condições, notificações).
  </Accordion>

  <Accordion title="Algo deu errado, como depurar?">
    1. Execute `openyida doctor --fix` — detecta e corrige automaticamente problemas comuns
    2. Execute `openyida env` — verifique o status de login e o ambiente
    3. Execute `openyida auth status` — verifique a validade da sessão
    4. Verifique [GitHub Issues](https://github.com/openyida/openyida/issues) para problemas conhecidos
    5. Se for um novo bug, envie um issue com: versão do Node.js, comando executado, saída de erro completa
  </Accordion>

  <Accordion title="Como contribuir para o OpenYida?">
    Leia o [Guia de contribuição](/pt/contributing) para o processo completo. Em resumo: faça fork do repo, faça alterações em uma branch de recurso, execute `npm test`, envie um PR. Contribuições assistidas por IA são bem-vindas!
  </Accordion>
