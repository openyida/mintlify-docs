## Pré-requisitos

Antes de começar, certifique-se de ter as seguintes dependências instaladas:

| Dependência | Versão | Comando de verificação |
|------|------|----------|
| [Node.js](https://nodejs.org) | ≥ 18 | `node --version` |

<Tip>
  Isso é tudo que você precisa. O OpenYida é uma implementação pura em Node.js — sem Python, sem Playwright, sem etapas de build.
</Tip>

---

## Passo 1 — Instalação

```bash
npm install -g openyida
```

Verifique a instalação:

```bash
openyida --version
```

---

## Passo 2 — Login no Yida

```bash
openyida login
```

O OpenYida abrirá uma janela do navegador (ou exibirá um código QR) para você autenticar com sua conta Yida/Alibaba. As credenciais são armazenadas em cache localmente — você só precisa fazer isso uma vez.

<Info>
  Se você estiver usando **Wukong** como ferramenta de IA, use a flag de login especial:

  ```bash
  openyida login --wukong
  ```
</Info>

---

## Passo 3 — Verificar ambiente

```bash
openyida env
```

Isso detecta a ferramenta de codificação de IA atualmente ativa (Claude Code, Cursor, Wukong, etc.) e exibe seu status de login atual. Execute este comando sempre que alternar ambientes.

---

## Passo 4 — Inicializar espaço de trabalho do projeto

```bash
openyida copy
```

Isso copia o modelo de espaço de trabalho `project/` para o diretório de trabalho da sua ferramenta de IA atual, permitindo que a IA acesse a configuração do seu aplicativo e os modelos de página.

---

## Passo 5 — Começar a construir

Agora basta conversar com seu assistente de codificação de IA. Aqui estão alguns prompts iniciais:

```
Construa para mim um sistema IPD no Yida para gerenciar o fluxo de trabalho completo de produção de chips
```

```
Ajude-me a construir um sistema CRM com leads, contatos e negócios
```

```
Crie um aplicativo de calculadora de salário pessoal
```

```
Gere um mini-jogo de felicitações de aniversário no Yida
```

A IA usará o pacote de habilidades do OpenYida para chamar os comandos CLI corretos em sequência: `create-app` → `create-form` / `create-page` → `publish` → `save-share-config`.

---

## Passo 6 — Publicar o aplicativo

Após a IA gerar o aplicativo, use o seguinte comando para publicar:

```bash
openyida publish
```

Ou pergunte à IA:
```
Publique a página [nome da página]
Torne a página acessível publicamente
```

---

## Executar diagnósticos

Se algo parecer errado, execute a ferramenta de diagnóstico integrada:

```bash
openyida doctor
```

Ele verifica sua versão do Node.js, status de login e configuração do ambiente, e pode corrigir automaticamente problemas comuns com `--fix`.
