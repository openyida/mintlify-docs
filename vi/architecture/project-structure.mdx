# Cấu Trúc Dự Án

OpenYida áp dụng thiết kế kiến trúc mô-đun với tổ chức mã nguồn rõ ràng, thuận tiện cho việc bảo trì và mở rộng.

## Mô Tả Mô-đun Cốt Lõi

### src/commands/

Lưu trữ triển khai tất cả lệnh CLI, mỗi tệp tương ứng với một lệnh con:

| Tệp | Chức năng |
|------|------|
| `create-app.ts` | Tạo ứng dụng Yida |
| `create-form.ts` | Tạo trang biểu mẫu |
| `create-page.ts` | Tạo trang tùy chỉnh |
| `publish.ts` | Xuất bản ứng dụng |
| `share.ts` | Chia sẻ ứng dụng |
| `data.ts` | Thao tác quản lý dữ liệu |
| `connector.ts` | Quản lý connector |
| `report.ts` | Thao tác báo cáo |
| `doctor.ts` | Công cụ chẩn đoán |
| `login.ts` | Xác thực đăng nhập |

### src/core/

Mô-đun logic nghiệp vụ cốt lõi:

- **yida-api.ts**: Đóng gói API nền tảng mở Yida, cung cấp gọi giao diện thống nhất
- **ai-service.ts**: Tích hợp nhiều dịch vụ AI (OpenAI, Claude, Thông Nghĩa Thiên Vấn, v.v.)
- **config.ts**: Quản lý tệp cấu hình cục bộ (~/.openyida/config.json)
- **auth.ts**: Xử lý trạng thái đăng nhập, làm mới Token, chuyển đổi nhiều tổ chức

### src/utils/

Hàm tiện ích chung:

- **logger.ts**: Xuất log, hỗ trợ phân cấp và màu sắc
- **file.ts**: Hàm hỗ trợ thao tác tệp
- **validator.ts**: Xác thực và kiểm tra đầu vào

### src/types/

Định nghĩa kiểu TypeScript, đảm bảo an toàn kiểu mã nguồn:

- **yida.ts**: Kiểu liên quan đến API Yida
- **ai.ts**: Kiểu liên quan đến dịch vụ AI
- **index.ts**: Xuất kiểu chung

## Quản Lý Cấu Hình

### Cấu Hình Cục Bộ

Vị trí lưu trữ: `~/.openyida/config.json`

```json
{
  "defaultOrg": "org-default",
  "orgs": {
    "org-default": {
      "accessToken": "xxx",
      "refreshToken": "xxx",
      "expiresAt": 1234567890
    }
  },
  "aiProvider": "openai",
  "aiConfig": {
    "apiKey": "sk-xxx",
    "model": "gpt-4"
  }
}
```

### Cấu Hình Dự Án

Có thể tạo `.openyidarc.json` trong thư mục gốc dự án:

```json
{
  "appId": "APP-XXX",
  "defaultEnv": "dev",
  "environments": {
    "dev": { "appKey": "xxx", "appSecret": "xxx" },
    "prod": { "appKey": "xxx", "appSecret": "xxx" }
  }
}
```

## Cơ Chế Mở Rộng

### Thêm Lệnh Mới

1. Tạo tệp mới trong `src/commands/`
2. Triển khai hàm xử lý lệnh
3. Đăng ký lệnh trong `src/index.ts`

### Thêm Skill Mới

1. Tạo thư mục Skill trong `skills/custom/`
2. Triển khai `skill.json` và logic chính
3. Tải bằng `openyida skill load`

## Quy Chuẩn Phát Triển

- Tất cả chức năng mới phải bao gồm định nghĩa kiểu
- Triển khai lệnh phải bao gồm tài liệu `--help`
- Chức năng cốt lõi phải có độ phủ kiểm thử đơn vị
- Tuân thủ cấu hình ESLint và Prettier
