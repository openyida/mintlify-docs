## Tạo Trình Kết Nối Thông Minh

Tạo trình kết nối chỉ với một lệnh curl:

```bash
openyida connector smart-create \
  --curl "curl 'https://api.example.com/v1/users' -H 'Authorization: Bearer xxx'" \
  --name "Trình Kết Nối Quản Lý Người Dùng" \
  --desc "Trình kết nối HTTP để quản lý dữ liệu người dùng"
```

Quy trình tạo thông minh:

1. **Phân tích thông tin API** - Trích xuất giao thức, Host, phương thức, đường dẫn, loại xác thực
2. **Khớp với trình kết nối hiện có** - Tìm trình kết nối hiện có cùng Host và xác thực
3. **Tạo cấu hình trình kết nối** - Tự động xây dựng định nghĩa trình kết nối định dạng OpenAPI
4. **Tạo hoặc cập nhật** - Tạo trình kết nối mới hoặc thêm action vào trình kết nối hiện có

## Lệnh Quản Lý Trình Kết Nối

### Liệt Kê Trình Kết Nối

```bash
openyida connector list
openyida connector list --page 1 --size 20
```

### Tạo Trình Kết Nối

```bash
openyida connector create "Tên Trình Kết Nối" "api.example.com" \
  --operations ./operations.json \
  --protocol https \
  --desc "Mô tả trình kết nối"
```

<Note>
Trường `method` trong tệp JSON operations được tự động chuẩn hóa thành chữ thường (ví dụ: `"POST"` sẽ trở thành `"post"`). Nếu không chỉ định `method`, giá trị mặc định là `"get"`.
</Note>

### Xem Chi Tiết Trình Kết Nối

```bash
openyida connector detail <connector-id>
```

### Xóa Trình Kết Nối

```bash
openyida connector delete <connector-id> --force
```

## Quản Lý Action

### Thêm Action Thực Thi

```bash
openyida connector add-action \
  --connector-id <id> \
  --operations ./new-action.json \
  --confirm
```

<Note>
Trường `method` trong cấu hình operations được tự động chuẩn hóa thành chữ thường (ví dụ: `"POST"` sẽ trở thành `"post"`). Bạn có thể sử dụng chữ hoa hoặc chữ thường. Nếu không chỉ định, giá trị mặc định là `"get"`.
</Note>

### Liệt Kê Action Hiện Có

```bash
openyida connector list-actions <connector-id>
```

### Kiểm Tra Action

```bash
openyida connector test \
  --connector-id <id> \
  --action <actionId> \
  --params '{"key": "value"}'
```

### Xóa Action

```bash
openyida connector delete-action <connector-id> <operation-id> --force
```

## Định dạng JSON operations

Flag `--operations` nhận một tệp JSON định nghĩa cấu hình action. Mỗi action trong mảng bao gồm `inputs` mô tả các tham số yêu cầu được nhóm theo vị trí gửi.

### Nhóm đầu vào

| Tên nhóm | Giá trị `paramLocation` | Mô tả |
|---|---|---|
| `Headers` | `header` | Header yêu cầu HTTP |
| `Body` | `body` | Trường body yêu cầu (POST/PUT) |
| `Query` | `query` | Tham số truy vấn URL |

### Trường `paramLocation`

Mỗi nhóm đầu vào và mỗi mục trong `childList` của nó phải bao gồm trường `paramLocation` cho biết tham số được gửi ở đâu trong yêu cầu HTTP. Các giá trị hợp lệ là `"header"`, `"body"` và `"query"`.

### Ví dụ JSON operations

```json
[
  {
    "operationId": "queryUsers",
    "summary": "Truy vấn người dùng",
    "description": "Truy vấn dữ liệu người dùng với phân trang",
    "url": "v1/users",
    "method": "post",
    "inputs": [
      {
        "name": "Headers",
        "paramType": "Object",
        "desc": "Header yêu cầu",
        "required": false,
        "paramLocation": "header",
        "childList": [
          {
            "name": "Content-Type",
            "paramType": "String",
            "desc": "Content-Type",
            "required": false,
            "defaultValue": "application/json",
            "paramLocation": "header"
          }
        ]
      },
      {
        "name": "Body",
        "paramType": "Object",
        "desc": "Body yêu cầu",
        "required": false,
        "paramLocation": "body",
        "childList": [
          {
            "name": "pageSize",
            "paramType": "Number",
            "desc": "pageSize",
            "required": false,
            "paramLocation": "body"
          }
        ]
      },
      {
        "name": "Query",
        "paramType": "Object",
        "desc": "Tham số truy vấn",
        "required": false,
        "paramLocation": "query",
        "childList": [
          {
            "name": "appType",
            "paramType": "String",
            "desc": "appType",
            "required": false,
            "paramLocation": "query"
          }
        ]
      }
    ],
    "parameters": {
      "header": [{ "name": "Content-Type", "value": "application/json" }],
      "body": { "default": "{\"pageSize\": 20}" },
      "query": [{ "name": "appType", "value": "" }]
    },
    "responses": { "type": "object", "properties": {} },
    "outputs": [
      {
        "name": "Response",
        "paramType": "Object",
        "desc": "Cấu trúc body phản hồi",
        "required": false,
        "childList": []
      }
    ]
  }
]
```

<Warning>
Trường `paramLocation` là bắt buộc trên cả nhóm đầu vào (ví dụ: `Headers`, `Body`, `Query`) và mỗi mục trong `childList` của nó. Thiếu trường này có thể khiến trình chỉnh sửa action của trình kết nối không hiển thị hoặc việc gọi trình kết nối thất bại.
</Warning>

## Thực Hành Tốt Nhất

### 1. Sử Dụng Biến Môi Trường Cho Thông Tin Nhạy Cảm

```bash
export API_TOKEN="token-bi-mat-cua-ban"
openyida connector smart-create --curl "curl -H 'Authorization: Bearer $API_TOKEN' ..."
```

### 2. Tạo Trình Kết Nối Phức Tạp Theo Từng Giai Đoạn

```bash
# Bước 1: Tạo trình kết nối cơ bản
openyida connector create "API DingTalk" "oapi.dingtalk.com"

# Bước 2: Thêm action từng cái một
openyida connector add-action --connector-id <id> --operations ./action1.json
openyida connector add-action --connector-id <id> --operations ./action2.json
```

### 3. Kiểm Tra Trước Khi Kích Hoạt

```bash
# Tạo nhưng không xác nhận
openyida connector add-action --connector-id <id> --operations ./action.json

# Kiểm tra action
openyida connector test --connector-id <id> --action <actionId>

# Xác nhận生效
# Hoàn thành xác nhận cuối cùng trong backend Yida
```

## Câu Hỏi Thường Gặp

<AccordionGroup>
  <Accordion title="Phải làm gì nếu phân tích lệnh curl thất bại?">
    Đảm bảo lệnh curl có định dạng đúng, bao gồm URL đầy đủ và headers cần thiết. Nếu chứa ký tự đặc biệt, hãy dùng dấu ngoặc kép.
  </Accordion>

  <Accordion title="Làm thế nào để cập nhật action của trình kết nối hiện có?">
    Sử dụng lệnh `connector add-action`, nếu ID action đã tồn tại, hệ thống sẽ hỏi bạn có muốn cập nhật không.
  </Accordion>

  <Accordion title="Hỗ trợ những phương thức xác thực nào?">
    Hiện tại hỗ trợ Bearer Token, API Key, Basic Auth, OAuth2. Thông tin xác thực trong lệnh curl được nhận diện tự động.
  </Accordion>
</AccordionGroup>
