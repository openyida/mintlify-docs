## Định dạng lệnh

```bash
openyida data <action> <resource> [args]
```

- **action**: `query` | `get` | `create` | `update` | `execute`
- **resource**: `form` | `process` | `task` | `subform` | `operation-records`

---

## Quản lý dữ liệu biểu mẫu

### Truy vấn dữ liệu biểu mẫu

```bash
# Truy vấn cơ bản
openyida data query form <appType> <formUuid>

# Truy vấn phân trang
openyida data query form <appType> <formUuid> --page 1 --size 50

# Truy vấn có điều kiện
openyida data query form <appType> <formUuid> \
  --search-json '{"field_xxx": "value"}'

# Truy vấn phiên bản cụ thể
openyida data query form <appType> <formUuid> --inst-id <formInstId>
```

### Lấy một bản ghi

```bash
openyida data get form <appType> --inst-id <formInstId>
```

### Tạo dữ liệu

```bash
openyida data create form <appType> <formUuid> \
  --data '{"field_xxx": "value", "field_yyy": 123}' \
  --dept-id <departmentId>
```

### Cập nhật dữ liệu

```bash
openyida data update form <appType> \
  --inst-id <formInstId> \
  --data '{"field_xxx": "new value"}' \
  --use-latest-version y
```

## Quản lý dữ liệu quy trình

### Truy vấn phiên bản quy trình

```bash
# Truy vấn danh sách quy trình
openyida data query process <appType> <formUuid> \
  --page 1 --size 20 \
  --instance-status RUNNING

# Trạng thái khả dụng: RUNNING (đang chạy), COMPLETED (đã hoàn thành), TERMINATED (đã chấm dứt)
```

### Lấy chi tiết quy trình

```bash
openyida data get process <appType> --process-inst-id <processInstanceId>
```

### Tạo phiên bản quy trình

```bash
openyida data create process <appType> <formUuid> \
  --process-code <processCode> \
  --data '{"field_xxx": "value"}' \
  --dept-id <departmentId>
```

### Truy vấn bản ghi phê duyệt

```bash
openyida data query operation-records <appType> \
  --process-inst-id <processInstanceId>
```

## Quản lý tác vụ

### Truy vấn danh sách tác vụ

```bash
# Tác vụ đang chờ
openyida data query tasks <appType> --type todo

# Tác vụ đã hoàn thành
openyida data query tasks <appType> --type done

# Do tôi gửi
openyida data query tasks <appType> --type submitted

# CC cho tôi
openyida data query tasks <appType> --type cc
```

### Thực thi tác vụ

```bash
openyida data execute task <appType> \
  --task-id <taskId> \
  --process-inst-id <processInstanceId> \
  --out-result AGREE \
  --remark "Đã đồng ý, vui lòng tiếp tục" \
  --data '{"comment": "Phê duyệt đã thông qua"}'
```

**Tùy chọn out-result:**
- `AGREE` - Đồng ý
- `DISAGREE` - Không đồng ý

## Quản lý biểu mẫu con

### Truy vấn dữ liệu biểu mẫu con

```bash
openyida data query subform <appType> <formUuid> \
  --inst-id <formInstId> \
  --table-field-id <fieldId> \
  --page 1 --size 20
```

## Sử dụng nâng cao

### Điều kiện truy vấn phức tạp

```bash
# Truy vấn có điều kiện nhiều trường
openyida data query form <appType> <formUuid> \
  --search-json '{
    "field_status": "approved",
    "field_amount": { "$gte": 1000 },
    "field_created_at": { "$gte": "2024-01-01" }
  }'
```

### Ví dụ script thao tác hàng loạt

```bash
#!/bin/bash
# Xuất dữ liệu biểu mẫu hàng loạt

APP_TYPE="APP_XXXXXXXX"
FORM_UUID="FORM-XXXX-XXXX"
PAGE=1

while true; do
  RESULT=$(openyida data query form $APP_TYPE $FORM_UUID --page $PAGE --size 100)
  
  # Xử lý dữ liệu trang hiện tại
  echo "$RESULT" >> ./export.json
  
  # Kiểm tra xem có trang tiếp theo không
  HAS_MORE=$(echo "$RESULT" | jq '.hasMore')
  if [ "$HAS_MORE" != "true" ]; then
    break
  fi
  
  PAGE=$((PAGE + 1))
done
```

## Ví dụ xuất dữ liệu

```bash
# Xuất dữ liệu trong khoảng thời gian cụ thể
openyida data query form <appType> <formUuid> \
  --search-json '{
    "field_created_at": {
      "$gte": "2024-01-01T00:00:00",
      "$lte": "2024-12-31T23:59:59"
    }
  }' \
  --size 100 > ./data-export.json
```

## Lưu ý

<Warning>
  Thao tác dữ liệu là vĩnh viễn, thao tác xóa và cập nhật không thể hoàn tác. Khuyến nghị thực hiện thử nghiệm quy mô nhỏ trước khi thực hiện thao tác hàng loạt.
</Warning>

<Info>
  Truy vấn phân trang mặc định 20 bản ghi mỗi trang, tối đa 100. Để xuất lượng lớn dữ liệu, khuyến nghị sử dụng vòng lặp phân trang.
</Info>
