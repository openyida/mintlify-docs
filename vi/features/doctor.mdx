## Sử Dụng Cơ Bản

### Chạy Chẩn Đoán

```bash
openyida doctor
```

### Sửa Chữa Tự Động

```bash
openyida doctor --fix
```

### Tạo Báo Cáo

```bash
# Định dạng Markdown
openyida doctor --report markdown

# Định dạng JSON
openyida doctor --report json

# Định dạng HTML
openyida doctor --report html
```

## Mục Kiểm Tra Môi Trường

| Mục | Mô Tả | Sửa Chữa Tự Động |
|--------|------|----------|
| Phiên bản Node.js | Kiểm tra nếu ≥ 16 |❌
| Phiên bản Python | Kiểm tra nếu ≥ 3.10 |❌
| Playwright | Kiểm tra nếu đã cài đặt | ✅ |
| Chromium | Kiểm tra nếu trình duyệt đã cài đặt | ✅ |
| gh CLI | Kiểm tra GitHub CLI |❌
| Đăng nhập gh | Kiểm tra trạng thái đăng nhập gh CLI | ✅ |
| config.json | Kiểm tra tệp cấu hình | ✅ |
| Skills | Kiểm tra cài đặt gói kỹ năng |❌
| Đăng nhập Yida | Kiểm tra trạng thái đăng nhập | ✅ |
| Kết nối mạng | Kiểm tra kết nối aliwork.com |❌

## Mục Chẩn Đoán Ứng Dụng

| Mục | Mô Tả |
|--------|------|
| Tệp PRD | Kiểm tra nếu tồn tại tài liệu yêu cầu |
| Mã nguồn trang | Kiểm tra tệp mã nguồn, console.log, tệp trống |
| Bộ nhớ đệm Schema | Kiểm tra định dạng tệp bộ nhớ đệm |
| React Hooks | Kiểm tra quy chuẩn sử dụng Hooks |

## Chế Độ Giám Sát Sức Khỏe

Khởi động giám sát liên tục, kiểm tra mỗi phút:

```bash
openyida doctor --monitor
```

<Warning>
Tính năng giám sát sức khỏe hiện đang ở giai đoạn beta. API và định dạng đầu ra có thể thay đổi trong các phiên bản tương lai.
</Warning>

Ví dụ đầu ra:

```
[14:32:01] Sức khỏe: 95/100  Đạt: 18/19 | Lỗi: 0 | Cảnh báo: 1

[14:33:01] Sức khỏe: 100/100➡
  Đạt: 19/19 | Lỗi: 0 | Cảnh báo: 0
```

Nhấn `Ctrl+C` để dừng giám sát.

## Chẩn Đoán Trực Tuyến

Chẩn đoán trực tuyến cho ứng dụng đã xuất bản:

```bash
openyida doctor --production --app <appId>
```

Nội dung kiểm tra:
- Tính hợp lệ của ID ứng dụng
- Nhật ký lỗi trực tuyến
- Chỉ số hiệu suất

## Ticket và VOC

### Tạo Ticket (Bug)

```bash
openyida doctor --create-ticket
```

Nhập tiêu đề và mô tả tương tác, tự động gửi đến GitHub Issues.

### Tạo VOC (Yêu cầu)

```bash
openyida doctor --create-voc
```

### Gửi Thông Minh

Hệ thống tự động phán đoán là Bug hay yêu cầu:

```bash
openyida doctor --auto-submit
```

## Kiểm Tra Trước

Tự động thực hiện kiểm tra trước các thao tác quan trọng:

### Kiểm Tra Trước Khi Xuất Bản

```bash
# Tự động thực hiện
openyida publish tự động chạy kiểm tra trước
```

### Kiểm Tra Trước Khi Tạo

```bash
# Tự động thực hiện
openyida create-app tự động chạy kiểm tra trước
```

## Loại Sửa Chữa

| Loại | Mô Tả | Ví Dụ |
|------|------|------|
| `AUTO` | Sửa chữa tự động | Tạo config.json, xóa bộ nhớ đệm hỏng |
| `COMMAND` | Sửa chữa bằng lệnh | Chạy `pip install playwright` |
| `MANUAL` | Sửa chữa thủ công | Cài đặt gh CLI, viết PRD |

## Tính Toán Sức Khỏe

Công thức tính điểm sức khỏe (0-100):

```
Sức khỏe = Tỷ lệ đạt × 100 - Lỗi × 10 - Cảnh báo × 3
```

| Điểm | Trạng Thái |
|------|------|
| 90-100 | Xuất sắc
| 70-89 | Tốt
| 50-69 | Bình thường
| < 50 | Cần cải thiện🔴
