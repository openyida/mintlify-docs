OpenYida 支持使用 `create-report` 和 `append-chart` 命令創建宜搭報表並添加圖表，實現數據可視化。

## 創建報表

### 基本用法

```bash
openyida create-report <appType> "報表名稱" '<圖表定義JSON或檔案路徑>'
```

第三個參數可以是內聯 JSON 字串或 JSON 檔案路徑。

### 示例：創建銷售報表

```bash
openyida create-report APP_XXXXXX "Sales Dashboard" '{
  "charts": [
    {
      "type": "bar",
      "title": "Monthly Sales",
      "dataSource": "sales_form",
      "xField": "month",
      "yField": "amount"
    }
  ]
}'
```

### 從檔案載入圖表定義

```bash
openyida create-report APP_XXXXXX "Annual Report" ./charts-config.json
```

## 向報表追加圖表

```bash
openyida append-chart <appType> <reportId> '<圖表定義JSON或檔案路徑>'
```

### 示例

```bash
openyida append-chart APP_XXXXXX REPORT_XXXXX '{
  "type": "line",
  "title": "Growth Trend",
  "dataSource": "form_sales",
  "xField": "month",
  "yField": "growth_rate"
}'
```

## 圖表類型

| 類型 | 說明 | 適用場景 |
|------|------|----------|
| **bar** | 柱狀圖 | 分類比較 |
| **line** | 折線圖 | 時間趨勢 |
| **pie** | 餅圖 | 部分與整體關係 |
| **table** | 數據表格 | 詳細數據查看 |
| **area** | 面積圖 | 累積趨勢 |
| **scatter** | 散點圖 | 相關性分析 |

## 圖表定義格式

### 柱狀圖

```json
{
  "type": "bar",
  "title": "Sales by Region",
  "dataSource": "form_uuid",
  "xField": "region",
  "yField": "amount",
  "aggregation": "sum"
}
```

### 折線圖

```json
{
  "type": "line",
  "title": "Traffic Trend",
  "dataSource": "form_uuid",
  "xField": "date",
  "yField": "visitors",
  "aggregation": "sum"
}
```

### 餅圖

```json
{
  "type": "pie",
  "title": "Market Share",
  "dataSource": "form_uuid",
  "categoryField": "product",
  "valueField": "share"
}
```

### 數據表格

```json
{
  "type": "table",
  "title": "Detail View",
  "dataSource": "form_uuid",
  "columns": [
    { "field": "name", "title": "Name" },
    { "field": "amount", "title": "Amount" },
    { "field": "date", "title": "Date" }
  ]
}
```

## 聚合函數

| 函數 | 說明 |
|----------|-------------|
| **sum** | 求和 |
| **avg** | 平均值 |
| **count** | 計數 |
| **max** | 最大值 |
| **min** | 最小值 |

## 完整配置示例

### sales-report-charts.json

```json
{
  "charts": [
    {
      "type": "bar",
      "title": "Monthly Sales",
      "dataSource": "form_monthly_sales",
      "xField": "month",
      "yField": "sales_amount",
      "aggregation": "sum"
    },
    {
      "type": "pie",
      "title": "Product Categories",
      "dataSource": "form_sales",
      "categoryField": "product_category",
      "valueField": "sales_amount"
    },
    {
      "type": "line",
      "title": "Sales Trend",
      "dataSource": "form_daily_sales",
      "xField": "date",
      "yField": "sales_amount",
      "aggregation": "sum"
    },
    {
      "type": "table",
      "title": "Sales Detail",
      "dataSource": "form_sales",
      "columns": [
        { "field": "order_no", "title": "Order No.", "width": 150 },
        { "field": "customer_name", "title": "Customer", "width": 200 },
        { "field": "product_name", "title": "Product", "width": 200 },
        { "field": "amount", "title": "Amount", "width": 120 },
        { "field": "sales_date", "title": "Date", "width": 150 }
      ],
      "pageSize": 10
    }
  ]
}
```

創建報表：

```bash
openyida create-report APP_XXXXXX "Sales Analysis Report" ./sales-report-charts.json
```

## 最佳實踐

1. **合理選擇圖表類型** - 趨勢用折線圖、對比用柱狀圖、佔比用餅圖
2. **控制圖表數量** - 單個報表建議不超過 6 個圖表
3. **使用數據表格** - 配合圖表展示詳細數據，方便查看原始值
4. **設置合適的分頁** - 數據表格建議每頁 10-20 條
