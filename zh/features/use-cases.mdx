OpenYida 适用于多种业务场景，以下是常见的应用类型示例。

---

## 🏢 企业业务系统

构建多模块业务平台，支持复杂表单、工作流和数据关联。

<AccordionGroup>
  <Accordion title="IPD（集成产品开发）">
    **提示词：**
    ```
    帮我用宜搭创建一个 IPD 系统，管理芯片生产全流程，
    包括项目立项、设计评审、流片、量产等阶段
    ```
    **产出：** 多表单应用，包含阶段跟踪、评审清单、里程碑管理、跨团队可见性。
  </Accordion>

  <Accordion title="CRM（客户关系管理）">
    **提示词：**
    ```
    帮我搭建一个 CRM，包含线索、客户、商机、跟进记录等模块
    ```
    **产出：** 关联表单：线索 → 客户 → 商机 → 合同，含状态跟踪和分配字段。
  </Accordion>

  <Accordion title="人事与请假管理">
    **提示词：**
    ```
    创建一个人事请假管理系统，包含请假申请、审批流程、假期余额统计
    ```
    **产出：** 请假申请表单含审批流程、余额计算器、经理看板。
  </Accordion>

  <Accordion title="项目管理">
    **提示词：**
    ```
    搭建一个项目管理应用，包含任务、里程碑、团队成员、进度跟踪
    ```
    **产出：** 任务列表含优先级/状态/负责人字段、里程碑跟踪器、甘特图视图。
  </Accordion>
</AccordionGroup>

## 🔌 API 集成（连接器）

通过 HTTP 连接器将宜搭应用与外部系统连接。

<AccordionGroup>
  <Accordion title="Webhook 通知">
    **提示词：**
    ```
    为我们的钉钉 webhook 创建一个 HTTP 连接器，添加 POST 动作发送表单提交通知
    ```
    **产出：** 配置好的连接器含测试能力，可在表单事件时触发。
  </Accordion>

  <Accordion title="从 curl 智能创建">
    **提示词：**
    ```
    从这个 curl 命令智能创建连接器：curl -X POST https://api.example.com/v1/users -H "Authorization: Bearer TOKEN" -d '{"name":"test"}'
    ```
    **产出：** 自动配置的连接器，提取基础 URL、请求头、动作参数。
  </Accordion>
</AccordionGroup>

<Note>
这些是起点。由于产出是可在宜搭平台完全编辑的应用（非黑盒配置），你随时可以通过宜搭可视化编辑器或给 AI 后续指令来优化生成的应用。
</Note>
