## 1. 编写详细提示词

生成应用的质量与提示词的详细程度成正比。对比：

<CodeGroup>

```text 模糊（避免）
创建一个 CRM
```

```text 详细（推荐）
创建一个 CRM，包含三个关联表单：
1. 线索表单 — 字段：姓名、公司、电话、邮箱、来源（下拉框：推荐/网站/活动/陌生拜访）、状态（下拉框：新线索/已联系/已合格/已 disqualify）、分配给（成员选择器）
2. 客户表单 — 从线索转化，字段：公司名称、行业、年收入、关键联系人、客户经理
3. 商机表单 — 关联客户，字段：商机名称、金额、阶段（下拉框：提案/谈判/赢单/输单）、预计成交日期、备注
```

</CodeGroup>

## 2. 使用需求文档

复杂应用先写 PRD，再交给 AI：

```
根据以下需求文档生成宜搭应用：

[粘贴 PRD — 字段名、类型、验证规则、工作流等]
```

这种方式比迭代提示词产生更准确的结果。

---

## 3. 增量迭代

不要试图在一个提示词中构建所有内容。先构建核心结构，再逐步添加复杂度：

```
第 1 步：创建基础员工信息表单
第 2 步：添加关联员工的请假申请表单
第 3 步：配置请假审批流程
第 4 步：添加显示待审批的经理看板
第 5 步：发布看板为公开页面
```

---

## 4. 每步验证

AI 创建每个组件后，让它验证：

```
显示 [表单名] 表单的当前 Schema
检查 [页面名] 页面是否已发布并可访问
```

及早发现问题，避免累积成大问题。

---

## 5. 复用项目工作目录

每个项目只运行一次 `openyida copy`，不要每会话都运行。`project/config.json` 文件存储 `appId` 和 `corpId` —— AI 用它知道目标应用。项目中途不要删除此文件。

---

## 6. 环境特定登录

不同 AI 工具需要不同登录方式：

| AI 工具 | 登录命令 |
|---------|----------|
| Claude Code、Cursor、VSCode | `openyida login` |
| 悟空 | `openyida login --wukong` |
| OpenCode、Aone Copilot | `openyida login` |

切换工具时，先运行 `openyida env` 确认激活环境。

## 7. 自定义页面图片使用 CDN

自定义页面含图片时，先上传到 CDN 确保可靠渲染：

```bash
openyida cdn-config   # 配置阿里云 OSS + CDN（一次性）
openyida cdn-upload ./my-image.png   # 获取稳定 CDN URL
```

然后在页面提示词中引用 CDN URL：
```
使用此 CDN URL 给落地页添加首屏横幅图片：https://cdn.example.com/banner.png
```

## 8. 处理多组织设置

可访问多个宜搭组织时：

```bash
openyida org list          # 查看所有可用组织
openyida org switch --corp-id <id>   # 切换到目标组织
openyida env               # 确认切换
```

创建应用前始终验证激活组织，避免在错误空间构建。

---

## 9. 调试前先运行 Doctor

花时间调试失败命令前，先运行：

```bash
openyida doctor --fix
```

自动检测并解决最常见问题：过期会话、缺失配置文件、环境不匹配。

## 10. AI 辅助 PR 欢迎

向 OpenYida 本身贡献时，完全接受 AI 生成的代码。在 PR 描述中注明使用的 AI 工具 —— 帮助维护者理解生成上下文，更高效地 review。
